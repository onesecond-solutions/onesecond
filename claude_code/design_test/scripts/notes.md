## v2 (2026-04-25) — 세로 탭바 + 사이드바 180px 축소

### 🛠️ v2 후속 수정 (2026-04-25 심야)

**피드백 반영 2건**:

1. **세로 탭 활성 색상이 밋밋함** → 사이드바 활성 버튼 수준의 임팩트로 강화
   - 기존: `background: var(--stg-bg)` (옅은 단계 배경), 텍스트는 단계 fg 색
   - 수정: `background: var(--stg-fg)` **solid fill** + `color: #fff` + `translateX(3px)` + 강한 그림자
   - 이모지 아이콘에 `filter: brightness(0) invert(1)` → 흰색 실루엣으로 통일감
   - 카운트 배지: `background: rgba(255,255,255,0.22)` 반투명 흰색

2. **원본의 슬라이드 애니메이션 효과 누락** → STEP 전환 기반 열 폭 애니메이션 복원
   - `.sc-grid` 기본 = STEP 1 (`3fr 1fr 1fr`, col-2 최대)
   - `.sc-grid.step-2` = col-3 최대 (`1fr 3.5fr 2fr`)
   - `.sc-grid.step-3` = col-4 최대 (`0.8fr 1fr 5fr`) — 본문 읽기 상태
   - 전환 시 `transition: grid-template-columns 0.35s ease` 로 부드러운 슬라이드
   - `selectScript()` / `selectBlock()` JS 함수로 인터랙션 활성
   - col-4 본문 HTML은 DOMContentLoaded 시점 캐시 후 STEP 3 전환 시 복원

**데모 흐름**:
- 진입 시 STEP 3 (본문 읽기 상태) 바로 보임
- col-2에서 **다른 스크립트 클릭** → STEP 3 → STEP 2 슬라이드 (col-4 접히며 col-3 확장, placeholder 표시)
- col-3에서 **블록 클릭** → STEP 2 → STEP 3 슬라이드 (col-4 확장, 본문 타이틀이 선택 블록으로 동기화)

- 가로 탭바는 좁은 화면에서 줄바꿈 이슈, 세로는 10단계 진행감을 직관적으로 전달
- 아이콘 레일(60px) 은 한국어 가독성 문제로 배제 → **텍스트 유지 + 최소 폭** 절충

### 어떻게

#### 1. B 사이드바 180px 축소

- `--sidebar-width: 220px → 180px`
- `.b` 외곽 패딩: `24px 16px → 24px 12px`
- `.menu-item` 패딩: `14px 16px → 12px 14px`
- `white-space: nowrap` 추가로 2줄 방지
- "함께해요" 좌측 보더 포함 `padding-left: 11px`

#### 2. 세로 탭바 (신규) `.sc-vtab-*`

- 180px 고정 폭 (반응형 1200px↓ 160px, 900px↓ 가로 래핑)
- 각 탭: 아이콘 + 라벨 + 카운트 배지
- 활성: 좌측 보더(단계 색) + 옅은 단계 배경 + 텍스트 볼드/색상
- `position: sticky`로 스크롤 시에도 항상 보이게

#### 3. 3열 그리드 (col-1 제거)

- v1의 col-1 단계 셀은 세로 탭바와 **중복** → 삭제
- `.sc-grid`: `130px 0.8fr 1.0fr 5.0fr` (4열) → `1fr 1.2fr 5fr` (3열)
- col-4(본문)가 차지하는 비율이 v1과 유사하게 유지됨 (약 73%)

#### 4. 색상·여백 원칙 (v1과 동일)

- 브라운 최소화 (chrome 중성화)
- 단계 팔레트 10색 유지
- 여백 확대 원칙 유지

## v1 (2026-04-25)

### 무엇을

`pages/scripts.html`의 STEP 3 상태(4열 모두 채워진 본문 읽기 중)를 정적 목업으로 재현 + v1 원칙 적용.

### 왜

- 원세컨드의 핵심 페이지 중 하나. 홈을 제외한 페이지들이 촘촘해 보인다는 팀장님 지적 검증 대상
- 10단계 브라운 그라데이션 팔레트가 어떻게 보일지 중성 chrome 위에서 확인 필요

### 어떻게

#### 1. 브라운 최소화 (chrome 에만)

| 요소 | 기존 | v1 |
|---|---|---|
| 페이지 배경·body 텍스트 | 크림 + 다크브라운 | 순백 + 차콜 |
| 보더·표면 2단계 | 웜 베이지 | 뉴트럴 그레이 |
| 탭바·열 그림자 | 웜 브라운 톤 `rgba(101,66,38,...)` | 뉴트럴 `rgba(17,24,39,...)` |
| `.sc-col.now` 배경 | `#FFFBF5` (옅은 크림) | `#FAFBFC` (옅은 뉴트럴) |
| `.sc-footer` 배경 | `--color-text-primary` (다크 브라운) | `--color-surface-2` + 보더 |

#### 2. 단계 팔레트 유지 (도메인 특화)

`--stg-1-fg` ~ `--stg-10-fg` 10단계 브라운 그라데이션은 **값 그대로 유지**.
- CLAUDE.md 명시적 예외 허용
- 단계별 상담 강도를 시각적으로 전달하는 도메인 의미
- 브라운이지만 "강조 포인트"로 분류됨 (활성 탭·활성 list-item·detail-tag·본문 strong)

활성 상태에서 사용:
- 탭 언더라인 색상 + 활성 배경
- 열 1 단계 셀 배경 + 보더
- 열 2·3 활성 항목 좌측 보더 + 타이틀 색상
- 열 4 detail-tag 배경
- 본문 `<strong>` 강조 색상
- `.sc-tip` 좌측 보더 + 배경

#### 3. 여백 확대 (촘촘함 해소 — 홈 외 원칙 첫 적용)
