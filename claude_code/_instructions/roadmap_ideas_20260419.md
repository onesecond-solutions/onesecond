## 🎯 관통 가치 (한 줄로)

**"기존 프로그램을 베끼지 않았다. 17년 현장 경험 + 함께해요 커뮤니티 공동 제작으로 처음부터 다시 만들었다."**

이것이 경쟁사가 절대 못 따라오는 원세컨드의 진짜 해자(垓字).

### 왜 이 가치가 중요한가

- 일반 개발사: 기존 CRM·보장분석 베끼기 → 기존 틀에서 못 벗어남
- 원세컨드: 백지에서 **17년 현장이 원하는 구조**로 설계
- "함께해요" 공간과 연결 → **사용자 의견 모아서 반영** → 진짜 공동 제작

### 투자자·원수사 미팅용 한 줄

> "우리는 기존 CRM·보장분석·카톡 툴을 참고하지 않습니다. 17년 TM·대면 영업 경험과 '함께해요' 커뮤니티 공동 제작으로 현장이 진짜 원하는 도구를 만듭니다."

## 🗺️ 기능별 상세 기획

### 1. 카카오톡 연동 (v2.0~v2.5)

**한 줄:** 카톡 대화창에서 원세컨드를 바로 실행하는 연동 도구

**왜 1순위인가:**
- TM·보험 상담의 **이미 절반 이상이 카톡 기반**
- 경쟁사(텔레잡 등)가 카카오 API 승인 난이도로 못 따라옴
- 투자자·원수사 미팅 시 **가장 임팩트 큰** 시연 주제
- 현재 원세컨드: 통화+PC 분리 → 연동 후: 카톡창 안에서 모든 게 해결

**차별점:**
- 단순 메시지 자동 발송이 아니라 **대화 맥락 인식해서 적합 멘트 자동 추천**
- 고객이 카톡으로 "보험 있어요"라고 치면 → 원세컨드가 반론대응 스크립트 팝업
- 함께해요에서 쌓인 **실전 반론 DB 활용** → 시간 지날수록 정확해짐

**함께해요 연계:**
- "카톡에서 이런 반론 왔는데 어떻게 답했더니 좋더라" 같은 현장 사례 누적
- 반론 패턴 6유형 구조에 실제 데이터 붙이는 피드백 루프

**난이도:** 🔴 상
**소요 예상:** 3~4개월 + 카카오 심사 시간
**핵심 리스크:** 카카오 비즈 API 심사, 개인정보 보호 설계

---

### 2. 보장분석 도구 (v2.5~v3.0)

**한 줄:** 17년 현장 경험 기반으로, TM 통화 흐름에 맞춰 재설계한 보장분석 도구

**왜 2순위인가:**
- 상품 페이지·스크립트와 자연스럽게 연결
- 기존 보장분석 프로그램이 시장에 많지만 **대부분 대면 영업 기준**
- TM 통화 흐름에 맞춘 보장분석은 **아직 시장에 없음**
- 스크립트 시너지: "보장분석→상품설명" 흐름이 통합됨

**차별점:**
- 기존 보장분석: 설계사가 입력 → 고객 설득용 차트 출력
- 원세컨드 버전: **TM 통화 흐름에 맞춰 단계별 질문-응답 구조**
- 고객이 한 문장씩 답할 때마다 실시간으로 분석·추천
- 함께해요의 질문 패턴을 학습하여 정확도 올라감

**함께해요 연계:**
- "보장분석 질문 중 이게 가장 잘 먹히더라" 현장 지식 누적
- 시간대·연령대·직업별 반응 패턴 집단지성 구축

**난이도:** 🟡 중~상 (로직 설계 난이도 높음)
**소요 예상:** 1~2개월
**핵심 리스크:** 보장분석 결과 신뢰성, 원수사별 상품 DB 업데이트

---

### 3. CRM (v3.0)

**한 줄:** 영업관리자가 아닌 실제 전화 거는 설계사 입장에서 설계한 고객 관리 도구

**왜 3순위인가:**
- 개인정보 보호 이슈가 가장 무거움 (원세컨드의 "고객정보 저장 안 함" 포지셔닝과 충돌)
- 함께해요·스크립트·보장분석이 먼저 자리 잡아야 CRM이 의미 있음
- 가장 무거운 기능 → 가장 나중에 해도 무방

**차별점:**
- 대기업 CRM: 영업관리자 대시보드 중심 (통계·리포트)
- 팀장님 버전: **실제 전화 거는 설계사 편의** 중심
- 기존 CRM을 쓴 적 없다는 게 오히려 **백지 설계의 장점**
- 현장 감각으로만 설계 → 불필요한 기능 없음

**함께해요 연계:**
- "CRM에 이런 기능 있으면 좋겠어요" 기능 요청을 받아 반영
- 설계사가 스스로 원하는 필드·뷰를 커스터마이징

**난이도:** 🔴 상 (DB 테이블 다수, RLS 복잡)
**소요 예상:** 2~3개월
**핵심 리스크:** 개인정보 보호법 준수, "고객정보 저장 안 함" 포지셔닝과의 조화

## 💡 관리자 사이드바 "개발 중" 섹션 설계

배포 후 구현 예정. 옵션 A 방식.

```
─── 일반 메뉴 ───
🏠 홈
📝 스크립트
💬 게시판
🗂 MY SPACE
📰 보험뉴스
⚡ Quick 메뉴
🤝 함께해요

─── 관리자 전용 ───
🛡️ 관리자

─── 🚧 개발 중 (관리자만) ───
💬 카카오톡 연동        [기획]
📊 보장분석             [설계 중]
👥 CRM                  [대기]
🤖 AI Call Assistant    [장기]
```

### 각 메뉴 클릭 시
클릭하면 해당 기능의 **기획 페이지**가 열림:
- 기능 개요
- 함께해요 연계 방안
- 현재 진행 상태
- 참고 자료 (화면 목업, 흐름도)
- 팀장님이 수시로 편집 가능

### 상태 뱃지 정의
- **[기획]**: 아이디어 단계, 구조 설계 중
- **[설계 중]**: UI 목업 작업 중
- **[개발 중]**: 실제 코드 작성 중
- **[베타]**: 관리자 테스트 가능
- **[대기]**: 선행 기능 완성 대기
- **[장기]**: 1년 이상 후 예정
