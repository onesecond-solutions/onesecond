## 1. 제품 정체성

### 원세컨드란
- 보험 TM 설계사용 **상담 실행 도구**
- **교육 플랫폼 아님, CRM 아님, 고객 관리 도구 아님**

### 핵심 카피
- 메인: "통화 중 멈추는 순간, 원세컨드가 이어갑니다"
- 서브: "생각하지 말고, 선택하세요"

### 상담 흐름 7단계 (디지털화 대상)
도입 → 필요성 → 상황확인 → 보장분석 → 상품설명 → 클로징 → 반론대응

### 홈 화면 스크립트 구조 10단계
1. 📞 도입 인사 (opening)
2. 🗨️ 관심 형성·도입 반론 (opening_rejection)
3. 🔥 필요성 강조 1 (need_emphasis)
4. 🔍 고객 상황 확인 (situation_check)
5. 📊 보장 분석 (analysis)
6. 📦 상품 설명 (product)
7. 🔥 필요성 강조 2 (need_emphasis_2)
8. ✅ 클로징 (closing)
9. ⚡ 반론 대응 (objection)
10. 🔵 2차 클로징 (closing_second)

### 타깃
- 신입 설계사 (학습·참고)
- 현장 설계사 (즉시 실행)

### 장기 비전 — "보험판 구글"
광고 기반 양면 시장 모델. 3단계 로드맵:
- 1단계 (1,000명): 상담 실행 도구

## 2. 절대 금지선

### 2-1. 고객정보 저장 금지
원세컨드는 **"고객정보를 저장하지 않는 내부 상담 도구"**. 이 포지셔닝은 블로그 영업 규제(보험업법·금소법) 경험의 교훈에서 나온 것.
- 고객 이름·연락처·주민번호·계약 정보 일체 저장 금지
- 저장이 아닌 "실행 도구" 방향성 유지

### 2-2. 보험업법·금소법 위반 콘텐츠 금지
- 보험업법 95조 광고규제, 금소법 22조, 소속사 내규 준수
- AI 콘텐츠 책임 이슈 사전 방어
- 애매하면 보수적 판단

### 2-3. 원세컨드 브리핑에서 T-Solution 확장 전략 언급 금지
- 브리핑·제휴·설명회에서는 원세컨드 범위까지만 설명
- T-Solution 확장 계획은 내부 전략 문서에서만

### 2-4. 미성년·아동 관련 부적절 콘텐츠 금지
어린이보험 등 정상 업무 문맥은 허용, 부적절한 맥락 생성 금지.

### 3-2. 앱 구조 (SPA)
- `app.html` = 고정 프레임 (A1/A2/B/C/D 영역)
- `pages/*.html` = D영역에 fetch()로 동적 로드
- child pages는 app.html의 IIFE 래핑으로 실행됨
- **이 구조 절대 변경 금지**

### 3-3. 코드 규칙 (위반 시 동작 불가)
1. **`app.html` / `app.js` 는 명시 요청 없이 수정 금지**
2. child pages의 모든 함수는 **`window.functionName` 전역 등록 필수** (IIFE 래핑 때문)
3. **`defer` 속성 금지** — 표준 부트 패턴은 `appstate:ready` 이벤트
4. **CSS는 `tokens.css` custom property만 사용** — 하드코딩 색상·radius 금지
5. **직각(0px, 2px) border-radius 금지** — 최소 `--radius-sm` (8px) 이상
6. **폰트 크기는 `em` 단위** — `px` 고정 금지
7. **E영역 콘텐츠 파일만 납품** — 프레임 파일은 건드리지 않음
8. `quick.html`은 IIFE 래핑 예외

### 3-4. 배포 루틴
1. Claude Code로 파일 수정
2. GitHub Desktop 열기 → 변경 파일 확인
3. Summary 입력 → Commit to main → Push origin
4. 1~2분 대기 (GitHub Pages 자동 배포)
5. 브라우저 `Ctrl+Shift+R` (강력 새로고침)

## 5. 권한 구조 (RBAC)

### 5-1. 현장 호칭 ↔ role 매핑
| 현장 호칭 | role | 구분 |
|---|---|---|
| 설계사, 팀장 | `member` | 팀장은 설계사 직군 내 직책 |
| 실장 | `manager` | 매니저 |
| 지점장, 센터장 | `branch_manager` | 지점 책임자 |
| 스텝, 총무 | `staff` | 운영 지원 |
| 보험사 담당자 | `insurer` | v2.0 이후 |
| 운영자 | `admin` | 시스템 관리자 |

### 5-2. PRO 접근 기준
```javascript
function isPro() {
  return plan === 'pro'
      || ['manager', 'branch_manager', 'admin'].includes(role);
}
```

### 5-3. 게시판 권한 (v1.1 기준)
| 게시판 | 설계사 | 매니저 이상 |
|---|---|---|
| 팀 공지 | 읽기만 | 읽기+쓰기 |
| 지점 공지 | 읽기만 | 읽기+쓰기 |
| 상품 Q&A | 읽기+쓰기 | 읽기+쓰기 |
| 인수 Q&A | 읽기+쓰기 | 읽기+쓰기 |
| 함께해요 | 읽기+쓰기 | 읽기+쓰기 |
| 보험사 게시판 (v2.0) | 읽기만 | 작성은 `insurer`·`admin`만 |

### 5-4. 관리자 예외 원칙 (중요)
**관리자(`admin`·`branch_manager`)는 자신이 설정한 숨김·제한을 본인에게 적용하지 않음.**

이유: 관리자는 설정을 하는 주체이므로, 본인이 숨긴 항목도 관리·복구를 위해 보여야 함.

예: 관리자가 "허브게시판 숨김" 설정해도, 관리자 본인 화면에서는 허브게시판 탭이 계속 보여야 함.

### 5-5. 1차 탭 고정 정책 (게시판)
- **팀게시판 / 지점게시판**: 항상 표시 (업무 핵심이므로 관리자 설정 불가)
- **허브게시판**: 관리자 토글 가능 (기본 표시)
- **보험사게시판**: 관리자 토글 가능 (기본 숨김 — 설계사 오해 방지)
