# 세션 시작 — START PROTOCOL + 오늘 작업 브리핑

## 본질

본 커맨드는 **두 자리 통째 정합**:
1. **본 세션 진입자(Code 본인)**가 START PROTOCOL 6단계 통째 점검 + 오늘 작업 브리핑 생성 → 팀장님 승인 받은 후 작업 시작
2. **AI 새 창 진입자(Claude AI 등)**에 통째 붙여넣을 컨텍스트 패키지 동시 생성

운영 원칙 (2026-05-24 신설, docs/README.md 정합):
- **sessions만 읽고 작업 시작 금지** — 현재 상태 판단 = strategy / decisions / work_orders / product 통째 기준
- **최근 세션 몇 개만 보고 전체 방향 판단 금지** — 큰 그림 = strategy + decisions
- **승인 전 작업 진입 금지** — 브리핑 본 후 팀장님 결재 받은 후 진입

```
git branch --show-current

```

미머지 브랜치 0건 → "없음" 기재.

### 3단계: START PROTOCOL REPORT 자동 생성 (8섹션)

2단계 통째 점검 후 본인이 통째 종합 판단 → **8섹션 양식**으로 작성. 전체 템플릿 + 데이터 소스 매핑 + Code 초안/Web 보강 구분 = `docs/ops/session_report_format.md` (진실 원천).

```
═══════════════════════════════
START PROTOCOL REPORT — {YYYY-MM-DD} / {HHMM}
═══════════════════════════════
[운영 규칙] 유휴 0 · 증거=push · 원장=진실 · main직접push 금지

[1. 프로젝트 상태판]   (트랙·상태·진행률·우선순위·이월 테이블 + 채굴팀 가동 상태 🔴/🟢)
[2. 트랙별 상세]       (완료(증거 PR#·해시)·진행중·미완료·차단·원칙정합)
[3. 결재 필요 사항]    ⚠️ Code 초안만 — 최종 Web(전략) 보강
[4. 리스크·충돌 예측]  ⚠️ Code 초안만 — 최종 Web(전략) 보강
[5. 향후 48시간 실행계획]   (D+0/D+1/D+2, 병행 트랙 ‖ 표기)
[6. 예정 작업 (Backlog)]
[7. 총괄 판단]         ⚠️ Code 초안만 — 최종 Web(전략) 보강 (★유휴 0 적발 포함)
[8. 경비 현황]         (docs/ops/cost_ledger.md 기반)
═══════════════════════════════
브리핑 결재 받은 후 작업 시작합니다.
```

**데이터 소스 매핑:**

- [1] 상태판 ← 트랙 진행률 + `git branch`
- [2] 완료(증거) ← `git log` (PR#·해시)
- 브랜치 ← `git branch -a` / 세션 ← `docs/sessions/` 최신 / 할 일 ← `docs/work_orders/`·`backlog.md`
- 채굴팀 가동 상태 ← 직전 세션 대비 지식엔진 적재 진행 여부 (멈춰 있으면 🔴)

**Code 초안 vs Web 보강:** [3]결재·[4]리스크·[7]총괄판단 = Code **초안만** / 최종 보강 = Web(총괄 기획자, 전략). [1][2][5][6][8] = Code 사실·증거 확정.

**오늘 우선순위 산출 방식 (가중치 고정 금지):**

- `strategy/` = 회귀 신호 / 본질 변경 점검
- `decisions/` = 최근 중요 결재 본질 + 결재 자리 확정
- `work_orders/` = active 자료 + 최근 갱신 자리
- `product/` = active 사양 + 최근 갱신 자리

→ 4 자리 통째 종합 판단 후 우선순위 추천 + 사유 명시. 특정 자리 가중치 고정 X.

```
    [2. 트랙별 상세]       (완료(증거)·진행중·미완료·차단·원칙정합)
```

## 주의사항

- 본 커맨드는 **읽기 전용**. 어떤 파일도 저장하지 않고 git 작업 일체 없음.
- `docs/README.md` + `docs/sessions/_INDEX.md` 0건 → 즉시 에러 보고 + 생성 안내.
- START PROTOCOL 6단계 통째 점검 = 강제. 한 자리 누락 시 본인이 안내 + 점검 진입.
- 중요 결정 판단 (2-2 자리) = 본인이 본질 판단 (시간순 단순 5건 X).
- 오늘 우선순위 산출 (3단계 자리) = 4 자리 통째 종합 판단 (특정 자리 가중치 고정 X).
- 본 커맨드 통째 가동 후 = 팀장님 결재 받기 전 본인이 작업 진입 X.
