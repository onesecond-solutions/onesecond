# 세션 시작 컨텍스트 패키지 생성

## 목적

새 Claude AI 창(웹/앱 채팅) 첫 메시지로 붙여넣을 "컨텍스트 패키지" 1회 생성.

본 패키지를 받은 AI는 추가 URL 요청·web_fetch 없이 즉시 메인 트랙 위에서 작업 진입할 수 있어야 한다.

배경: AI는 매 세션 컨텍스트가 리셋되고 메모리는 요약본이라 부정확. Claude Code가 진실 원천(GitHub) 위에서 직접 패키지를 만들어 AI에 1회 복붙하는 것이 가장 안전하고 빠르다 (Code 우선 패러다임, 2026-04-30 결정).

## 절차

### 1단계: 현재 시각 확인

`date +"%Y-%m-%d %H:%M KST"` 실행해서 패키지 헤더 생성 시각으로 사용.

### 2단계: 정보 수집

다음 항목을 순차 수집하고 변수로 기억한다. 비어있으면 "없음"으로 기재.

#### 2-1. `docs/sessions/_INDEX.md` 읽기 (필수)

전체 본문을 읽어 다음 정보 추출:

- **Phase 1 진도** — `🎯 현재 메인 트랙 — design_test 시안 라이브 승격` 섹션의 진행 상태 표에서 ✅ 개수 / 9 = 완료 비율 (NN%)
- **직전 마감 페이지** — 가장 최근 ✅ 표시된 페이지 (commit 날짜 기준)
- **별 트랙 부채 목록** — `🚧 미해결 이슈 (인계)` 섹션 번호 + 한 줄 요약
- **결정 대기 항목** — `📋 결정 대기 항목` 섹션 번호 + 한 줄 요약

#### 2-2. 최신 세션 MD 식별

`ls -t docs/sessions/*.md 2>/dev/null | head -5` 실행 후 `_INDEX.md` 제외 가장 최근 파일 1건 식별.

GitHub URL 생성: `https://github.com/onesecond-solutions/onesecond/blob/main/docs/sessions/<파일명>`

해당 파일 본문을 읽어 **결정 컨텍스트(직전 세션 핵심 결정)** 3~5건 추출. 보통 "결정", "확정", "마감", "완료" 키워드 또는 미해결 이슈/결정 대기 섹션에서.

#### 2-3. _INDEX.md URL 고정값

`https://github.com/onesecond-solutions/onesecond/blob/main/docs/sessions/_INDEX.md`

#### 2-4. 진행 중 작업 / 미머지 브랜치

다음 명령으로 main에 머지 안 된 브랜치 식별:

```
git branch -a --no-merged main
```

각 브랜치의 마지막 커밋 시각·메시지:

```
git log -1 --pretty=format:"%ci %s" <브랜치명>
```

브랜치 상태 추정 (사람 판단 영역이라 추정만):
- 작업 중: 최근 24시간 내 커밋 있음
- 머지 대기: 24시간 이상 / 작업 멈춤
- 충돌: `git merge-base --is-ancestor` 등으로 정밀 판정 어려우면 "확인 필요"로 표기

미머지 브랜치 0건이면 "없음" 기재.

#### 2-5. 다음 작업 후보 추출

`_INDEX.md` Phase 1 표에서 ❌(미진행) / 🔄(부분 진행) 페이지 + `🚧 미해결 이슈` 시급 항목 + 별 트랙 큰 그림 (보험뉴스 자동 증식 엔진 등 메모리 인덱스 참조) 종합해 3~5개.

### 3단계: 패키지 출력 (정확히 이 형식)

화면에 다음 형식 그대로 출력. **마크다운 코드블록 안에 출력하지 말고**, 평문으로 출력해 팀장님이 그대로 복사해 새 AI 창에 붙여넣을 수 있게 한다.

```
═══════════════════════════════════════════
SESSION START PACKAGE — AI에 그대로 붙여넣기
생성 시각: {YYYY-MM-DD HH:MM KST}
═══════════════════════════════════════════

[1. 메인 트랙 현 위치]
- Phase 1 진도: {N/9 완료 (NN%)}
- 직전 마감: {페이지명} ({MM/DD})
- 진행 중 작업: {브랜치명 또는 "없음"}

[2. 최신 세션 MD URL]
{https://github.com/onesecond-solutions/onesecond/blob/main/docs/sessions/<파일명>}

[3. _INDEX.md URL]
https://github.com/onesecond-solutions/onesecond/blob/main/docs/sessions/_INDEX.md

[4. 미머지 / 작업 중 브랜치]
- {브랜치명} (마지막 커밋: YYYY-MM-DD HH:MM, 상태: {머지 대기/작업 중/확인 필요})
{없으면 "없음"}

[5. 별 트랙 부채 {N}건]
1. {부채 제목 한 줄}
2. {부채 제목 한 줄}
...

[6. 다음 작업 후보]
- {후보 1}
- {후보 2}
...

[7. 결정 컨텍스트 — 직전 세션 핵심 결정]
- {결정 1}
- {결정 2}
...

═══════════════════════════════════════════
[8. AI 행동 지시 — 본 패키지 받은 직후 수행]

① 위 1~7 내용으로 즉시 맥락 재구성
② 추가 URL 요청 / web_fetch 금지 (본 패키지가 완전)
③ 첫 응답 형식:
   "팀장님, 컨텍스트 복원 완료. 현 위치는 [메인 트랙 1줄] 입니다.
    다음 작업 어느 방향으로 진행할까요?"
④ 응답 톤: 메모리 §대화 방식 절대 원칙 따름 (한국어 현장 언어 / YES-NO 우선 / 단계별)
⑤ 작업 진입 시 작업지시서는 채팅 코드블록 형태만 (파일 생성 금지)
⑥ 모든 영문 표기 소문자 onesecond 통일

[9. 새 세션 진입 시 자동 부착 안내]
팀장님이 본 패키지 붙여넣은 후, AI가 다음 사항 자동 점검:
- 진행 중 브랜치 머지 여부 확인 필요? → "머지하셨나요?" 1회 질문
- 별 트랙 부채 중 시급한 것 있는지? → AI 우선순위 추천
- 다음 메인 트랙 진입 시 갭 분석 필요한 페이지인지? → 사전 안내

═══════════════════════════════════════════
END OF SESSION START PACKAGE
═══════════════════════════════════════════
```

### 4단계: 출력 후 자체 안내 1줄

패키지 출력이 끝나면 그 아래 한 줄 추가:

```
👆 위 패키지 통째로 복사해서 새 AI 창 첫 메시지로 붙여넣어주세요.
```

## 주의사항

- 본 커맨드는 **읽기 전용**. 어떤 파일도 저장하지 않고 git 작업 일체 없음.
- `_INDEX.md`가 없으면 즉시 에러 보고 + 생성 안내. 본 커맨드는 `_INDEX.md` 위에서만 동작.
- Phase 1 진도 추출 시 "✅ 완료" / "✅ **시안 통째 적용**" / "✅ **부분 흡수 ... 완주**" 등 ✅ 마크 모두 완료로 카운트.
- 결정 컨텍스트 3~5건은 직전 세션 MD에서 사람이 봤을 때 "이 세션에서 무엇이 결정·마감됐는가"를 한 줄씩 요약. 추정 무관 / 본문에 명시된 것만.
- 본 패키지는 AI가 메인 트랙 위에서 즉시 진입할 수 있게 하는 게 목표. 부가 정보 욕심내지 말고 1~9 섹션 골격 유지.
