# 세션 종료 인계 노트 자동 커밋

## 목적

세션 종료 시 AI(Claude AI 채팅 창)가 작성한 인계 노트를 팀장님이 본 커맨드 인자로 그대로 붙여넣으면, Claude Code가 다음을 자동 처리:

1. `docs/sessions/YYYY-MM-DD_HHmm.md` 신규 파일 작성
2. `docs/sessions/_INDEX.md` 상단 최신 세션 요약에 한 줄 추가 + `마지막 갱신` 갱신
3. main 브랜치에 단일 커밋 → push
4. GitHub URL 출력

배경: 세션 인계 자동화 — Code 우선 패러다임 정착 (2026-04-30 결정). 팀장님이 매 세션 사람 다리 역할 부담 제거.

## 사용 형태

```
/session-end

[인계 노트 본문 — AI가 작성한 자유 마크다운 그대로]
```

본 커맨드는 **명령 줄 뒤(또는 다음 줄부터)에 붙은 텍스트 전체**를 인계 노트 본문으로 받는다.

## 절차

### 1단계: 인자 검증

본문이 비어있으면 즉시 에러 출력 + 사용 안내:

```
❌ 인계 노트 본문이 비어 있습니다.

사용법:
  /session-end
  [여기 AI가 작성한 인계 노트 본문 붙여넣기]

본문은 자유 마크다운입니다. AI가 작성한 형식 그대로 받습니다.
```

본문 있으면 다음 단계 진행.

### 2단계: 현재 시각 확인 + 파일명 생성

`date +"%Y-%m-%d %H:%M %Z"` 실행해서:

- 파일명 형식: `YYYY-MM-DD_HHmm.md` (예: `2026-04-30_2330.md`)
- 저장 경로: `docs/sessions/<파일명>`
- 헤더 메타용 ISO 시각 별도 보관

### 3단계: 파일 작성

`docs/sessions/<파일명>` 신규 작성. **인계 노트 본문 위에 다음 헤더 자동 부착**:

```
---
세션 종료 시각: {YYYY-MM-DD HH:MM KST}
주요 마감 작업: {본문 첫 줄 또는 "오늘 마감" 항목 첫 항목 자동 추출, 없으면 빈값}
생성 도구: Claude Code /session-end
---

{인계 노트 본문 그대로}
```

**주요 마감 작업 자동 추출 규칙:**

본문에서 다음 패턴 순서로 시도해 첫 매치 1줄만 사용:

1. `오늘 마감:` 또는 `## 오늘 마감` 다음 첫 불릿 (`- ` 또는 `* `)
2. `이번 세션:` 또는 `## 이번 세션` 다음 첫 불릿
3. 본문 최상단 첫 불릿
4. 모두 없으면 빈값

추출된 1줄에서 트리밍·이모지 제거 후 헤더에 기재.

### 4단계: `_INDEX.md` 갱신

`docs/sessions/_INDEX.md` 읽고 다음 두 곳 갱신:

#### 4-1. `🗓️ 최신 세션 요약 (시간 역순)` 섹션 상단 한 줄 추가

기존 첫 항목 위에 다음 형식으로 신규 항목 삽입:

```
- `docs/sessions/<파일명>` — {YYYY-MM-DD} {시간대} ({주요 마감 작업})
```

시간대 분류:
- 05:00~11:59 → "오전"
- 12:00~17:59 → "오후"
- 18:00~23:59 → "저녁"
- 00:00~04:59 → "심야"

주요 마감 작업이 빈값이면 "(주요 마감 항목 본문 참조)"로 대체.

#### 4-2. 파일 최상단 메타 갱신

```
> **마지막 갱신:** {YYYY-MM-DD} {시간대}
```

라인을 오늘 날짜·시간대로 갱신.

⚠️ 본 단계는 한 줄 추가 + 메타 갱신만 자동. 메인 트랙 진행 표(`🎯 현재 메인 트랙`) / 미해결 이슈 / 결정 대기 등 구조적 갱신은 **본 커맨드 범위 밖** (이전 세션 종료 시 AI가 인계 노트에 미리 반영했어야 하는 영역, 또는 별도 세션에서 수동 갱신).

### 5단계: Git 커밋 & 푸시

다음을 순차 실행:

```
git add docs/sessions/<파일명> docs/sessions/_INDEX.md
git commit -m "docs(sessions): 세션 종료 인계 노트 ({YYYY-MM-DD} {HH:MM})"
git push origin main
```

⚠️ 현재 브랜치가 main이 아니면 즉시 정지 + 보고:

```
❌ 현재 브랜치가 main이 아닙니다 (현재: {branch}).
세션 인계 노트는 main에 직접 커밋되어야 합니다. 먼저 main 체크아웃 후 재실행하세요.
```

### 6단계: 결과 출력

```
✅ 세션 인계 노트 커밋 완료.

생성 파일: docs/sessions/<파일명>
커밋 SHA: {short hash}
커밋 메시지: docs(sessions): 세션 종료 인계 노트 ({YYYY-MM-DD} {HH:MM})
GitHub URL: https://github.com/onesecond-solutions/onesecond/blob/main/docs/sessions/<파일명>
_INDEX.md 갱신: 최신 세션 요약 1건 추가 + 마지막 갱신 메타 동기화

다음 세션 시작 시 /session-start 로 컨텍스트 패키지 출력하세요.
```

## 주의사항

- 본 커맨드는 **AI가 인계 노트를 이미 작성했다는 가정** 위에서 동작. Code가 자체 인계 노트를 생성하지 않음 (AI 책임 영역).
- 인계 노트 본문 내용 검증·요약·재가공 일체 안 함. **받은 그대로** 저장.
- main에 직접 커밋. 별 트랙 브랜치에서 실행 시 거부.
- `_INDEX.md` 구조적 갱신(메인 트랙 표, 미해결 이슈 등)은 본 커맨드 범위 밖. AI가 인계 노트에 미리 정리해둔 내용을 **다음 세션 시작 시** 사람·AI 협업으로 _INDEX.md에 반영하는 것이 표준 흐름.
- 파일명 충돌 시(같은 분 단위 재실행) 기존 파일 덮어쓰지 말고 즉시 보고:

```
❌ docs/sessions/<파일명> 이미 존재. 1분 후 재시도하거나 파일명 충돌 확인 필요.
```

- 본 커맨드는 **자동 정합성 검증 안 함**. 인계 노트 정확성은 AI 책임.
