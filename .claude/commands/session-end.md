# 세션 종료 요약 생성

## 절차

### 1단계: 현재 시간 확인

현재 날짜·시간을 확인해서 파일명 생성 기준으로 사용한다.

- 파일명 형식: YYYY-MM-DD_HHmm.md (예: 2026-04-23_1830.md)
- 타이틀용 시간대 판단:
  - 05:00~11:59 → "오전"
  - 12:00~17:59 → "오후"
  - 18:00~23:59 → "저녁"
  - 00:00~04:59 → "심야"

### 2단계: Git 정보 수집

다음 명령을 순차 실행:

- git log --since="midnight" --pretty=format:"- %h %s (%an, %ar)" --no-merges
- git status --short
- git log --since="midnight" --name-only --pretty=format:"" | sort -u | grep -v "^$"
- git log -1 --pretty=format:"%h %s"
- git branch --show-current

각 결과를 변수로 기억. 비어있으면 "없음"으로 기재.

### 3단계: MD 템플릿 작성

아래 템플릿대로 파일 생성. {중괄호} 부분은 실제 값으로 채운다.

> **생성 도구:** Claude Code /session-end 커맨드
> **작업 디렉토리:** onesecond

### 4단계: 파일 저장

docs/sessions/{YYYY-MM-DD}_{HHmm}.md 경로에 UTF-8로 저장.
디렉토리 없으면 먼저 생성.

### 5단계: ⭐ docs/sessions/_INDEX.md 자동 갱신 (큰 그림 동기화)

> **목적:** Claude Code가 다음 세션 시작 시 `_INDEX.md` 한 파일만 읽고 큰 그림 정합성을 검증할 수 있도록 한다. 본 단계 누락 시 2026-04-28 사례(큰 그림 놓치고 작업 진행) 재발 위험.

**갱신 절차:**

1. `docs/sessions/_INDEX.md` 파일 존재 확인. 없으면 새로 생성 (초기 템플릿: 본 커맨드 부록 또는 `docs/work_order_template.md` 참조).
2. 본 세션의 작업 내용 / 미해결 이슈 / 결정 대기 항목을 다음 섹션에 반영:
   - **🎯 현재 메인 트랙**: 진행 상태 표 갱신 (이번 세션에서 새로 완료된 영역 ✅로 변경, 새로 식별된 영역 추가)
   - **🚧 미해결 이슈 (인계)**: 본 세션에서 새로 식별·해결된 이슈 추가/제거
   - **📋 결정 대기 항목**: 새 결정 대기 추가, 결정된 항목 제거
   - **🗓️ 최신 세션 요약**: 본 세션의 새 MD 파일을 시간 역순 맨 위에 추가
   - **📌 폐기/보류된 트랙**: 본 세션에서 폐기·보류 결정된 트랙 추가
   - **🔄 진행 중·완료된 별건 트랙**: 본 세션의 별건 작업 결과 반영
3. 파일 최상단 `**마지막 갱신:** {YYYY-MM-DD}` 라인을 오늘 날짜로 갱신.

⚠️ Claude Code가 본 세션 대화 맥락에서 직접 알 수 없는 "사람 판단 영역"(예: 결정 대기 vs 폐기 등)은 추정으로 채우지 말고 팀장님 보강 요청 메시지를 담아둔다(예: `- [ ] 팀장님 확인 필요`).

### 6단계: Git 커밋 & 푸시

- git add docs/sessions/{생성한_파일명} docs/sessions/_INDEX.md
- git commit -m "docs: 세션 요약 {YYYY-MM-DD} {시간대} + _INDEX 갱신"
- git push origin main

### 7단계: 결과 보고

팀장님께 다음 출력:

- 생성 파일 경로 (세션 요약 MD)
- _INDEX.md 갱신 요약 (어떤 섹션이 어떻게 바뀌었는지 1~2줄)
- 커밋 해시
- GitHub 링크
- 팀장님이 채워야 할 섹션 4곳 안내
