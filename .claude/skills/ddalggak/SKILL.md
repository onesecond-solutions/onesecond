### 카드 만들기 = 2종 (절대 혼동 금지)

| 구분 | 보험사 자료실 딸깍 카드 | Quick 메뉴 카드 만들기 |
|---|---|---|
| 입력 | 자료실에 올라온 **첨부 이미지** | 사용자가 직접 올리는 **이미지 · TXT · PDF** (최대 5MB) |
| 처리 | **Gemini 핵심 추출** (Edge Function `gemini-card`) | 입력을 그대로 카드화 (**Gemini 안 씀**). PDF는 PDF.js로 텍스트 추출(스캔 PDF 미지원) |

| 함수 | `openCard`/`renderCard` (app.html) | `renderMakeCardTool`/`mcHandleFile`/`mcFire` |

### 이미지 라이트박스 딸깍 = 4줄 (보험사 자료실)

이미지 미리보기(라이트박스 `openLb`) 안 딸깍 메뉴 = **4줄**:
1. 다운로드 저장 (= 다른 이름으로 저장, 위치 선택 창 `showSaveFilePicker`)
2. MY SPACE 저장
3. 복사
4. 카드 만들기 (= Gemini 핵심 추출)

→ 콘텐츠 카드 우상단 팝오버(아래 "정확히 3개")와 **구분**. 이미지 라이트박스는 다운로드가 더해져 4줄.

### Gemini 카드 (`gemini-card` Edge Function)

- 입력 이미지 URL → 출력 `{brand, name, rows[]}` JSON → 기존 카드 디자인 재사용
- 키 = Supabase secret `GEMINI_API_KEY` (앱 코드·SQL에 평문 0 — 2026-05-22 노출 사고 재발 방지)
- 모델 = ListModels로 자동 선택(flash 우선) — 모델명 고정 금지(404 회피)
- 호출 = `window.db.fetch('/functions/v1/gemini-card')` (401 시 토큰 자동 갱신)

### 카드 하단 형식

- 작성자 **이름 · 전화번호** (`os_user`에서, 둘 다 없으면 줄 생략)
- 면책 = "※ 상담 참고용 · 보장은 약관 기준"
- ❌ "원세컨드" 등 서비스명 노출 금지 / ❌ 카드 헤더에 첨부 파일명 노출 금지

### 적용 범위 (현재)

- ✅ **보험사 자료실** = 첨부 이미지에 미리보기+딸깍+카드 **자동** (새 글도 자동 / 카테고리 무관 / 이미지 한정)
- ⏸ **MY SPACE** = 보류 (팀장님 따로 지시). 경로 = 딸깍 "MY SPACE 저장" 추천 / `library.image_url` 필드 존재 / 현재 `doSave`는 mock
- ⏸ 현장의 소리 · 매니저룸 등 = 각 영역 첨부 기능(fetch·상세 화면) 선행 필요
- 자료실 딸깍 카드 = **이미지 전용** (PDF는 다운로드만). PDF 카드화(소식지 등)는 별 트랙
- 미리보기+딸깍 본체는 전역 공용(`openLb`) → 새 영역은 "이미지를 `window.imgs`에 넣고 `openLb` 호출" 연결만 추가

### 핵심 회귀 신호

- ❌ 두 카드 만들기 혼동 (자료실=Gemini 추출 / Quick=직접 제작·PDF 포함)
- ❌ Gemini 키를 앱 코드·SQL Editor에 입력 (Edge Function secret에만)
- ❌ 카드에 "원세컨드" 노출 / 카드 헤더에 첨부 파일명 노출
- ❌ 이미지 라이트박스 딸깍을 3줄로 되돌리기 (이미지 = 4줄)

## 본질

**딸깍은 기능이 아니라 액션 허브.**

- 클릭 = 작은 팝오버 메뉴 표시
- 메뉴 = **정확히 3개** (모든 콘텐츠 동일)
- 복사는 메뉴 항목 중 하나일 뿐
- 콘텐츠 유형별 추가 메뉴 = **없음** (확장 X)

## 메뉴 (정확히 3개)

| 항목 | 본질 | 가동 상태 |
|---|---|---|
| 📁 MY SPACE 저장 | 본인 자료함 저장 | 플레이스홀더 (alert "준비 중") |
| 📋 복사 | 본문 → 클립보드 (카카오톡 붙여넣기용) | ✅ 실제 가동 |
| 🪪 카드 만들기 | 고객 전달용 카드 생성 + 자동 복사 | ✅ 실제 가동 (MVP) |

**메뉴 추가 금지** = 콘텐츠 종류와 무관하게 항상 3개.

## JS 함수 (`_new/app.html`)

### 메뉴 자체 자체

- `showDdalggakMenu(btn)` — 팝오버 표시
- `closeDdalggakMenu()` — 외부 클릭/ESC 닫기
- `ddalggakAction(action, btn)` — 항목 클릭 핸들러
  - `'copy'` — 복사 (실제 가동)
  - `'mySpace'` — MY SPACE 저장 (플레이스홀더)
  - `'makeCard'` — 카드 만들기 모달 자체 열기

### 카드 만들기 모달 자체 자체

- `openMakeCardModal(btn)` — 모달 표시 + 본문 추출
- `closeMakeCardModal()` — 모달 닫기
- `setNoticeMode(on)` — [안내문 추가] / [추가 안함] 토글
- `updateCardPreview()` — textarea 입력 시 우측 미리보기 라이브 반영
- `saveSignature()` — 이름/전화번호 localStorage 저장
- `finalizeMakeCard()` — 우측 미리보기 통째 → 클립보드 자동 복사 → "복사 완료" → 모달 닫기

## 카드 만들기 흐름 (MVP)

```
[카드 만들기] 클릭
  ↓
모달 열림 (현재 카드 본문 자체 자체 자체 우측 미리보기 자체 표시)
  ↓
첫 사용자 = Settings 영역 표시 (이름 + 전화번호 입력)
  → [저장] 클릭 → localStorage 저장 → Settings 영역 숨김
이후 사용자 = Settings 영역 숨김 (자동 사용)
  ↓
상단 토글 = [안내문 추가] / [추가 안함]
  ↓ 안내문 추가          ↓ 추가 안함
좌측 textarea 표시       좌측 textarea 숨김
  ↓
textarea 입력 = 우측 미리보기 안내문 영역 라이브 즉시 반영
  ↓
우측 미리보기 최하단 = "이름 | 전화번호" 자동 자체 (localStorage 자체)
  ↓
[카드 만들기] 클릭
  ↓
우측 미리보기 통째 → 클립보드 자동 복사
  ↓
"복사 완료. 카카오톡에 붙여넣기 하세요." 안내
  ↓
모달 닫기
```

## localStorage 구조

## 카드 최하단 형식 (팀장님 명시)

```
임태성 | 010-1234-5678
```

또는

```
임태성 · 010-1234-5678
```

(파이프 `|` 권장. 가로 1줄. 세로 2줄 금지.)

### 금지 사항

- ❌ 세로 2줄
- ❌ 서비스명 / 고객센터 / 이메일 / 홍보문구 포함
- ❌ 5개 이상 항목 (메뉴 = 정확히 3개)
- ❌ confirm() / prompt() 사용 (모달 안 자체 자체)
- ❌ 고정 안내문 자동 삽입 (사용자 직접 입력 자체)
- ❌ 콘텐츠 유형별 추가 메뉴 (메뉴 = 모든 콘텐츠 동일)
