## 0. 한 줄 정의

원세컨드 검색 = **보험판 구글** — 사이트에 쌓이는 상품·인수 정보를 실시간 인덱싱해 1초 안에 결과를 주는 보험 검색 인프라.

- 검색 범위 = **사이트 내부 정보에 한정** (외부 웹 검색 X)
- 데이터는 계속 업데이트 (라이브 누적)
- 사용자 본질 욕구 = "검색으로 빠른 결과"

---

## 1. 검색 이중구조 (핵심)

원세컨드 검색은 **범위(scope)가 다른 두 검색기**로 구성된다.

### ① 전체 검색 (상단 / 공개 인프라)

- **범위:** 허브 게시판 DB
- **대상:** 상품·인수 정보 (DB화된 것만)
- **성격:** 누가 검색해도 나오는 공동 자산 = "보험판 구글" 본체

### ② 팀/지점 검색 (그룹 / 격리 포함)

- **범위:** 전체(허브) ＋ 그 그룹의 자료 **합산** 검색
- **합산 대상:**
  - 허브 DB (전체)
  - ＋ 단체방 (입력폼 1차 분류분만)
  - ＋ 네비게이션방 (입력폼 + 자료 + 대화내용)
  - ＋ 스마트 게시판
- **성격:** 팀/지점 울타리 안에서만 도는 검색

---

## 2. 공간별 검색 범위 분기 (정밀)

| 공간 | 검색 대상 | 검색 제외 |
|---|---|---|
| **단체방** | 글 작성 입력폼 (1차 분류) | 대화창 내용 (휘발성) |
| **네비게이션방** | 입력폼(1차 분류) + 자료(첨부) + 대화내용 | — |
| **스마트 게시판** | 전체 (정리된 자산) | — |
| **허브 게시판** | 전체 (상품·인수 DB) | — |

**핵심 원칙: "입력폼 = 검색 대상 / 대화창 = 제외"**

- 입력폼에 1차 분류를 박으면 글 작성 순간 메타데이터가 붙음 → 검색·인덱싱·허브 승격이 그 분류를 타고 자동으로 흐름 (사후 분류 불필요)
- 대화창 잡담을 검색에서 빼서 검색 품질을 입구에서 방어

> **[확인 필요]** 단체방은 대화 제외, 네비방은 대화 포함 — 이 차이의 근거.
> 웹 가설: 네비방 대화 = 긴급 운영 흐름·보험사 이슈로 검색 가치 있음 / 단체방 대화 = 잡담 비중 높아 제외. **팀장님 확정 필요.**

## 3. OCR 파이프라인 (Make + OCR API)

사이트에 올라오는 **모든 자료(텍스트 + 이미지)**를 데이터베이스화.

```
자료 업로드 (텍스트 + 이미지)
      ↓
Make + OCR API — 이미지 속 글자까지 추출
      ↓
텍스트화 → 인덱싱
      ↓
검색 대상 (이미지로 올린 인수기준표·보험료표도 검색됨)
```

- 보험 현장 자료 대부분이 이미지(캡처·사진) → OCR이 "진짜 보험판 구글"의 핵심
- OCR은 전부 돌린다. **다만 결과가 들어가는 인덱스가 갈린다 (§4)**

---

## 4. 인덱스 분기 (규제선 = 가장 중요)

OCR 결과물이 **어느 인덱스에 들어가느냐**가 규제선을 가른다.

| 자료 출처 | OCR | 들어가는 인덱스 | 검색 노출 |
|---|---|---|---|
| 허브 게시판 | O | **허브 인덱스** | ① 전체 검색 |
| 단체방·네비방·스마트게시판 | O | **그룹 격리 인덱스** | ② 그룹 검색만 |

**분기 기준 = 게시판 위치 (자동)**

- 사용자는 "어디 올릴지"만 정함 → OCR·검색·격리가 자동으로 따라옴
- AI 키워드 판별 X (오분류 시 비교자료 공개 누출 = 규제 사고)

**규제 핵심:**

- 상품·인수 자료 → 허브 OK
- 보험료 비교 자료 등 → 허브 금지 (광고 규제) → 그룹 격리 인덱스에만

---

## 5. 허브 승격 (단체방·네비방 → 허브 DB)

단체방·네비방의 상품·인수 자료를 허브로 올리는 절차.

- **방식:** 관리자(매니저/admin) 수동 승격 — *웹 권장, 팀장님 확정 대기*
- **근거:**
  1. 규제 안전 — 무엇이 공개되느냐를 사람이 한 번 거름
  2. 품질 — 휘발성 잡담이 허브로 새는 것 방지
  3. 기존 9역할 권한에 "허브 승격" 버튼만 추가 (새 시스템 X)
- 자동 흡수는 v4.0 AI 단계에서 재검토

> **[결정 대기]** §5 승격 방식 = 관리자 수동 (웹 권장). **팀장님 최종 확정 필요.**

---

## 6. 검색 결과 표시 (UI ↔ 거버넌스 연결)

- **유형 뱃지 필수:** [허브] [스마트게시판] [단체방] [네비방] [상품] [인수] → 출처를 1초에 구분 (OS §10 출처 뱃지 정합)
- **타이핑 즉시 결과:** instant search, 엔터 불필요
- **빈 화면 없음:** 검색창 열면 최근 검색 + 자주 찾는 인수기준 노출

---

## 7. 검색기 UI 위치 (디자인 시스템 spec과 연결)

- **헤더 상시 검색바:** 모든 페이지 A영역, 인디고 포커스 + ⌘K 힌트
  - 플레이스홀더: "상품·인수기준·질병명으로 검색"
- **전역 검색 오버레이 (⌘K):** 화면 전체 덮는 검색 레이어, 결과 유형별 그룹
- 상세 톤·hex = 디자인 시스템 spec(①) 참조
