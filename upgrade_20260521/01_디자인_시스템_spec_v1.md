## 0. 방향 한 줄

기존 브라운 전면 폐기. **Linear의 구조 철학(빠르고 절제된)을 흡수하되, 색은 인디고 + 무채색 골격으로 새로.** 다크 기본 + 라이트 사용자 선택.

- ❌ "Linear처럼 보이게" (다크+보라 그대로 복제)
- ✅ "Linear처럼 빠르고 깔끔하게" (구조·반응성 흡수)

## 2. 색상 사용 원칙 (3대 원칙)

1. **강조색은 아껴 쓴다.** 인디고는 주요 버튼·현재 위치·중요 링크에만. 나머지는 무채색. 모든 걸 인디고로 칠하면 촌스러워짐.
2. **깊이는 테두리가 아니라 명도 차이로.** 카드를 선으로 두르지 말고 배경보다 밝은 면으로 띄움. 선은 정말 필요할 때만.
3. **이전 브라운 완전 폐기.** `--color-brand`, `--brand-*` 스케일 전부 버림. 별칭으로 연명 X (그게 토큰 무너진 원인). 새 토큰은 단일 스케일로.

## 3. 타이포그래피

### 서체 — Pretendard 단독

- **본문·UI 전체 → Pretendard.** 한글 가독성 최강, 숫자·영문도 깔끔. (Linear가 Inter로 통일한 것처럼 우리는 Pretendard로)
- **디스플레이(큰 제목) → 별도 폰트 X.** Pretendard 무게(700~800)로 위계.
- **숫자·코드 강조 → JetBrains Mono** (보험료·수치 표시 시만 제한적)
- **폐기:** DM Sans 1순위 로딩 (한글 서비스에 영문 본문 폰트 1순위는 버림)

### 사이즈 — 16px 기준 1.25 계단

| 역할 | 크기 | 무게 | 용도 |
|---|---|---|---|
| Display | 28px | 800 | 페이지 대표 타이틀 (드물게) |
| H1 | 22px | 700 | 페이지 제목 |
| H2 | 18px | 700 | 섹션 헤더 |
| **Body** | **16px** | 400~500 | 본문 기준 ⭐ |
| Body-sm | 14px | 500 | 카드 설명·보조 |
| Label | 13px | 600 | 라벨·메타 |
| Caption | 11px | 700 | 뱃지·태그 (대문자) |

### 가독성 디테일

- **줄 간격:** 본문 1.6, 제목 1.3 (한글은 영문보다 빽빽 → 넉넉히)
- **자간:** 제목만 살짝 좁게(-0.01~-0.02em), 본문은 건드리지 않음
- **굵기 대비 분명히:** 본문 400 / 강조 600 / 제목 700+. 어중간한 500 남발 금지

## 4. 폰트 크기 컨트롤러 (확대/축소)

> 배경: 초기엔 보험설계사 평균 연령 감안해 폰트를 키웠으나, 확대/축소 컨트롤러로 대체. 기본은 이상적 크기, 필요한 사람만 키움.

- **4단계:** 90% / 100% / 110% / 125% (16px = 100% 기준, 무한 확대 금지)
- **적용 범위:** 본문 영역(D영역)만. 사이드바·헤더는 고정 (전체 확대 시 레이아웃 뒤틀림 방지)
- **설정 기억:** 한 번 키우면 저장 → 다음에도 유지 (평균 연령 높은 사용자 배려)

## 5. 동적 효과 철학

화려한 효과의 90%는 영상·플래시가 아니라 순수 코딩(CSS/SVG/Canvas). 다만 **어디에 절제하고 어디에 풀지**가 세련됨.

| 화면 | 동적 효과 | 이유 |
|---|---|---|
| **운영 화면** (스크립트·게시판·검색 등) | 절제 — 부드러운 호버, 스르륵 등장만 | 상담 중 1초 도구, 빛 입자 둥둥 = 정신없고 느림 |
| **외부 소개/홍보 화면** (향후) | Linear급 화려함 풀기 OK | 처음 방문자 홀리는 마케팅 용도 |

원칙: 운영형 홈(home_v2)은 랜딩페이지가 아니므로 운영 화면 = 절제.

## 6. 실제 작업량 (숨은 핵심)

> app.html raw 확인 결과 — 토큰만 갈아끼우면 끝이 아님.

디자인 개편은 **세 겹**:

1. **tokens.css 재설계** — 인디고+무채색, 라이트/다크 양쪽 동등 설계 (단일 스케일, 별칭 X)
2. **app.html 하드코딩 브라운 전수 제거** ⭐ — 토큰 아닌 직접 박힌 브라운 색출 필요:
   - `theme-color #8B6F47`
   - B영역 그라데이션 `#A0522D → #8B4423`
   - 알림 배지·아바타·quick-btn 그림자 `rgba(160,82,45,...)`
   - → 전부 토큰 참조로 전환
3. **pages/*.html 각 페이지** — 같은 작업 반복

**Code 작업 시 "브라운 하드코딩 전수 제거"를 명시 항목으로 박을 것** (안 그러면 빠뜨림).

## 7. 다크/라이트 토글 구조

- admin_v2가 이미 5종 톤 토글(`body[data-admin-tone]`) 구조 보유 → 검증된 패턴
- 사용자 페이지는 `:root` 라이트 직박음 + 다크 분기 없음 → **다크 분기 신설 필요**
- 권장: `:root` = 라이트 기본, `[data-theme="dark"]` = 다크 override (또는 반대)
- **[확인 필요]** 토글 상태 저장 위치 = localStorage vs users 테이블 → Code가 `js/admin_v2.js` 저장 방식 확인 후 통일
