## §0. 대원칙 (재해석 — 그대로 복사 금지)

레퍼런스를 **그대로 복사하지 않는다.** 원세컨드 디자인 토큰으로 **재해석**한다.
원세컨드의 핵심은 **보험 실무 데이터 검색**이다. 예쁜 것보다:
1. **읽기 쉬움**
2. **비교 쉬움**
3. **클릭 판단 쉬움**
— 이 셋이 항상 우선. 장식이 판단을 방해하면 장식을 버린다.

기반 토큰: `--ac:#6366F1`(인디고) · 라이트 단일(`data-theme="light"` 고정) · 본문 16 / 목록·미리보기 14~15 · radius-sm 이상 · 라인 아이콘만 · 이모지/원문자 금지. (실효 토큰 = **app.html 인라인 `:root`**. tokens.css·tokens-v2.css는 app.html 미연결 → 토큰 단일화가 선행 과제.)

## §1. 5계열 매핑 + 현재 화면 배정 (실측 기반)

### 1. Stripe 계열 — 복잡한 데이터·표·카드·상태 배지 정돈
| 화면 | 뷰 | 현재 | 게이트 |
|---|---|---|---|
| 상품 라인업 | `#v-product-lineup` | 좌 회사목록 + 우 조망↔상세 | 전체개방 |
| 알릴의무 | `#v-goji` | 3컬럼 좌회사·중리스트·우상세 | 임태성 |
| BMI | `#v-tool` 계산기 + 검색 `_svBmiHtml`(4축 비교) | 분산 | 도구=개방 / 검색=임태성 |
| 운영센터 | `#v-admin` | 9탭 + 6열 KPI 관제 | admin |
| 삼성 라인업/납면 | `#v-samsung-lineup`·`-waiver` | 표 | 임태성 |

### 2. Linear 계열 — 좌목록+우상세, 선택 하이라이트, 상태/번호
| 화면 | 뷰 | 현재 | 게이트 |
|---|---|---|---|
| 보험 Q&A | `#mys-qna` · `#iv-today` | 좌목록+우뷰어 split | 공용 |
| 업무노트(스크립트) | `#v-scripts` | 칩 필터 + 카드 그리드 | 공용 |
| 글쓰기 게시판 | `#v-together`·마이스페이스 | split + 인라인 글쓰기 | 좋아요·댓글=임태성 |

### 3. Notion 계열 — 긴 텍스트를 문서형 카드·콜아웃으로
| 화면 | 뷰 | 현재 | 게이트 |
|---|---|---|---|
| 자료실 | `#v-insurer-vault` | 탭 + 칩 + split + 캘린더 | insurer_*·admin |
| 라이브러리(자료함) | `#v-myspace` `#mys-vault` | 폴더트리 20/30/50 | 공용 |
| 교육자료 | — | **미구현(신규 트랙)** | — |
| 보험용어 | — | **미구현(신규 트랙)** | — |
| 솔루션북 재가공 | — | **미구현(신규 트랙)** | — |

### 4. Webflow 계열 — 마케팅 섹션, 기능 가치 고급 설명
| 화면 | 위치 | 현재 |
|---|---|---|
| 홈 배너 | `#osBannerCarousel`(`#v-home`) | 히어로 캐러셀 4슬라이드 |
| 랜딩·기능 소개 | `index.html`·`about.html`·`pages/landing.html` | app.html **밖** 별도 정적 |

### 5. Apple 계열 — 홈 검색기 첫 화면·핵심 메시지에만 제한
| 화면 | 뷰 | 현재 | 게이트 |
|---|---|---|---|
| 홈 검색기 hero | `#homeSearch`(`#v-home`) | 워드마크+태그라인+큰 검색창 | 임태성(진행 중·재작업 필요) |
| 핵심 메시지 | `.hs-philo`·`.hs-head` | 중앙 타이포 철학문구 | 상동 |

> **주의(대표 지시):** Apple은 홈 검색·핵심 메시지 영역에만. **전체 업무 화면에 과적용 금지.** 업무 화면(데이터·리스트)은 Stripe·Linear가 기준.

## §2. 미구현 대상 (신규 트랙 — 이번 디자인 작업과 분리)
- **보험용어(용어사전)** — app.html 0건
- **교육자료 독립 화면** — 미구현(설명서 예시 라벨만)
- **숨고 솔루션북 재가공 콘텐츠** — 없음
- **랜딩·기능 소개** — app.html 밖 별도 정적 파일(Webflow 계열 적용은 별도)
→ 이들은 "디자인 재해석"이 아니라 "새 화면 신설"이라 데이터 유기체 트랙과 함께 별도 계획.

## §3. 적용 우선순위 (읽기·비교·클릭 판단 우선)
0. **선행 — 디자인 토큰 단일화**: app.html 인라인 `:root` 기준으로 색·타이포·radius 정리(모든 계열의 기반). 변화 0에서 시작(별칭).
1. **Stripe — 상품 라인업 / 알릴의무**: 보험 실무 데이터 비교·판단의 핵심. 원세컨드 가치를 가장 잘 드러냄. 효과 최대.
2. **Linear — 보험 Q&A / 업무노트**: 리스트·좌목록우상세 정돈.
3. **Apple — 홈 검색기**: 진행분(임태성 게이트)을 시안대로 제대로 마무리.
4. **Notion — 자료실 / 라이브러리**: 문서형 정리.
5. **Webflow — 홈 배너**: 마케팅 블록.
6. **신규(§2)**: 별 트랙.

---

## §4. 구현 방식 (얕은 적용·전면 수정 재발 방지)
- **한 화면씩.** app.html 전면 일괄 수정 금지. 진행 중 기능 PR과 혼합 금지.
- **기능 로직 무변경.** 라우터·state·검색 함수 안 건드림. 레이아웃/마크업/스타일 + 기존 함수 배선만.
- **먼저 design-preview 시안 → 임태성 게이트 화면 1곳 선적용 → 검수 → 전체 개방 결재.**
- **작업지시서에 구체 명세 필수**(여백값·배치·컴포넌트·상태). "시안 참조" 한 줄 금지 — 이게 홈 검색 얕은 결과의 원인이었음.
- 게이트 밖·기능 회귀 0 검증.
