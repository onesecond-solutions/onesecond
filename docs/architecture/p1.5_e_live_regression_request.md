## 6-4. ⑥ 보강 진입 직전 — branches/teams 라이브 상태 캡처 의뢰

**원칙:** `CLAUDE.md` DB 규칙 = "DB 상태 단정 금지 — 검증 SQL 먼저". row 부재 vs 다른 ID 분기 정확히 파악 후 보강 SQL 결정.

**Chrome SQL Editor 복붙 (SELECT only, BEGIN/COMMIT 불필요):**

```sql
-- ============================================================
-- P1.5-E ⑥ branches/teams 라이브 상태 캡처
-- 신버전 onesecond-v1-restore-0420 (P1.5-A에서 신버전 확인 완료)
-- ============================================================

-- ① branches 전수 raw
SELECT * FROM public.branches ORDER BY name;

-- ② teams 전수 raw
SELECT * FROM public.teams ORDER BY name;

-- ③ 코드 박힌 UUID 5건 존재 검증
SELECT
  '396edf6a-15db-4b69-a6b9-dae74b08cd33'::uuid AS expected_id,
  '더원지점' AS expected_name,
  EXISTS (SELECT 1 FROM public.branches WHERE id = '396edf6a-15db-4b69-a6b9-dae74b08cd33'::uuid) AS exists_in_branches
UNION ALL
SELECT '95088922-c76e-45b5-bb68-8ce1e914a6c4'::uuid, '1팀',
  EXISTS (SELECT 1 FROM public.teams WHERE id = '95088922-c76e-45b5-bb68-8ce1e914a6c4'::uuid)
UNION ALL
SELECT '741bce8a-e264-4f69-bc23-a7fff3478bec'::uuid, '2팀',
  EXISTS (SELECT 1 FROM public.teams WHERE id = '741bce8a-e264-4f69-bc23-a7fff3478bec'::uuid)
UNION ALL
SELECT '482c8c2d-b6f6-4188-9657-80f30d22431f'::uuid, '3팀',
  EXISTS (SELECT 1 FROM public.teams WHERE id = '482c8c2d-b6f6-4188-9657-80f30d22431f'::uuid)
UNION ALL
SELECT '5fccd362-9ee3-4165-8960-7cb0b7ec72fa'::uuid, '4팀',
  EXISTS (SELECT 1 FROM public.teams WHERE id = '5fccd362-9ee3-4165-8960-7cb0b7ec72fa'::uuid);

-- ④ branches/teams 컬럼 구성 확인 (시드 INSERT 시 NOT NULL 컬럼 정합화 위함)
SELECT column_name, data_type, is_nullable, column_default
FROM information_schema.columns
```
