## §1. 팀장님 요청 본문 (2026-05-22 받음, 통째 보존)

### 현재 문제

KCP 심사 담당자가 다음 사유로 심사를 진행하지 못하고 있음.

> 신청 URL 접속 시 판매 상품 및 서비스 확인 불가

현재 onesecond.solutions 접속 시 로그인이 필요하여 심사자가 서비스 내용을 확인할 수 없음. PG 심사 담당자는 별도 계정으로 로그인하지 않음.

따라서 현재 구조는 심사 관점에서:

```
onesecond.solutions
↓
로그인 필요
↓
서비스 확인 불가
↓
심사 보류
```

상태로 판단될 가능성이 높음.

### 필수 수정 사항

#### 1. 공개 랜딩페이지 구축

- 메인 도메인 https://onesecond.solutions 접속 시 로그인 없이 서비스 정보를 확인할 수 있어야 함

#### 2. 랜딩페이지 필수 내용

**서비스 소개:**
- 원세컨드가 무엇인지 명확하게 설명
- 예시: 보험 검색 플랫폼 / 보험업 업무관리 SaaS / 보험 설계사 및 GA 대상 서비스

**서비스 기능:**
- 예시: 보험 정보 검색 / 고객 관리 / 문서 관리 / 업무 관리 / 보험 상품 비교

**이용 대상:**
- 보험설계사 / GA / 보험대리점 / 보험업 종사자

**요금제 표시:**
- 심사자는 가격 확인을 중요하게 봄
- 최소한 아래 중 하나는 필요: 요금제 / 이용료 / 무료체험 정책
- 예시: Starter 월 29,000원 / Business 월 99,000원
- "서비스 준비중" / "출시 예정" 보다는 실제 가격 공개가 심사에 유리함

#### 3. 사업자 정보 노출

푸터에 반드시 표시:
- 상호
- 대표자명
- 사업자등록번호
- 사업장 주소
- 고객센터 이메일

#### 4. 필수 정책 페이지

공개 URL 필요:
- 이용약관
- 개인정보처리방침
- 환불정책

→ 푸터 링크 연결

#### 5. 결제 테스트 환경

포트원 메일 기준 PG 계약 진행을 위해 결제페이지 및 결제모듈 호출 필요:
- 결제 버튼
- 결제 페이지
- 테스트 결제창

→ 최소 동작 필요. 실결제까지는 아니어도 됨.

### 권장 구조

```
/         → 공개 랜딩페이지
/pricing  → 요금제
/terms    → 이용약관
/privacy  → 개인정보처리방침
/refund   → 환불정책
/login    → 로그인
/app      → 로그인 후 서비스
```

### 목표

심사 담당자가 로그인 없이:
- 서비스 확인
- 가격 확인
- 사업자 확인
- 약관 확인
- 결제 흐름 확인

→ 현재 가장 큰 병목은 통신판매업신고번호가 아니라 **로그인 전 서비스 정보 부재**로 인한 PG 심사 지연.
