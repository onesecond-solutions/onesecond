# 원세컨드 현황 및 전략 재정립 브리핑
**작성:** 기획실장(Claude Web) → 총괄실장(Code)
**일자:** 2026-06-09
**목적:** 단순 작업 보고가 아닌, 운영 체계 재정립 + 다음 단계 전략 수립

---

## 1. 큰 방향 (불변)
- **보험판 구글 = 검색 중심 플랫폼**이 본질
- MY SPACE / 라이브러리 / CRM / 지식엔진은 그 위의 레이어
- 현 단계 시금석: **"현장 설계사가 설명 없이 쓸 수 있는 도구"**

## 2. 현재 진행 상황 (라이브 실측)
- 라이브러리 대개편: 라벨(#504) · 다이어리(#507) · 재정렬(#509) · 메모 복사복구(#512) 전부 라이브 / sort_order DB(#508 적용)
- 지식엔진 사이클1 적재: **✅ 완료** — 92건 적재, 184 중복 → ctid 정리 → 92 검증
  - ※ 대표님 초안엔 "적재 대기"였으나 **실제 완료**로 갱신
- 머지 대기: #506(라벨 통일) · #508(sort_order 아카이브)
- **미해결 기능 결함**: 삭제 버튼이 실제로 삭제 안 됨 (library 메모/자료 추정 — #513은 스크립트 RLS만). 진단·수정 진행 중
- UX 이슈: 글쓰기 버튼 우상단 고립 + 하위 칩(전체·자료·스크립트·메모·즐겨찾기)을 작성 버튼으로 오인
- D영역(우측 대시보드): 현재 전 탭 노출 → 대표님 지시로 **'고정 패널 → 자연 스크롤 흐름' 재검토** (이전 '전 탭 공통 sticky' 결정에서 수정 → #511 일부 재검토 필요)

## 3. 가장 큰 문제 (대표님 진단에 동의)
- 기능이 아니라 **인지 구조가 병목**
- 칩이 저장 위치 / 조회 메뉴 / 작성 버튼 중 무엇인지 안 보임
- 개발 속도는 충분, **사용자 인지 설계가 못 따라감**
- 기획실장 판단: 큰 재개발 없이 라벨·배치·흐름으로 풀린다 → **"생각하지 않아도 되는 UX"** 원칙 적용

## 4. 수용된 방향 (대표님 확정)
- 호칭·조직 통일 / 대표 중심 결정 / 3실장 역할 분리
- **라이브러리 = 저장소 아닌 업무공간**
- 기능 추가보다 **무사고 UX 우선**
- **글쓰기 버튼 ↔ 하위 칩 역할 명확 분리**
- **D영역 = 고정 sticky보다 자연 스크롤 흐름 재검토**

## 5. 총괄실장 판단 요청 (기획실장 권고 우선순위)
1. **삭제 버그 먼저** — UX 아닌 기능 결함, 데이터 관리 불가·라이브 통증. 작고 즉효
2. **사용자 인지 UX 정리** (핵심 병목): 글쓰기 발견성 + 명칭 변경(메모장/자료함) + 글쓰기 → 유형 선택 + 칩 = 조회 메뉴 인지. 한 트랙
3. **라이브러리 구조 + D영역(자연 스크롤) 재정리** — 2번과 한 호흡
4. **#506 · #508 머지** — 저비용, 짬에
5. **92 적재 = 이미 완료, 결정 불요**

→ 순서: **결함(1) → 인지 UX(2·3) → 정리 머지(4)**

## [대표님 의견 반영]
지금은 기능 단계가 아니라 "사용자가 이해하는 구조"로 넘어가는 **전환점**. 개발 속도보다 **인지 정합**이 성공 변수. 의견은 평등하게 존중하되 최종 결정은 대표.
