# 세션 시작 보고 양식 — START PROTOCOL REPORT (8섹션)

> **목적:** 매 세션 시작 시 Code가 출력하는 보고를 8섹션 표준 양식으로 고정한다.
> **교체 이력:** 기존 5항목 브리핑(환경 + 오늘 작업 브리핑) → 8섹션 양식 (2026-06-09 결재).
> **부착 경로:** `CLAUDE.md` § 매 세션 시작 시 보고 절차 + `/session-start` 슬래시 커맨드.
> **운영 규칙:** 유휴 0 · 증거=push · 원장=진실 · main 직접 push 금지.

## 데이터 소스 매핑

| 자리 | 데이터 소스 |
|---|---|
| [1] 상태판 | 트랙 진행률 + `git branch` |
| [2] 완료(증거) | `git log` (PR#·커밋 해시) |
| 브랜치 | `git branch -a` |
| 세션 흐름 | `docs/sessions/` 최신 1~3개 |
| 할 일 | `docs/work_orders/` · `backlog.md` |
| 채굴팀 가동 상태 | 직전 세션 대비 지식엔진 적재 진행 여부 — 멈춰 있으면 🔴 표시 |
| [5] 48시간 계획 | work_orders 우선순위 + 병행 가능 트랙 |
| [6] Backlog | `docs/work_orders/backlog.md` |
| [8] 경비 | `docs/ops/cost_ledger.md` (값 확정 후) |

## 총괄팀장(Code) 확정 + 협업 보강 — 역할 분리 (2026-06-09 정정)

| 섹션 | 총괄팀장 (Code) | 협업 보강 (기획팀장 Web · GPT 비서관) |
|---|---|---|
| [1] 상태판 | ✅ 확정 (git·문서 사실) | — |
| [2] 트랙별 상세 | ✅ 확정 (증거 기반) | — |
| **[3] 결재 필요** | ✅ **확정** | 🤝 협업 보강 |
| **[4] 리스크·충돌** | ✅ **확정** | 🤝 협업 보강 |
| [5] 48시간 계획 | ✅ 확정 | 🤝 협업 조정 |
| [6] Backlog | ✅ 확정 | — |
| **[7] 총괄 판단** | ✅ **확정** | 🤝 협업 보강 |
| [8] 경비 현황 | ✅ 확정 (원장 기반) | — |

- **총괄팀장(Code)이 사실·증거(git, 문서, 라이브 상태)와 전략 판단을 모두 확정한다.** 추측 금지, 떠넘기기 금지.
- **[3]결재·[4]리스크·[7]총괄판단도 총괄팀장(Code)이 끝까지 확정한다.** 초안만 내고 멈추지 않는다. 기획팀장(Web)·GPT 비서관은 새로운 시각·전략을 **협업으로 보탠다(대체가 아니라 보강)**.
- 보고는 총괄팀장(Code)이 대표님께 **직접** 출력·보고하고, 대표님 결재 후 작업 진입 (승인 전 작업 진입 금지).

## 운영 원칙 4종 (헤더 고정)

| 원칙 | 본질 |
|---|---|
| 유휴 0 | 짧은 작업(클릭 검수 등) 중에도 병행 가능한 트랙(채굴 적재 등)을 멈춰 두지 않는다 |
| 증거=push | "완료"는 PR 머지·push로 증명한다. 말로 완료 선언 금지 |
| 원장=진실 | 경비·상태는 원장 파일이 진실 원천. 기억·추측 단정 금지 |
| main 직접 push 금지 | 제품 코드는 PR + Deploy Preview 거쳐 머지 (CLAUDE.md 배포 프로세스 정합) |
