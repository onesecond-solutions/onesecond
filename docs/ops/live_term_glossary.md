# 원세컨드 용어 사전 — 라이브 표시 용어 ↔ 코드 식별자 ↔ 폐기 대상 옛 용어

> **작성:** 2026-07-09 (조사) · 총괄팀장 검토 후 커밋 예정
> **범위:** app.html · js/* · pages/* 정적 근거 기반. DB 접속·SQL 실행 없음. **제품 코드는 무수정, 본 문서 1개만 생성.**

## 이 문서의 목적과 사용법

라이브 화면 표시 용어는 개명됐는데(예: "스크립트"→"업무노트") 코드의 함수명·뷰 id·테이블·클래스는 옛 이름 그대로 남아 있다. 이 사전은 **(1) 옛 용어가 어디에 있는지, (2) 그게 사용자 화면에 보이는 텍스트인지 코드 내부 식별자인지**를 구분해, 대표가 안전하게 폐기 판단을 하도록 만든다.

- **대화·소통·화면 문구 = §1 "라이브 표시 용어"만 사용.** (CLAUDE.md START PROTOCOL "라이브 화면 용어로만 대화" 원칙 정합 — `feedback_use_current_live_ui_terms`)
- **코드를 가리킬 때 = §1의 "코드 식별자" 열을 참조** (뷰 id·함수·테이블).
- **폐기 판단 = §2(화면 노출 옛 용어)만 실제 폐기 대상.** §3(코드 내부 식별자)은 소통·표시와 무관하므로 당장 폐기 불필요.
- **§4(혼용/정본 미확정)는 대표 결정 필요 목록.**

**두 가지 태그:**
- **[화면노출]** = 사용자에게 보이는 텍스트(라벨·버튼·제목·안내문·placeholder). → 폐기/교체 대상.
- **[코드내부]** = 함수명·뷰 id·클래스·변수·주석·테이블/컬럼. → 화면·소통 무관, 리네임은 별도 트랙(당장 폐기 불필요).

## §2 화면 노출 옛 용어 목록 (폐기 대상) — [화면노출]만

사용자에게 실제 텍스트로 보이는 옛 용어. **라이브 용어로 교체해도 코드 로직에 영향 없음**(안정키는 코드 내부에 별도 존재).

### 2-1. "스크립트" → **업무노트** (개명 미완 · 최우선 폐기 후보)

라이브러리 칩·글쓰기 탭은 이미 "업무노트"로 바뀌었으나(app.html:2201·3504·5654·5710 등), **전용 화면·홈·Quick 진입점은 아직 "스크립트"**로 남아 혼재:

| 파일:라인 | 노출 위치 | 제안 라이브 용어 |
|---|---|---|

| `app.html:2083` | 홈 안내문 "라이브러리, **스크립트**를 하나의 흐름으로…" | 업무노트 |
| `app.html:2347` | 업무노트 화면 제목 "**스크립트** · 보험 상담 화법" | 업무노트 |

### 2-2. "현장의 소리 / 스마트 게시판 / 단체방 / 지점 게시판" — 숨김 화면

### 2-3. "공동 스페이스" — 라이브러리 탭 (사용자에게 보임)

| 파일:라인 | 노출 위치 |
|---|---|
| `app.html:2154` | 라이브러리 탭 "**공동 스페이스**" (`mysTab(this,'team')`) |

→ 메모리 노트(`project_space_to_scope_direction`)상 "공동 스페이스"는 개인/팀/지점 **범위로 흡수** 방향이나, 라이브러리 탭 라벨로 아직 노출 중. 사이드바 "팀 스페이스/지점 스페이스"와 개념 중복 → **정본 결정 필요(§4).** (삭제는 대표 확정 시에만 — 방향만 정해짐, 실행 착수 아님)

**§2 화면 노출 옛 용어 총계:** 3계열 / 확정 [화면노출] **약 12건** (2-1 스크립트 6건 + 2-2 숨김화면 5건 + 2-3 공동스페이스 1건). 2-1의 하위유형(콜/채팅/녹취 스크립트)은 확인 필요라 별도.

**대표 폐기 판단이 필요한 핵심 항목:**
1. **"스크립트"→"업무노트" 화면 문구 6건 교체** (진행 중 업무노트 정렬 건과 직결) — 하위유형 라벨 처리 방향만 확정하면 안전.
2. "공동 스페이스" 탭 라벨 유지/개명/제거 방향.
3. 현장의 소리 계열 죽은 텍스트 = 화면 숨김이라 시급도 낮음(정리 트랙으로).

## §3 코드 내부 식별자 (리네임 별도 트랙 · 당장 폐기 불필요) — [코드내부]
