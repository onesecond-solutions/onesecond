# 멀티 에이전트 팀 구조 — 운영 규칙

## 1. 조직도
임태성 팀장 (최종 의사결정자)
  ↑ 보고는 총괄팀장에게서만 받는다
총괄팀장 (Claude Web) — 전략·우선순위·작업지시서·총괄 보고
  ├─ 데이터팀장 (Opus 파이프라인) — 채굴·검수, 워커(Haiku 6대) 관리
  ├─ 개발팀장 (Claude Code) — 구현·PR·커밋
  └─ 검수팀장 (Claude Chrome) — 라이브 검수 (SELECT only)

## 2. PC 역할 고정 (충돌 방지)
- 집 PC 1 = 데이터 트랙 (채굴·OCR·정리) + 퇴근 후 총괄 보고 확인 겸용
- 집 PC 2 = 검수·테스트 트랙
- 회사 PC = 출근 중 총괄 보고 확인 + 머지·승인·방향 지시
- 보고 확인·승인은 읽기 작업이라 같은 PC의 데이터 작업과 충돌하지 않는다
  (금지는 같은 브랜치·같은 파일을 동시에 고쳐 쓰는 경우뿐)

## 3. PC 전원·재부팅
- 두 집 PC: 절전(잠자기) "안 함" 설정 (본체가 잠들면 작업이 멈춤)
- 모니터는 꺼도 무방 (본체는 계속 작동)
- 재부팅 = 주 1회 (최대 2주를 넘기지 않는다)

## 4. 브랜치 규칙
- main 직접 push 금지 (총괄팀장도 금지 — 지시·검수·머지만)
- 데이터팀 = data/* / 개발팀 = feat/* / 검수팀 = qa/* / 문서 = docs/*
- 같은 파일 동시 작업 금지

## 5. 보고 라인
작업자 → 각 팀장 → 총괄팀장 → 팀장님
팀장님은 총괄팀장에게만 보고받는다.

## 6. 밤 운영 (무인 가동)
원칙: 밤에는 결과(PR)만 쌓고, 머지·DB 쓰기는 아침에 팀장님이 일괄 처리한다.

6-1. 일감 큐 — 대화형 AI(Code)는 작업 하나를 끝내면 멈추고 다음 지시를 기다린다.
     밤에 쉬지 않게 하려면 작업지시서에 독립 작업을 여러 개(예: 1·2·3·4·5)
     미리 쌓아둔다. 각 팀장은 하나를 끝내면 다음으로 자동 진행한다.
6-2. 자동화 트랙 — 채굴·OCR 등 기계적 반복은 사람 대화 없이 도는 스케줄러
     (cron / GitHub Actions / Supabase)에 올린다. 이쪽은 진짜로 밤새 멈추지 않는다.
6-3. 아침 처리 — 팀장님이 밤사이 쌓인 PR·SQL 후보·보고를 확인하고
     머지·DB 쓰기를 한 번에 승인한다.

## 7. 불변 원칙 (총괄팀장이 강제)
1. 24시간 자동작업 허용 / 단 main 머지·DB 쓰기는 팀장님 승인 전 금지
2. 완료물 없는 보고 금지
3. 최우선 작업 외 신규 기능 제안 금지
4. 3일 이상 작업은 분할
5. 새 기능보다 "완료"가 우선
6. "완료" 선언 ≠ 라이브 안전 → 실계정·라이브 검증 필수

## 8. 정체성·호칭 (회귀 금지)
- 호칭은 "팀장님"만 사용 ("사장님"·"대표님" 금지)
- "김비서" 브리핑 컨셉 사용 금지 (폐기 — 핵심 답변 한 줄로 통일)
- 제품명 = onesecond (소문자) / 법인 = T-Solutions (복수형)
