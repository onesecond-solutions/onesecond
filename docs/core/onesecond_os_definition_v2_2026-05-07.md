# PART A — 원세컨드 전략 재수정 핵심 17섹션

> 본 PART는 팀장님이 2026-05-07에 작성하신 **전략 재수정 핵심**입니다. 본 PART의 모든 내용이 우선합니다.

지금까지 재정의된 방향 기준으로 보면 원세컨드는 더 이상:

- 스크립트 SaaS
- 게시판 서비스
- 보험 CRM

이 아닙니다.

**지금부터는: "보험 상담 흐름 운영체제"**

쪽으로 재정의되고 있습니다.

---

## §1. 원세컨드는 무엇인가

원세컨드는 보험 설계사용:

**"상담 흐름 유지 도구"**

입니다.

핵심은:
- 고객관리
- 계약관리
- CRM
- 교육

이 아니라,

👉 **상담 중 멈추지 않게 하는 것**

입니다.

---

## §2. 원세컨드의 진짜 경쟁자

경쟁 SaaS가 아닙니다.

실제 경쟁자는:
- 카카오톡 단톡방
- 엑셀
- 개인 메모
- 저장 안 된 노하우

입니다.

---

## §3. 왜 원세컨드가 시작되었는가

17년 현장 흐름에서 나온 문제:
- 상담 중 막힘
- 질문 폭탄
- 보험사 기준 계속 변경
- 카톡방 정보 휘발
- 개인 경험 의존
- 설계사마다 설명 품질 차이

이걸 해결하려고 시작.

---

## §4. 원세컨드의 핵심 철학 — 무료 원세컨드

절대:
- 고객정보 저장 안 함
- CRM 안 함
- 교육 플랫폼 안 함

왜냐면:

**"상담 실행 도구"**

정체성을 지켜야 하기 때문.

---

## §5. 무료 원세컨드의 본질

원세컨드는:
- 상담 실행
- 질문 흐름
- 카톡 대체
- 현장 정보 연결
- 보험뉴스 해석
- 자료 탐색
- 네비게이션

을 위한 플랫폼.

즉:

**"설계사 내부 흐름 플랫폼"**

입니다.

---

## §6. 스마트 게시판 전략이 바뀜

**예전:** 커뮤니티 게시판 느낌

↓

**지금:** "질문 데이터 생성 시스템" 으로 변경.

**핵심은:** 질문을 잘 쓰게 만드는 구조.

---

## §7. 스마트 게시판의 진짜 목적

질문 자체보다 중요한 건:

**질문이 구조적으로 축적되는 것**

입니다.

그래서:
- 질문 유형 유도
- 인수/상품 분기
- 단계형 입력
- 추천 UI

를 넣기 시작.

---

## §8. 네비게이션방의 재정의

네비게이션방은 단순 채팅방이 아닙니다.

**"실시간 운영 컨트롤타워"**

입니다.

여기서:
- 질문 생성
- 긴급 흐름
- 보험사 이슈
- 현장 반응

이 실시간으로 발생.

즉: **데이터 생성 엔진** 역할.

---

## §9. 게시판의 역할도 바뀜

게시판은 커뮤니티가 아니라

**"정리된 운영 자산 저장소"**

가 됨.

네비게이션방에서 발생한 흐름을:
- 정리
- 구조화
- 검색 가능 상태

로 만드는 공간.

---

## §10. 보험사 게시판의 위치도 바뀜

보험사 게시판은 메인 서비스가 아님.

**공급 레이어** 입니다.

사용자는 보험사 게시판에 직접 들어가는 게 아니라, 스마트 게시판에서:

- [메리츠]
- [DB]
- [롯데]

출처 뱃지만 보게 됨.

---

## §11. 보험뉴스의 역할

보험뉴스는 단순 기사 모음이 아닙니다.

**흐름:**

```
뉴스 → 해석 → 현장 적용 → 질문 → 스크립트 → 저장
```

즉: **뉴스 소비가 아니라 실행 흐름 생성** 입니다.

---

## §12. 스마트 설계서가 등장한 이유

17년 통찰: **"고객은 PDF를 받아도 안 본다"**

그래서:

**설계사:**
- PDF 업로드
- 형광펜
- 설명 메모

**고객:**
- 모바일 링크
- 강조 부분 탭
- 설명 확인

흐름으로 바뀜.

---

## §13. CRM은 결과로 등장함

원세컨드는 CRM 만들려고 시작한 게 아닙니다.

**흐름:**

```
상담 실행 → 질문 → 자료 → 설계서 → 고객 반응 → 상담 흐름 저장
```

하다 보니: **CRM으로 자연스럽게 흘러감.**

---

## §14. 그래서 사이트가 분리됨

### 원세컨드 (무료)
- 상담 실행
- 질문 흐름
- 카톡 대체
- 보험뉴스
- 네비게이션방
- 내부 참고

### 원세컨드 PRO (9,900원)
- 고객 흐름 저장
- 상담 이력
- 스마트 설계서
- CRM
- 운영 SaaS

---

## §15. 사이트를 나누는 이유

리스크 때문이 아니라:

**"정체성 보호"**

때문.

무료 원세컨드의:
- 고객정보 저장 X
- 교육 플랫폼 X
- CRM X

철학을 깨지 않기 위해 PRO를 분리.

---

## §16. 가장 중요한 최신 통찰

**"대면 영업도 결국 시작은 전화"**

**TM 흐름:**
```
접촉 → 관심 → 보장분석 → 예약
```

이 그대로 대면 시장에도 적용됨.

그래서 원세컨드는 TM만이 아니라 **대면 시장까지 흡수 가능성**이 생김.

---

## §17. 원세컨드의 최종 방향

원세컨드는 보험 CRM에서 출발한 게 아니라,

**"상담 흐름 디지털화"**

에서 시작함.

그리고 흐름이 축적되면서:

```
질문 데이터 → 검색 → 스크립트 → 설계서 → 고객 반응 → CRM
```

으로 확장되는 구조.

# PART B — Code 시스템 아키텍처 매핑

> 본 PART는 PART A의 17섹션을 **시스템 아키텍처 관점**으로 재구성한 Code 분석입니다. PART A와 충돌 시 PART A 우선.

---

## B-1. 핵심 재정의 (한 문장)

**원세컨드 = 보험 상담 흐름 운영체제 (Counseling Flow OS)**

CRM·SaaS·게시판이 아니라, **설계사가 상담 중 멈추지 않게 하는 흐름 OS**. 그 흐름 위에서 질문·자료·뉴스·네비게이션이 운영됨.

---

| OS 영역 | 원세컨드 매핑 | PART A 근거 |

---

## B-3. 5-Layer 아키텍처 (흐름 관점)

```
Layer 1. 실행 레이어        [스크립트 / 자료 / 뉴스]
                            ↓ 상담 중 즉시 접근
                            
Layer 2. 흐름 레이어        [네비게이션방 (실시간) / 스마트 게시판 (단계형 입력)]
                            ↓ 질문 생성 + 실시간 흐름
                            
Layer 3. 자산화 레이어      [게시판 (정리 저장) / 보험사 게시판 (공급)]
                            ↓ 정리·구조화·검색
                            
Layer 4. 고객 인터페이스    [스마트 설계서] ← PRO 한정
                            ↓ 고객 반응
                            
Layer 5. 흐름 저장          [CRM] ← 결과로 등장 (시작점 X)
```

각 Layer는 **흐름의 단계**와 매핑. CRM은 출발점이 아닌 Layer 5의 자연스러운 결과물.

## B-6. 운영 원칙 (Code 도출)

### 무료 OS (onesecond.solutions)
1. **격리 원칙** — 고객 식별정보(이름·생년월일·주소·전화번호) 입력·저장 차단 (정규식 + 안내 박스 + 가이드 모달)
2. **흐름 우선** — 상담 중 멈추지 않게 하는 모든 기능 (Layer 1~2) 우선
3. **자산화 원칙** — Layer 2 흐름 → Layer 3 정리·검색 자동 흐름
4. **공급 분리** — 보험사 게시판(공급) ↔ 스마트 게시판(소비) 명확히 분리

### PRO OS (pro.onesecond.solutions)
1. **흐름 저장 허용** — 고객 상담 흐름 저장 (스마트 설계서 + CRM)
2. **연결 우선** — 무료 OS의 Layer 3에서 PRO OS Layer 4·5로 자연 흐름
3. **결제 단순** — 9,900원 (§14, TM·대면 양 시장 흡수 심리적 가격대)

### 9역할 RBAC 정합
| 역할 | 무료 OS | PRO OS |
|---|---|---|
| `admin` | 전사 (전체 OS 관리) | 전사 |
| `ga_*` 4역할 | Layer 1~3 사용 | 결제 후 Layer 4·5 사용 가능 |
| `insurer_*` 4역할 | Layer 1~3 + **공급 레이어** (보험사 게시판) | 보험사 결제 후 Layer 4·5 가능 |
