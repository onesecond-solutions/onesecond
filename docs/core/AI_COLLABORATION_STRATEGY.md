## 2. 핵심 운영 철학 4종 (절대 흔들리지 않을 것) ⭐⭐⭐

원세컨드는 다음 4 철학으로 정의된다. 어떤 작업이든 이 철학을 회귀시키면 큰 사고:

### ✅ 현장의 소리 = 질문 운영 시스템
- ❌ 커뮤니티형 게시판 / CRM·SaaS / 일반 게시판
- 설계사가 현장에서 질문하고 운영자가 답하는 시스템
- 게시판 형태이지만 본질은 "질문 답변 운영"

### ✅ home_v2 = 운영형 홈
- ❌ 랜딩페이지 / 마케팅 페이지
- 4팀 가입자가 들어오는 메인 진입로
- Hero + 카드 그리드 + 통합 인증 모달
- 5/15 후 root `index.html`로 이전 예정

### ✅ Quick 메뉴 = 긴급 실행도구
- ❌ 일반 메뉴 / 부가 기능
- 설계사가 통화 중 멈춰서는 순간 1초 안에 실행하는 도구
- 1턴 골격 + 4그룹 + 검색

### ✅ 고정 프레임 구조 = 앱 셸
- ❌ 다중 페이지 / iframe·include 동적 frame
- `app.html` 단일 셸 + `pages/*` 동적 메뉴 로드
- `window.AppState` / `window.db` / `window.Auth` 통신
- 사이드바·탭바·검색은 셸이 담당

## 3. 5/10 본 세션 사고 학습 (영구 박음)

### 사고 본질
Code가 `_INDEX.md` 521줄을 200줄까지만 읽고 "Phase 1 잔여 5/15 후" 옛 분류를 그대로 인용 → 게시판 본진(메인 트랙 2 = 4팀 오픈 핵심 진입로)을 후순위로 잘못 분류.

### 사고 흐름
1. 세션 진입 → `_INDEX.md` 읽기 (Read 25K 토큰 한도)
2. offset 0 + limit 200 → 200줄까지만 들어감
3. line 29 "Phase 1 잔여 5/15 후" 표 보고 그대로 인용
4. 게시판 본진을 "5/15 후" 트랙으로 분류
5. 다음 트랙 후보 = #30 Custom SMTP / #46 동적 lookup / 종료 옵션 제시
6. 팀장님 지적 = "현장의 소리 작업해야 하는 거 몰라? 왜 아무런 액션이 없는거지"

### 사고의 진짜 원인
- (A) **컨텍스트 폭증** — `_INDEX.md` 521줄로 통째 안 들어감
- (B) **단일 진실 원천 부재** — 옛 분류와 새 본질이 동시 박힘
- (C) **명명 모호** — "메인/보조/융합/별/별건/임시 트랙" 7~8종
- (D) **활성 ↔ 폐기 페어 잔존** — `v1` / `v2` 둘 다 본 PC

### 처방 (2026-05-10 시점)
- **1단계 응급 안정화** — CURRENT 계층 신설 (`docs/00~04_*.md`) + `_INDEX.md` 분할 5 파일 + 본 문서
- **2단계 실제 구조 진단** — 본 PC raw 기반 active vs legacy 구분
- **3단계 운영 체계 재설계** — 폴더 마이그레이션 + archive 통합 (별 세션)

## 4. AI가 방향 자주 잃는 원인 (코드 관점) — 영구 명문화

### A. 이중·삼중 진실 원천 (Duplication of Truth)
- `_INDEX.md` ↔ `MEMORY.md` ↔ `CLAUDE.md` ↔ `docs/core/` ↔ `docs/specs/v2_*.md`
- 같은 결정이 여러 위치에 박혀 있어 갱신 시 일부만 동기화 → AI가 옛 raw 참조

### B. 컨텍스트 윈도우 vs 큰 그림 토큰 폭증 ⭐ 본 세션 사고 본질
- `_INDEX.md` 521줄 = 25K 토큰 한도 초과
- AI가 200줄까지만 읽고 후순위 분류 잘못

### C. 임시·보조·메인 트랙 명명 혼란
- 7~8종 명명 + Phase 1 / 1.5 / 2 / admin_v2 Phase D 4중 phase

### D. 폴더 구조 4중 archive + 폴더 typo
- `docs/deprecated/` + `claude_code/_archive/` + `_context/` + `_instructions/` + `_docs/`
- `claude_deskktop/` typo

### E. page · shell · template 3중 레이어 모호
- `pages/_template.html` (코드 기준서, 페이지 X)
- `pages/home.html` (셸 안 룰렛 카드)
- `pages/home_v2.html` (독립 메인 진입로)
- `app.html` (셸)

## 6. 다음 AI에게 (인계 톤)

이 프로젝트는 비개발자(팀장님)가 운영하는 보험 현장 운영 OS다. 다중 AI(Claude Code / Claude AI / Claude in Chrome / GPT)가 협업하는 구조이며, 컨텍스트 격차가 가장 큰 위험이다.

**작업 진입 시 무조건:**
1. CLAUDE.md + CURRENT 3 파일(`00~02`) 통째 읽기
2. 핵심 철학 4종 회귀 신호 즉시 보고
3. _INDEX.md 200줄 초과 시 분할 처방 위반 (Code 통째 못 읽으면 사고 재발)
4. "활성 vs 폐기" 구분 못 하겠으면 즉시 팀장님께 확인

**절대 하지 말 것:**
- 옛 raw 추정 인용 (구버전 Supabase / `docs/deprecated/` / 폐기 트랙)
- 핵심 철학 회귀 (현장의 소리 SaaS화 / home_v2 랜딩 / Quick 일반 메뉴 / 고정 프레임 → 다중 페이지)
- 팀장님 확인 전 "완료" / "완벽" 선언
