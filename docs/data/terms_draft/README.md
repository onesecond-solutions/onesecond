# terms_draft — 용어 채굴물 + 검수 큐

> 직원 에이전트가 수집한 용어 초안을 여기에 push → 팀장 코드가 외부 출처 대조 검수.
> 규격: `docs/decisions/2026-06-06_knowledge_cycle_v1.md`

## 파일 규약

- **채굴물 (직원 push):** `terms_NN.md` (50건/파일, 1사이클 = 6파일 = 300건)
- **검수 리포트 (팀장 코드 push):** `review_batchNN.md` (배치별)
- **DB 적재:** 이 폴더에서 **하지 않음**. 적재 SQL 생성=코드 / 실행=팀장님 단독.

## 용어 항목 필수 필드 (직원 채굴 시)

- 용어명 / 정의 / 분류(계약·보장·인수·판매·민원 등) / 검색 키워드
- **출처 URL** (없으면 검수에서 자동 '하' 판정)

## 검수 절차 (팀장 코드)

1. 배치(50건)별로 각 용어를 **외부 공신력 출처**와 대조
   (금융감독원 · 생명보험협회 · 손해보험협회 · 공식 용어집 등)
2. confidence 부여:
   - **상** — 복수 출처 일치
   - **중** — 단일 출처 일치 또는 경미한 표현 차
   - **하** — 출처 불일치 · 확인 불가 · 정의 모호
3. **출처 URL 누락 항목 = 자동 '하'** + 사유 "출처 없음"
4. 리포트 push: `review_batchNN.md` (용어별 판정 · confidence · 대조 출처 URL · 수정 제안)
5. 전 배치 완료 시 요약 보고: 상/중/하 건수 + '하' 목록 (팀장님 눈검수용)

## 눈검수 기준 (팀장님)

- 사이클 1~3: '하' 전건 + '중' 표본
- 사이클 4~: '하'만 (직전 승인율 95%↑ 시)

## 불변 원칙

- 출처 없는 항목 적재 금지
- 미승인분 검색 비노출
- 보험사 편향 X / 고객정보 유입 0 / 용어·지식 = 전 등급 무료
- 직원 채굴물 임의 수정 금지 (지적은 리포트로만)
