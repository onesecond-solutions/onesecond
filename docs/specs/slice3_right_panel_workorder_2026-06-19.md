## § Phase B. OCR 본문 원본/정제본 분리 (B-2) + 패널 표시

### B-1. 데이터 (DDL — 대표 결재)
- 원본(raw) 보존 + 정제본(clean) 컬럼 신설 (PR #832가 식별한 3테이블 대상)
- `search_text` 역할 재정의:
  - 표시 = 정제본(clean)
  - 검색 인덱스 = 원본 + 정제본 **합집합** (붙여쓰기·띄어쓰기 양쪽 매칭)
- 제안 DDL은 Code 작성 → 대표님 Dashboard 직접 실행. Code 실행 금지.
- 삭제 가드: 파일 삭제 시 raw + clean 동시 제거(메모리 삭제 3가드 준수)

### B-2. OCR 파이프라인 (`ocr-extract` / `ocr-batch`)
- raw 추출 = 현행 유지("원문 그대로, 생략 금지")
- 정제 단계 추가. 정제 프롬프트 제약(★강하게):
  - "내용 불변. 띄어쓰기·줄바꿈·문단만 복원."
  - "숫자·금액·보험사명·담보명·고유명사 변경/요약/생략/추가 절대 금지."
  - temperature 0
- 기존 축적분 = `ocr-batch` 재처리로 정제본 백필 (raw 보존, 이미지 재OCR 불요)

### B-3. 검색 (합집합)
- trgm 인덱스가 원본 + 정제본 둘 다 커버

### B-4. 표시 UI (우측 패널 본문)
- 본문 영역 상단에 [정제본 | 원본] 탭 2개, 기본 선택 = 정제본
- 정제본 탭: polished 조판(줄간격 1.8·문단 간격·검색어 강조·라이트 카드·스크롤)
- 원본 탭: raw 그대로(현행) — 대조용
- 검색어 하이라이트는 양쪽 탭 모두 적용
- 이미지/오버레이 정책은 메모리 #30 불변(딸깍은 이미지 클릭→기존 풀스크린
  오버레이 호출, 패널에 새로 만들지 않음)
