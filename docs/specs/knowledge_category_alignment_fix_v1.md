## 0. 배경

라이브 카테고리 화면에서 본문이 좌측 내비에 붙어 보입니다.

원인은 **기획팀장이 스펙에 정렬을 명시하지 않은 것**입니다.
`knowledge_category_layout_v2.md` §8-4에 "본문 폭 710px"만 적고
좌우 여백·정렬 규칙을 쓰지 않아, 구현이 좌측 정렬로 나왔습니다.

지식 페이지(`caregiver-history.html` 등)는 우측 사이드바가 있어 균형이 잡히는데,
카테고리 화면은 우측이 비어 한쪽으로 쏠려 보입니다.

## 1. 이번 범위 — 2건

### 1-1. 본문 중앙 정렬 (우선순위 1)

콘텐츠 블록을 **화면 가운데에 놓고 좌우 여백을 확보**합니다.

```css
.page{
  max-width: 1010px;   /* 본문 710 + 간격 40 + 우측 여유 260 */
  margin: 0 auto;      /* ★ 이 줄이 빠져 있었다 */
  padding: 30px 28px 80px;
}
```

- 읽는 본문 폭 **710px 불변** (기존 가독성 표준)
- 좌측 내비와 본문 사이에 여백이 생기는지 실측
- 우측 레일을 이번에 넣지 않으므로, 본문은 1010px 안에서 좌측 정렬되고
  **1010px 블록 자체가 중앙**에 옵니다

> 참고: 우측 260px 자리는 §3-2 항목이 들어올 자리입니다.
> 지금은 비워두되 **max-width는 1010px로 잡아** 나중에 레일을 넣을 때
> 본문 위치가 흔들리지 않게 합니다.

### 1-2. 상단 로고 → 홈 링크

카테고리 화면 상단 좌측에 **onesecond 로고**를 두고, 클릭 시 홈(`/`)으로 이동합니다.

```
[1s] onesecond
```

- 표기는 **소문자 `onesecond`** (대문자 ONESECOND / OneSecond 금지)
- 링크 대상 = `/` (de-iframe 백링크와 동일)
- 560px 이하에서는 마크만 남기고 텍스트 숨김
- 로고 마크는 시안이 임시입니다. **실제 앱 로고 자산이 있으면 그것으로 교체**하세요

### 1-3. 기존 "‹ 홈으로" 버튼 정리

로고가 홈으로 가므로 같은 동작이 둘이 됩니다.
**"‹ 홈으로" 버튼은 제거**하고 로고로 통합합니다.

## 3. 이번 범위 제외 — 사유 명시

시안 v3에 그려져 있으나 **이번에 구현하지 않습니다.**

### 3-1. 상단 검색창

현재 지식 자료가 3건입니다. 목록을 눈으로 훑는 편이 빠릅니다.
검색이 실제로 일을 하려면 자료가 쌓여야 합니다.

**도입 조건: 한 카테고리 안에서 목록을 훑기 어려워질 때.**

### 3-2. 우측 레일 (최근 본 자료 / 함께 쓰는 도구)

**"최근 본 자료"는 기획팀장이 화면만 그린 것입니다.**
열람 기록을 저장하는 구조가 있는지 확인하지 않았습니다.
데이터가 없으면 항상 빈 박스가 됩니다.

**선행 조건: 열람 기록 저장 여부 확인.** 없으면 별도 설계가 필요합니다.

"함께 쓰는 도구"(보장분석·스크립트 링크)는 링크만 걸면 되지만,
레일 자체를 이번에 만들지 않으므로 함께 보류합니다.

### 3-3. 검토 요청

위 두 항목은 **원장(레지스트리) 구축과 자료 축적이 끝난 뒤** 재논의합니다.
지금은 PR-A(원장 파일 생성)가 우선입니다.

이번 건은 **스펙 누락으로 발생한 재작업**입니다. 기록으로 남깁니다.

| 누락 | 결과 |
|---|---|
| 본문 폭만 쓰고 **정렬·여백 미기재** | 좌측 정렬로 구현 → 재작업 |

# 부록 — 총괄팀장 대조 검증 (2026-07-24)

> 기획팀장 원문은 **위까지 무수정 보존**. 아래는 총괄팀장이 실제 코드·라이브·대표 확정을 대조한 결과다.

## §1-1 중앙 정렬 = **채택 (A안, 대표 승인)**

콘텐츠 블록(`.kc-top`/`.kc-tabs`/`.kc-chips`/`.kc-scroll`)에 `max-width:1010px` + 좌우 `auto` 중앙 정렬 적용.
읽는 본문 `.kc-body` 710px은 불변, 우측 260px 자리는 향후 레일용으로 지금은 비워둔다.

**총괄 실측(적용 전, 2026-07-24) — 문제 재현값:**

| 폭 | 좌측 여백 | 우측 빈 공간 |
|---|---|---|
| 1280 | 28px | 486px |
| 1920 | 28px | 1126px |

**총괄 실측(적용 후, 2026-07-24) — `python -m http.server` + Playwright(Chromium) 헤드리스,
`window._kcShow('axis-cancer')` 직접 렌더로 확인(계정 정보 localStorage 주입 없음):**

| 폭 | 1010px 블록 좌/우 여백(대칭) | 710px 본문 우측 잔여(화면 끝까지) |
|---|---|---|
| 1280 | 107 / 107 | 407 |
| 1440 | 187 / 187 | 487 |
| 1920 | 427 / 427 | 727 |

- 세 폭 모두 `.kc-body` 실측 폭 710px 유지(변화 없음).
- 좌측 107 / 427 값은 총괄 사전 시뮬레이션과 정확히 일치. 사전 시뮬레이션이 적은 "우측 407 / 727"은
  1010px 블록의 우측 대칭 여백이 아니라 **710px 본문 기준 화면 끝까지의 잔여 공간**(대칭여백 + 블록 내 300px
  여유)이었음을 실측으로 확인·구분해 기록한다. 자세한 산출 근거는
  `docs/specs/knowledge_category_layout_v2.md` 부록 G-1 참조.
- 768 / 375에서는 `max-width:1010px`가 걸리지 않아 좌우 여백 불변(14px, `.wrap` 기존 패딩),
  가로 오버플로 0, 탭 줄 가로 스크롤 유지 확인.
- 칩 줄 `:empty{display:none}` 계속 동작, 긴 목록(80건 주입)에서 `.kc-scroll` 내부 스크롤만 발생하고
  뷰 루트 밖으로 안 넘침, 6축 전부 렌더, 비로그인 게이트 → `v-home` 강제 확인, 콘솔 에러 0(5개 폭 전수).
- 구현은 **CSS 전용**(`css/knowledge-category.css`) — `js/knowledge-category.js` DOM 구조 변경 없음.

## §1-2 상단 로고 추가 = **불채택**

사이드바 상단(`app.html:1992`)에 원세컨드 로고가 이미 있어, 카테고리 화면에 또 두면
한 화면에 로고가 2개가 된다. 시안 v3이 `.shell`/`.lnav`/`.appbar`를 직접 그린 **독립 페이지 전제**로
작성돼 우리 SPA 구조와 어긋난다(V3 타이포 트랙의 iframe 전제 불일치와 같은 유형).

## §1-3 '‹ 홈으로' 제거 = **불채택**

근거로 든 "로고를 누르면 홈으로 간다"가 사실이 아니었다(실측 onclick = 사이드바 접기/펴기).
또한 원문 스펙 v2 §4-3 "상단에 `‹ 홈으로` 링크는 유지"와 자기모순이다.
**단, 2026-07-24 별도 PR(`fix/side-logo-home`, 건 1)로 사이드바 로고를 홈 이동으로 고쳤으므로,
이후 '‹ 홈으로' 중복 여부는 재논의 대상**이다(이번 PR 범위 아님).
