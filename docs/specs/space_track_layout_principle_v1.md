## 7. 전달 전략 — 바텀시트 단계 공개 (감추기 > 만들기)

### 7.1 1단계 (현재) — 위장된 홈
- 바텀시트 = SPACE 권한 트리. A영역(상단 헤더) 바로 아래까지 끌어올려 고정.
- 핸들/드래그 어포던스 숨김 → 사용자는 시트를 "화면"으로 인지.
- 시트는 D영역 위에 오버레이(앱 셸 구조 유지, 다중 페이지 X).
- 이 단계에서 검색 미제공(트리 네비게이션만).

### 7.2 준비 (비공개)
- 검색 홈(D영역)을 사용자에게 노출하지 않고 준비.
- 검색기 + 검색결과 작업이 끝날 때까지 시트 뒤에 가려둠.

### 7.3 2단계 (공개) — 커튼 내리기
- 검색기·검색결과 완성 시: 핸들을 "드래그 가능"으로 전환 + 코치마크(스포트라이트).
- 안내문: "이걸 잡고 내려보세요 / 클릭해보세요" → 시트 닫힘 → 검색 장착 홈 공개.
- 잠금→해제 전환은 1회 강한 온보딩 순간. 어설프면 버그로 인지됨.

### 7.4 3단계 (정착) — 이원 사용
- 트리 시트(권한 축) / 검색 홈(자료 축)을 구분해 사용.

### 7.5 끊으면 안 되는 끈 3
1. 스코프 바인딩: 시트 트리 선택 SPACE → 홈 검색 범위로 인계(window.AppState 공유).
2. 잠금→해제 전환 + 코치마크 1회.
3. 데스크탑 등가 매핑: 좌측 드로어=트리, 중앙=검색 홈. 모바일 먼저.

### 7.6 공개 트리거 (확정)
- 개인 정착 신호 후 점진 공개 → 안정 확인 후 잔여 일괄.
- 공개 여부 = 서버 제어 원격 플래그(재배포 없이 토글). 날짜·기준 하드코딩 금지.

## 부록 A. 총괄팀장 현 코드 진단 (2026-06-12, Code)

> §7을 현재 코드 기준으로 평가한 결과. 미래 세션이 매번 재조사하지 않도록 박제.
> **결론: §7은 PR 하나짜리가 아니라 "검색 홈 트랙이 선결인 대형 멀티트랙".**

### A-1. 이미 있는 기반 (재사용)
| 자산 | 현재 상태 |
|---|---|
| 바텀시트 셸 `.os-sheet` | 3단 스냅(46px / 42vh / 84vh)·핸들·그립·드래그 존재. **단 현재 내용물은 스크립트/채팅/자료 트레이(`OSSheet`)** — 권한 트리 아님 |
| `app_settings` 테이블 + 어드민 토글 UI | 존재 (admin_v2.js·admin-console.js 읽기/쓰기) |
| `window.AppState` 단일 객체 | 존재 (js/auth.js, 인증·role·plan만 — 스코프 축 없음) |

### A-2. 만들어야 하는 것 (격차)
| # | §7 요구 | 현재 | 공사 |
|---|---|---|---|
| 1 | 바텀시트 = SPACE 권한 트리 (§7.1) | 시트는 트레이 용도(OSSheet) | 중 — 시트 콘텐츠 교체/신설 |
| 2 | **검색 홈(D영역) 준비** (§7.2) | **app.html에 없음.** 현재는 방 로컬검색만(`toggleLocalSearch`) | 🔴 대형 — 검색기+검색결과 = 별도 본진 트랙 |
| 3 | 스코프 바인딩 시트→홈 (§7.5-1) | AppState에 스코프 축 없음 | 중 — 상태 축 신설 |
| 4 | 잠금→해제 + 코치마크 (§7.3·7.5-2) | 없음 | 소~중 — 1회 온보딩 |
| 5 | 원격 플래그 공개 토글 (§7.6) | app_settings 있으나 **라이브 SPA가 런타임에 안 읽음**(2026-06-12 #601 발견 = 소비처 끊김) | 중 — 라이브 배선 신규 |
| 6 | 데스크탑 등가 매핑 (§7.5-3) | 모바일/데스크탑 동일 셸 | 소~중 |

### A-3. 핵심 의존성 — 병목은 "검색 홈"
- §7.3 "커튼 내리기"로 공개하려 해도, **공개할 검색 홈(§7.2)이 아직 없다.**
- 즉 §7 전체의 병목은 시트(1)도 플래그(5)도 아닌 **검색 홈(2)**. 나머지는 검색 홈이 있어야 의미가 생긴다.
- 검색 홈의 연료 = **채굴된 지식 자료**(`knowledge_entries`). 채굴은 시간이 걸리는 장기 작업.
  → **"채굴부터 준비하며 시간을 번다"가 맞는 순서** (2026-06-12 대표님 판단).

### A-4. 진행 순서 (권고)
1. **검색 자료 채굴** — 검색 홈의 연료 적재. 채굴팀 재가동(현재 🔴 멈춤, 914 정체·자동 스케줄러 부재). 장기·병행.
2. **검색 홈(D영역) 트랙** — 검색기+검색결과. §7의 8할.
3. 시트=권한 트리 재설계 + 스코프 바인딩(AppState 스코프 축).
4. 원격 플래그 라이브 배선 + 코치마크 = 공개 스위치.

> ⚠️ §7.6 원격 플래그는 "재배포 없이 토글"을 전제하나, 현재 라이브 SPA가 `app_settings`를 런타임에 읽지 않는다(소비처 끊김). 이 배선이 선행되어야 §7.6이 성립한다.
