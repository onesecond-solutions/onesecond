# 삼성화재 상품 라인업·납입면제 독립 페이지 2종 — 화면 사양 v1

> 작성: 2026-06-28 / 기획팀장(Web) 작업지시서 → 총괄팀장(Code) 구현
> 선행: 이전 `v2_insurer_hub_layout` 작업지시서2 **전면 폐기**. 본 문서가 유일한 유효 사양.
> 브랜치: `feat/samsung-lineup-waiver` / main 직접 push 금지
> 게이트: 임태성 실장 user_id `98c5f4f9-...` (회사 허브 진입). 일반 사용자·어드민 포함 타계정 무영향.

## 1. 범위 (이 3가지뿐)

1. **삼성화재 회사 허브 = 수정 금지.** BMI·결제·콜센터·청구·원전산·소식지 일절 미접촉. 유일 변경 = 아코디언의 "상품 라인업" 줄이 [준비 중]을 떼고 클릭 시 독립 페이지로 이동 + "상품별 납입면제" 줄 신규 추가(클릭 시 독립 페이지).
2. **상품 라인업 독립 페이지** (`#v-samsung-lineup`) — 아코디언 펼치기 아님. 19종 표. 열: 대분류 | 상품명 | 일반형·무해지 및 적용 종 | 가입연령 | 핵심 특징.
3. **상품별 납입면제 독립 페이지** (`#v-samsung-waiver`) — 15종 표. 열: 상품명 | 납입면제 사유 | 무해지형 추가 사유.

**공통 — 뒤로가기:** 두 페이지 뒤로가기 → 삼성화재 회사 허브의 "검색 상태"로 복귀(검색어 "삼성화재" 유지, `_hsSearch` 재실행).

## 2. 데이터

- 라인업 19종 / 납입면제 15종 = 본 작업 시점 기획팀장 전달 정형 JSON 그대로 화면 상수로 적재(견본 아님, 전량).
- 출처 표기: 삼성화재 GA소식지 2026-06 (라인업 p.17 / 납입면제 p.16).
- DDL/DB 불필요(화면 사양). 통합검색 미연결.

### 도메인 규칙 (전보험사 공통, 메모리 `insurance_surrender_value_product_types`)
- 무해지형 = 해지환급금 미지급+일부지급형(납입 중 해지 시 환급금 없음/거의없음, 보험료 저렴).
- 일반형 = 표준형(해지 시 환급금 발생). "무해지" 단어 없으면 거의 일반형.
- 삼성화재는 종(種)별 `구분` 명시 → 명시값 우선. 상품명 추정은 타 보험사용 fallback.

## 3. 구현 (로드맵 #v-roadmap 게이트 패턴 동일)

- 신규 뷰 2개 `#v-samsung-lineup`·`#v-samsung-waiver` (`.body .view`).
- showView 라우트 가드: 두 key는 `_canSeeCompanyHub()` 아니면 `home` 강제(직접 URL `?view=samsung-lineup` 차단). VALID_VIEWS 미등록 = 부팅 직접진입도 home.
- 로더 `_samsungRender(key)`: 진입 시 표 렌더. 이중 가드(`_canSeeCompanyHub`).
- 허브 진입 `_openSamsung(kind)`: 현재 검색어 저장 후 showView. 뒤로 `_samsungBack()`: showView('home') + 검색어 복원 + `_hsSearch` 재실행.
- 허브 `row()` 헬퍼에 `opts.nav` 추가(내부 뷰 이동). 삼성화재일 때만 2줄 nav, 타 회사는 기존 [준비 중] 유지.

## 4. 하지 않는 것

## 6. 보완 v2 (2026-06-29) — 대분류 세로 병합(rowspan) + 하단 안내문

기획팀장(Web) 작업지시서 보완 v2 반영. 범위 확장 아님(라인업 페이지 한정).

- **대분류 세로 병합(rowspan)**: `_samsungLineup` 배열에서 같은 `대분류`가 **연속하는 구간**을 한 그룹으로 묶어 첫 행에만 rowspan 셀(`.smsg-cat`), 이후 행은 대분류 칸 생략. 행마다 반복 표기 금지. 소식지 p.17 Line-up과 동일.
  - 그룹: 신상품2·치료비1·건강4·간편8·자녀1·운전자1·가정1·치아1 = 8그룹/19행(rowspan 합계=전체 행 검증).
  - 대분류 셀 = 세로 가운데 정렬·가운데 정렬·굵게. 색칠·배지 없음.
  - 그룹 경계 가로 구분선 = 그룹 첫 행(`tr.smsg-grp`) 상단 2px(첫 그룹은 thead 경계 중복 방지로 생략).
- **하단 안내문**: 표 바로 아래 `.smsg-note`(보조 텍스트·작게·흐리게, 강조/배지 없음).
  - 문구: "본 라인업은 삼성화재 GA소식지 2026-06(p.17) 기준으로, 실제 가입 조건은 변경될 수 있으며 정확한 사항은 약관 및 원수사 자료를 확인하시기 바랍니다."
- 데이터·행 순서·대분류 묶음 = json 그대로(재정렬·재분류·중분류 생성·색칠·원본 이미지 임베드 없음).
- 적용 = 라인업만(`_samsungRender` else 분기). 납입면제 표 미접촉.

## 7. 보완 v3 (2026-06-29) — 운영형태 원문 그대로 + 상품명 줄바꿈 방지

v2의 운영형태 배지화(무해지/일반형 분해) **철회·교정**. 원본 소식지엔 "일반형" 표기가 없는 상품이 있고, 무해지/일반형은 내부 분류 개념이라 화면 표시에 넣지 않음.

- **운영형태 = 원문 그대로**: `_samsungLineup`에 `운영형태_원문` 필드 신설(기존 `환급형` 배열·`_smHwanCell` 배지 렌더 제거). 값 = 삼성화재 GA소식지 2026-06 **p.17 Line-up '운영형태' 컬럼 원문**(`upgrade_20260521/삼성화재 202606/samsung_lineup.jpg` 이미지 직접 추출, 19종 전량). 종 구분은 ` / `, 속성은 `, `. 배지·색·임의 변환 없음.
  - 예: 365 당당한="1종 무해지, 납입면제 / 2종 납입면제" · 천만안심="1종 납입면제 / 2종 일반형 / 4종 무해지, 납입면제" · 오토바이/가정/치아="일반형". 원문에 "일반형" 있는 상품만 일반형 표시.
- **컬럼 헤더** "일반형·무해지 및 적용 종" → **"운영형태"**.
- **상품명 줄바꿈 방지**: 상품명·운영형태·가입연령 = `white-space:nowrap`(한 줄 유지). 핵심특징 컬럼이 나머지 폭 흡수(wrap). 긴 이름(365 당당한 새로고침 100세 등) 두 줄 깨짐 해소.
- 대분류 rowspan(v2)·행 순서·하단 안내문 유지. 납입면제 표 미접촉.
