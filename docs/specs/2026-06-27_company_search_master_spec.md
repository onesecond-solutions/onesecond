## §1. 전역 표준 — 타이포그래피 기준선 + 토큰화 [✅ ACTIVE / ❓선결 1건]
목적: "키워줘" 과잉보정(너무 키움+너무 진하게) 구조적 차단. 기준선 = 구글급 절제. 가독성 = [크기·대비·행간·위계·여백] 시스템. "면은 무채색, 색은 신호점에만"의 타이포 판.

**선결 Q**: 타이포 토큰 실효 위치 = app.html inline :root(line 37) vs tokens-v2.css 중 어디? 분리 상태면: 토큰은 inline :root에 정의(실효), tokens-v2.css엔 미러만(연결은 별도 결재).

**토큰 명세**:
```
--fs-caption:12 / --fs-meta:13 / --fs-body:14(base) / --fs-body-lg:15
--fs-title-sm:16 / --fs-title:18 / --fs-heading:20(최대)
--fw-regular:400 / --fw-medium:500   (600·700 가독성 레버 사용 금지)
--lh-tight:1.4 / --lh-base:1.6 / --lh-relaxed:1.7 / --lh-dense:1.5
```

**적용 규칙(자ruler — 모든 후속 수정 동일 적용)**:
1. 본문 기본 = 14px+400+1.6. 크게·진하게가 기본 아님.
2. 위계 = 크기차+색+여백. bold 남발 금지(--fw-medium까지).
3. 라벨=무채색(secondary/muted), 값=primary, 신호색(accent)은 신호점만(전화·링크).
4. "키워줘" = 한 노치만(14→15). 두세 노치 점프·굵기 보정 금지.
5. 11px 미만 금지 / 본문 16px 초과 금지(헤딩 제외).

**1차 적용 대상**: 살아있는 회사 허브 아코디언 한정(동결 영역 미적용).

## §2. AI검색 배너 — 플랫폼 운영자(대표님) 단독 게이트 [✅ ACTIVE]
확정: 검색창 하단 "원세컨드 AI 검색" 배너를 대표님(T-Solutions, cross-tenant)에게만 노출. 조직 관리자·팀장·지점장·일반 = 노출 X. CRM 단독 베타와 동일 scope.
구현: 운영자 아닐 때 DOM 미렌더(display:none 금지). 판정 = SECURITY DEFINER 함수(서버 권위, 클라 role 분기 금지). 기존 운영자 판정 함수 있으면 재사용, 없으면 신설 제안만(DDL 결재 후).
가드: 비운영자 3계정 = 배너 마크업 부재(숨김 아님) + AI검색 엔드포인트 강제 호출 시 서버단 차단.

## §3. 회사 허브 아코디언 (LIVE 영역 내부)
**3-1. BMI "29 초과 방진" 누락 = 추출 버그 [❓선결]** — 표시 결정 아님. _hubPick이 회사 블록을 일부만 잘라와 내용 실제 누락. 선결: BMI content_html 삼성화재 구조 SELECT(미회신). 구조 확인 후 회사 블록 전체 추출되게 보강. 동결 영역에 두지 말 것 — 추출 고쳐야 살아남.

**3-2. 콜센터 홈페이지 링크 추가 [🔒 BLOCKED]** — 콜센터 content_html에 URL 없음(false 확인). 현재 불가. 풀이: §5 insurers 마스터에 회사별 홈페이지 URL 채운 뒤 연결(콜센터 카드만 따로 채우지 말 것).

**3-3. 콜센터 번호 = accent 색 [❓선결]** — content_html 평문이라 CSS 불가. 구조 정비 필요(§1 타이포 규칙 3과 연동).

**3-4. 보험금 청구 안내 = 새창 링크 [🔒 BLOCKED → ❓선결]** — 펼침 X, 해당 보험사 청구 페이지로 새창(원전산과 동일 패턴, rel="noopener"). 회사별 청구 URL 필요 → §5 insurers 마스터에 채운 뒤 배선.

**3-5. 원전산 설계 바로가기 [✅ 구현 완료 / ❓URL 소스 선결]** — 새창 정상 구현(유지). 선결(배선 정합용): 현 URL 출처 = insurers 테이블 vs 하드코딩 배열? / 회사당 1개 vs 채널·심사유형 분기? 기획 확정: URL 소스는 insurers 마스터 단일 source로 수렴(신규 매핑 테이블 신설 X).

**3-6. 상품 라인업 [❓선결 — 대형]** — 삼성화재 19장 이미지 멀티모달 추출 후 연결.

## §4. 상품 라인업 아래 — 동결 영역 (대표님 재정리 대기) [⏸ FROZEN 전부]
※ 손대지 말 것. 현재 머지 상태 보존. 아래는 재정리 시 택일할 후보로만 기록.
- 4-1. 일반 결과 카드·소식지 목록 노출 여부 — 회사 허브와 중복 의심. 빼기/유지 대표님 결정.
- 4-2. 소식지 칩 자료명 구분 — 6월 3개 = 중복 아님(GA소식지/순통치 판매비법서/6월1주차 1매형 = 다른 3종). 해법은 칩에 자료명 부착("6월·GA소식지" 등)이나, 이 영역 살릴 때만.
- 4-3. 일반 결과 클릭 = 새탭 연결 — 4-1에서 "유지" 결정 시에만 필요(미리보기 제거됨).
- 4-4. 탭·요약줄 카운트 = "노출 준비된 건수" 기준 전환 — 적재량 아님, 정제·승인분만 COUNT(현재는 전부 0). 이 영역 처리 시 동반. 안내글(post_type='guide') COUNT 제외 유지.

## §5. 데이터 레이어 — insurers 마스터 (공급) [❓선결 / 거버넌스]
회사 × URL(원전산·청구·홈페이지) = insurers 마스터 단일 source.
- 신규 전용 테이블 신설 금지, 기존 공급 레이어 골조 재사용.
- posts(수요)와 절대 분리 — 섞으면 후복구 불가.
- 스키마는 회사 × (채널 대면/TM · 심사유형 일반/간편) 변형 수용 가능하게(데이터는 회사당 1개로 시작).
- GitHub 승인 파일 = 상위 진실. Supabase 읽기 전용·직접 INSERT 금지. 컬럼 신설 = 대표님 DDL 결재.

## §6. 공통 검증 — 완료 선언 요건
- 변경 항목은 시크릿 GA 라이브 3계정 게이트(일반/팀장·매니저/지점장) 통과.
- DB·Storage 무변경 항목은 행 수·파일 수 동일 확인.
- 완료 = PR 해시 + 라이브 측정 후 선언(자체 점검 통과만으로 완료 선언 금지).
- CRUD 4종 기본 점검, U/D 제외 시 사유 명시.
