# 회사별 비교형 공통 레이아웃 스펙 v1

- **작성:** 2026-07-05 · 총괄팀장(Code) 조사
- **상태:** 스펙(착수 전) — 대표 승인 대기. **즉시 리팩터 금지.**
- **결재:** 상품라인업 `pl-*` 레이아웃을 회사별 비교형 공통 레이아웃의 **기준 화면**으로 채택 (2026-07-05 대표).
- **성격:** 조사·기록 문서. 코드 변경 0. 통합검색(`_svRender`) 트랙과 분리.

---

## 0. 배경

검색축이 늘어나며(상품 라인업 · 담보 검색 · 알릴의무 검색 · BMI …) 축마다 레이아웃이 다르면 유지보수가 붕괴한다. 조사 결과 `pl-*` 레이아웃은 **이미 소식지(`#v-newsletters`, `nlp-` prefix)에서 CSS·DOM 골격을 재사용**하고 있어 mode-agnostic 성이 검증됐다. 따라서 상품라인업 `pl-*`을 회사별 비교형 공통 레이아웃의 기준으로 삼아 3모드로 확장한다.

---

## 1. 공통 레이아웃 목적

상품 / 담보 / 알릴의무 검색축을 **같은 화면 구조로 재사용**한다. 축이 늘어도 레이아웃은 하나, 데이터·필터·상세만 모드별로 주입한다.

---

## 2. 공통 슬롯

| 슬롯 | DOM (`pl-*`) | 렌더 함수 | 역할 |
|---|---|---|---|
| 좌측 회사 목록 | `.pl-side` > `.pl-pool`(손해/생명) · `.pl-search` · `#pl-coList` | `_plRenderSide` | 회사 목록 + 건수, 손/생 토글, 검색 |
| 상단 필터 칩 | `.pl-themes` · `.pl-scope`(전체조망/회사별) · `.pl-cnt` | `_plRenderThemes` · `_plRenderCount` | 모드별 축 필터 |
| 중앙 회사별 카드 | `.pl-body` > `.pl-main` | `_plRenderAll`(전체조망) · `_plRenderCo`(회사별) · `_plCardAll` | 회사별 상품 카드 그룹 |
| 우측 상세 패널 | `.pl-drawer` > `.pl-dr-in` | `_plOpenDrawer` | 선택 항목 상세 |

- **CSS:** `.pl-*` 전체 재사용(소식지 재사용으로 검증). 추가 최소.
- **주 진입축이 모드마다 다름:** 상품=회사(좌측 우선) / 담보·알릴의무=필터(상단 우선). config로 처리하되 UX 미세조정 필요.

---

## 3. 1차 공통화 대상 (3모드)

| 슬롯 | ① 상품 라인업 (기준) | ② 담보 검색 | ③ 알릴의무 검색 |
|---|---|---|---|
| 좌 회사목록 | 회사(손/생) | 그 담보 취급 회사 | 회사 |
| 상단 필터 | 테마 19종 | **담보축 8종** | **건강체/유병자 · 고지코드/유병자 세분화** |
| 중앙 카드 | 상품(상품군 그룹) | 회사별 담보상품 + 금액 | 회사별 상품 + 고지유형·코드 |
| 우 상세 | 상품군·연령·특징·테마 | 담보·금액·출처·확인상태 | 고지코드·입원수술·중대질환·납면 |
| 배지 | HOT/NEW | O / △ / 확인필요 | 확정 / 코드확인 / 유형확인 |
| 정렬 | 회사 | 확인상태 우선(§6) | 신뢰등급 |

**① 상품 라인업** = 검색·비교용 대표 화면, 공통 레이아웃의 기준 화면.
**② 담보 검색** = 좌 회사목록 / 상단 담보 필터 / 중앙 회사별 담보상품 카드 / 우 상세 구조로 재사용 검토.
**③ 알릴의무 검색** = 좌 회사목록 / 상단 건강체·유병자·고지코드 필터 / 중앙 회사별 상품·고지유형 카드 / 우 상세 구조로 재사용 검토.

## 4. 공통화 후보 — 로드맵 (즉시 미포함)

- 로드맵(`#v-roadmap`, `rm-*`)은 **원본 스펙 / 검수용 씨드** 성격이다. 상품별 심사구분·가입연령·보험기간·납입기간·가입금액·**인수기준·고지기준**·강점·판매상태·출처 같은 깊은 원문성 정보를 가진다.
- 상품라인업이 검색·비교용 **가공 화면**인 것과 성격이 다르므로 **즉시 공통화 대상에 포함하지 않는다.**
- 다만 레이아웃상 로드맵의 3칼럼은 `pl-*` 공통 레이아웃에 흡수 가능한 **장기 후보**로 기록한다(라인업 레이아웃이 상위 호환).
- ⭐ **로드맵 seed의 `인수기준`·`고지기준` 필드는 알릴의무 확정 데이터 확대의 1차 원료다.** 채굴팀장이 원문 소싱/조사 트랙에서 **1순위로 검토**할 자료로 표시한다.
  - 소스: `docs/data/2026-06-27_product_lineup_seed.json` (raw_seed 초안, 검수 전).
  - 필드: `coverType`(심사구분) · `underwriting`(인수·고지기준 원문) 등.

## 5. 데이터 adapter 구조

레이아웃은 공통, **데이터 형태만 adapter로 흡수**한다.

**공통 회사-그룹 모델**
```
{ companies: [ { name, section, color, items: [ { … } ] } ] }
```

**모드별 adapter (raw → 공통 모델)**

| adapter | 원본 데이터 | 비고 |
|---|---|---|
| 상품 데이터 adapter | `data/insurer_products_2607.json` (246) | 기존 `loadProductLineup` 매핑 재사용 |
| 담보 데이터 adapter | `design-preview/dambo-product-links-v1.json` (113) | 회사명·상품명·표준담보명·금액·확인상태·출처·**노출제외**. `노출제외:true` 필터 반영 |
| 알릴의무 데이터 adapter | (현재 `goji-search-taste.html` **인라인 RECS 114**) | **JSON 분리 선행 필요** (`goji-*-v1.json`) |
| **로드맵 후보 adapter** | `docs/data/2026-06-27_product_lineup_seed.json` | **기록만.** 즉시 구현 X. 인수/고지 필드는 알릴의무 원료 |

**모드별 config (레이아웃 공통, 아래만 주입)**
- 상단 필터축(테마 / 담보 / 고지코드) + 값 목록 + 매칭 함수
- 카드 보조필드(금액·확인상태 / 고지유형·코드·등급)
- 상세 필드셋 · 배지 스킴 · 정렬 규칙

**변경 예상 파일:**

## 8. 후속 착수 조건

아래 4가지 **전부 충족 + 별도 승인** 후에만 공통화 착수:
1. 상품 246건 통합검색 연결 **안정화 완료** 후
2. 대표가 **공통 레이아웃 스펙 승인** 후
3. **알릴의무 JSON 분리 여부 결정** 후
