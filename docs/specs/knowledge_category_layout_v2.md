## 0. 배경

지금까지 홈에 지식 자료를 낱개 카드로 늘어놓았습니다.
자료가 늘어나면 홈이 카드밭이 되고, 자료가 적으면 빈 티가 납니다.

앞으로는 **큰 카테고리를 그릇으로 쓰고 그 안에 자료를 담습니다.**
카테고리는 보장분석 4축과 같은 축으로 잡아, 보장분석 → 지식 자료 흐름이 이어지게 합니다.

---

## 1. 확정 규칙 8가지

| # | 규칙 |
|---|---|
| 1 | 홈에서는 **큰 카테고리 카드만** 둔다 |
| 2 | 카테고리 안에서는 **제목 · 한 줄 설명 · 업데이트일**이 있는 문서 목록을 보여준다 |
| 3 | 자료가 내용상 **두 갈래 이상으로 나뉘면 건수와 무관하게** 그룹화한다 |
| 4 | 문서 노출 범위는 **새 visibility 값을 수동 입력하지 않고 기존 발행등급에서 유도**한다 |
| 5 | 공개 내용과 내부 내용은 **가급적 별도 문서 또는 별도 보충자료로 분리**한다 |
| 6 | 카운트는 **현재 사용자에게 실제 노출되는 문서 수**로 계산한다 |
| 7 | 읽는 시간은 **자동 계산 전까지 표시하지 않는다** |
| 8 | 카테고리 전환은 **기존 스크립트 페이지의 탭·칩 컴포넌트를 재사용**한다 (신규 제작 금지) |

## 2. 카테고리 구성

보장분석 4축 + 2개. 홈 카드는 이 6개로 고정합니다.

| 카테고리 | 상단 한 줄 (갈고리) |
|---|---|
| **의료실비** | 같은 실손보험이라도 가입 시기에 따라 보장과 자기부담이 다릅니다. |
| **암** | 일반암이라는 말에 회사마다 다른 범위가 들어 있습니다. |
| **뇌 · 심장** | 뇌졸중 진단비가 있다고 뇌혈관질환 전체가 보장되는 것은 아닙니다. |
| **수술비** | 수술비라는 이름이 같아도 지급 횟수와 인정 범위는 다릅니다. |
| **간병** | 간병보험은 누가 간병하느냐에 따라 지급 구조가 달라집니다. |
| **기타** | (준비 중) |

### 상단 문장 작성 원칙
카테고리 설명은 **요약문이 아니라 첫 번째 질문 또는 갈고리**입니다.
"암보험의 보장 범위와 치료비 구조 변화를 정리했습니다" 같은 문장은
목차 제목을 풀어쓴 것뿐이라 읽을 이유를 만들지 못합니다.

한 줄로 짧게 쓰되, **읽어야 할 이유가 문장 안에 있어야** 합니다.

## 3. 노출 범위 — 발행등급에서 유도

### 3-1. 새 필드를 사람이 입력하지 않는다

`visibility: public / advisor / internal` 같은 값을 별도로 두면
발행등급과 두 벌로 관리하게 되고, 둘이 어긋나는 순간 추적이 불가능해집니다.

**문서 노출 범위는 본문에 포함된 발행등급에서 시스템이 계산합니다.**

| 문서 안에 든 사실의 등급 | 노출 범위 |
|---|---|
| `customer_ok`만 | 일반 공개 (비로그인 포함) |
| `internal_only` 포함 | 로그인한 설계사만 |
| `customer_blocked` 포함 | 게재 불가 |

### 3-2. 다만 제작 단계에서 분리한다

문서 안에 `customer_ok`가 대부분이고 `internal_only` 한 문장만 섞였다고
문서 전체를 설계사 전용으로 묶으면, **공개할 수 있는 좋은 자료까지 같이 잠깁니다.**

그래서 운영 규칙은 이렇습니다.

> 공개 가능한 내용과 내부 내용을 **가능한 한 같은 문서에 섞지 않는다.**
> 불가피하게 섞인 경우 **공개본과 설계사용 보충자료를 분리한다.**

예시:
```
암주요치료비 변천사        → customer_ok 공개 본문
암주요치료비 상담 포인트    → internal_only 별도 보충자료
```

### 3-3. 목록에 표시
설계사 전용 문서는 목록에서 **`설계사 전용` 배지**로 구분합니다.
비로그인 사용자에게는 배지가 아니라 **문서 자체가 노출되지 않습니다.**

## 4. 카테고리 전환 — 탭 · 칩 (신규 제작 금지)

### 4-1. 기존 컴포넌트를 그대로 쓴다

스크립트 페이지에 이미 **탭(윗줄) + 칩(아랫줄)** 2단 전환 UI가 구현돼 있습니다.
지식 카테고리도 구조가 같으므로 **그 컴포넌트를 재사용합니다.**

```
스크립트 페이지 (기존)
  탭 = 상담 단계 (인트로 / 신규 상품 문의 / 보험 점검 문의 / 설명 자료 / 상담 CTA)
  칩 = 보장 영역 (의료실비 / 암 / 뇌·심장 / 수술비 / 종합보험 / 간병인)

지식 카테고리 (신규)
  탭 = 카테고리 (의료실비 / 암 / 뇌·심장 / 수술비 / 간병 / 기타)
  칩 = 그룹     (전체 / 변천사·구조 / 확인 포인트 / 상담 자료)
```

새 컴포넌트를 만들지 않습니다. 설계사가 이미 아는 조작이라 학습 비용이 0입니다.

### 4-2. 칩은 조건부로만 나타난다

- 카테고리에 **그룹이 없으면 칩 줄 자체를 감춘다** (`display:none`)
- 그룹이 생기면 나타나고, 맨 앞에 `전체` 칩을 둔다
- 탭·칩 모두 **뒤에 건수를 표시**한다 (노출 기준 카운트)

### 4-3. 홈 복귀 동선
탭으로 카테고리를 옮길 수 있으므로 홈까지 돌아갈 필요가 없습니다.
다만 상단에 `‹ 홈으로` 링크는 유지합니다.

---

## 5. 카운트

**전체 등록 건수가 아니라 현재 사용자에게 실제 보이는 문서 수**입니다.

```
비로그인     자료 2건
설계사 로그인  자료 6건
```

- `customer_blocked`는 **어느 카운트에도 들어가지 않습니다.**
- 팀장님·검수 화면에서만 `게재 보류 3건` 형태로 별도 표시합니다.
- 탭·칩에 붙는 건수도 **같은 기준**으로 계산합니다.

카운트가 로그인 전후로 같으면 "4건은 어디 갔나"가 됩니다. 반드시 분리하세요.

---

## 6. 그룹화

### 기준 = 건수가 아니라 성격

자료 수로 임계값을 정하지 않습니다.
**성격이 두 갈래 이상으로 분명히 나뉘면 건수와 무관하게 묶습니다.**

암 자료가 7건이어도 아래처럼 갈리면 바로 그룹화합니다.

```
변천사 · 구조
  · 암 분류 체계 변천사
  · 암주요치료비 변천사
  · 재진단암 담보의 변화

확인 포인트
  · 대장점막내암, 어디까지 보장되나
  · 유사암과 소액암은 같은 말인가

상담 자료  ← internal_only
  · 암 상담 진행 순서
```

성격이 한 갈래면 건수가 많아도 그냥 나열합니다.
**미리 그룹을 만들어두면 각 그룹에 1건씩 들어가서 더 허전해집니다.**

---

## 7. 읽는 시간 — 지금은 표시하지 않는다

시안 v1에 있던 "읽는 데 8분"은 **기획팀장이 어림잡은 값**이었습니다.
근거 없는 숫자를 UI에 노출하지 않는다는 원칙에 걸립니다.

**자동 계산 기능이 붙기 전까지는 표시하지 않습니다.**
정확하지 않은 "8분"보다 날짜만 보여주는 편이 낫습니다.

나중에 계산할 때도 조건이 있습니다.

> 문서 전체가 아니라 **현재 사용자에게 노출되는 본문 기준**으로 계산한다.
> 설계사용 내부 보충 내용까지 포함해 공개 페이지 읽는 시간이 길게 계산되면 안 된다.

## 8. 폰트 크기 규격 (V3 가독성 표준 준수)

### 8-1. 원칙
**읽는 문장은 전부 15px 이상. 14px 미만은 캡션 전용.**

목록 화면이라고 본문보다 작게 잡으면 안 됩니다.
특히 **한 줄 설명은 목록이 아니라 본문**입니다 — 이걸 읽고 문서를 열지 말지 정하기 때문입니다.

### 8-2. 확정 크기

| 요소 | 크기 | 굵기 | 비고 |
|---|---|---|---|
| 카테고리 제목 (h1) | **27px** | 800 | V3 h1과 동일 |
| 갈고리 문장 | **16px** | 400 | 본문 |
| 문서 제목 | **19px** | 800 | |
| **문서 한 줄 설명** | **15.5px** | 400 | **본문 최소선 이상** |
| 탭 | **15px** | 600 / 800(활성) | |
| 칩 | **14px** | 700 | |
| 하단 안내 | **15px** | 400 | 읽는 문장 |

### 8-3. 캡션 전용 (14px 미만 허용)

| 요소 | 크기 |
|---|---|
| 탭·칩 뒤 건수 | 11.5px |
| "보장 영역" 라벨 | 11.5px |
| 업데이트일 | 12.5px |
| NEW / 설계사 전용 배지 | 10.5px |

**읽는 문장에는 14px 미만을 쓰지 않습니다.**

### 8-4. 본문 폭
**710px** (기존 가독성 표준). 카테고리 페이지는 긴 글이 아니라 목록이므로
710px 안에서도 답답하지 않습니다.

**정렬·좌우 여백 → 부록 참조.** (원문에 정렬 규칙이 없어 라이브가 좌측 정렬로 나온 재작업 건. 부록 G)

## 9. 화면 규격

### 9-1. 구조

```
‹ 홈으로

┌──────────────────────────────────────┐
│ 의료실비 2 │ 암 2 │ 뇌·심장 │ 수술비 │ 간병 1 │ 기타 │  ← 탭 (카테고리)
├──────────────────────────────────────┤
│ (전체 6) (변천사·구조 3) (확인 포인트 2) │      ← 칩 (그룹, 있을 때만)
└──────────────────────────────────────┘

보장 영역                                  ← kicker 11.5px
암                                        ← h1 27px
일반암이라는 말에 회사마다                    ← 갈고리 16px
다른 범위가 들어 있습니다.
자료 2건                                   ← 노출 기준 카운트 13px
══════════════════════════════════════   ← 2px 진한 선

암 분류 체계 변천사                          ← 19px 800
일반암에서 무엇이 빠져 있는가                 ← 15.5px muted
NEW · 2026-07                            ← 12.5px
──────────────────────────────────────
암주요치료비 변천사
진단비 중심에서 치료 행위 기준으로
2026-07
──────────────────────────────────────
```

### 9-2. 스타일 규칙

- **카드 박스를 쓰지 않는다.** 각 자료는 행 + 얇은 구분선. 자료가 2건이어도 허전하지 않다
- hover 시에만 배경·좌우 패딩이 생긴다
- 카테고리 헤더 아래 **2px 진한 구분선**으로 목록과 분리
- 탭 줄은 좁은 화면에서 **가로 스크롤** (스크롤바 숨김)

### 9-3. 빈 카테고리

"준비 중" 한 줄로 끝내지 않습니다.
**그 영역이 무엇을 다룰지는 설명**해서, 빈 페이지가 아니라 예고편이 되게 합니다.

```
아직 등록된 자료가 없습니다
준비되는 대로 이곳에 올라옵니다.
다른 영역의 자료는 지금 보실 수 있습니다.
```

단, 카테고리 헤더의 **갈고리 문장은 자료가 없어도 표시**합니다.
그래야 빈 화면에도 내용이 생깁니다.

## 10. 하지 말 것

| 금지 | 이유 |
|---|---|
| 카테고리 안에 또 카드 그리드 | 홈에 이미 큰 카드가 많다. 반복되면 답답해진다 |
| 탭·칩 컴포넌트 신규 제작 | 스크립트 페이지에 이미 있다. 재사용한다 |
| `visibility` 수동 입력 필드 신설 | 발행등급과 두 벌 관리가 된다 |
| 건수 임계값으로 그룹화 (예: 20건) | 근거 없는 숫자. 성격 기준이어야 한다 |
| 전체 등록 건수를 카운트로 표시 | 로그인 전 사용자에게 안 보이는 문서까지 세게 된다 |
| 읽는 시간 임의 입력 | 검증 안 된 숫자를 UI에 노출하는 것 |
| **목록이라고 본문보다 작은 폰트 사용** | **한 줄 설명은 읽는 문장이다. 15px 이상** |
| 완전한 메모장 형태 | 제목·설명·날짜·NEW는 유지해야 목록이 기능한다 |

## 11. 전체 흐름

```
홈
 └ 큰 카테고리 카드 6개
    └ 카테고리 상세 (탭·칩 + 노트형 문서 목록)
       └ 개별 지식 페이지 (독립 URL, de-iframe 완료분)
```

- 홈 = 큰 카드
- 카테고리 내부 = 탭·칩 전환 + 노트형 문서 목록
- 문서 클릭 = 독립 지식 페이지로 이동
