# 어드민 권한 모델 — "상태는 본다, 내용은 안 본다" (2026-06-14 결재)

> 결재: 대표님 + 전략실장(김실장) 합의. 총괄팀장(Code) 채택 권장.
> 성격: 영구 권한 철학. 향후 모든 권한·가시성 작업의 기준(진실 원천).
> 관련: [[role_system.md]] · `docs/migrations/2026-06-14_vault_admin_read.sql`(자료실 사고대응) · 검색 데이터 현황판(PR #674)

## 핵심 원칙

**어드민은 "전체 내용을 보는 관리자"가 아니라 "전체 상태를 보는 관리자"다.**

업계 표준(구글 워크스페이스·슬랙·노션·드롭박스 비즈니스·MS365)과 동일한 **Break Glass** 모델:
"기본적으로 사용자의 공간은 사용자의 것 / 시스템 운영·보안·법적 대응·사고 조사 시에만 관리자 열람."

## 5단계 권한

| 단계 | 대상(9 role) | 볼 수 있는 것 |
|---|---|---|
| 1 일반 사용자 | ga_member·ga_staff·insurer_member·insurer_staff | 본인 자료·본인 검색 결과만 |
| 2 팀장 | ga_manager·insurer_manager | 팀 자료·팀 공지·팀 통계 (개인 자료 X) |
| 3 지점장 | ga_branch_manager·insurer_branch_manager | 지점 자료·지점 현황·지점 통계 (개인 자료 X) |
| 4 어드민(상태) | admin | **전체 "상태"** — 사용자 수·자료 수·OCR 진행·검색 인덱싱·팀/지점별 자료 수·업로드 현황·에러 현황. **내용 아님.** |
| 5 특별 열람(Break Glass) | admin (예외) | 사고·신고·법적 대응 시에만 개별 파일 열람. **사유 입력 + 영구 로그 필수.** |

## 어드민 원칙 (체크리스트)

- ✅ 전체 현황·통계·작업 상태 조회 (현재/작업중/완료 레이아웃 포함)
- ✅ OCR 진행 현황·검색 인덱싱 현황 조회
- ❌ 사용자 개인 자료·개인 메모·개인 다이어리 **기본 열람 불가**
- ⚠️ 사고·신고(개인정보 유출/저작권/악성파일)·법적 분쟁·시스템 장애 시에만 **특별 열람**
- ⚠️ 특별 열람 = **사유 기록 + 열람 로그 영구 보관(삭제 불가)**

## 갭 트랙 — Break Glass 로깅 (착수 대기)

사고대응 열람(자료실/공지)을 누를 때:
1. **사유 입력 모달** — 사유(필수)·열람자(admin)·시각·대상.
2. `admin_access_log` 테이블 영구 기록 — **UPDATE/DELETE 차단(append-only)**, RLS로 삭제 불가.
3. (선택) 일정 기간 후 자동 알림/리뷰.

→ DB 테이블 신설 필요(대표/Chrome DDL). 박제 후 별 트랙으로 진행.

## 방향 정합

대표님이 만들고 있는 **OCR 현황판·검색화 현황판·지식엔진 현황판**이 정확히 이 모델(어드민=상태)의 구현체다. 전국 조직망으로 커질수록 "전체 내용을 보는 관리자"가 아니라 "전체 상태를 보는 관리자"가 신뢰의 핵심.
