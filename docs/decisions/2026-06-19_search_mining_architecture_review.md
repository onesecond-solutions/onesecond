**주의(충돌 아닌 정리 포인트):**
- **수요≠공급 분리**: 현재 `posts`를 `board_type`(qna/insurer/community)로 구분 = 이미 분리 충족. 별 테이블 신설은 과함 → **board_type 유지 권고**.
- **search_text 단일 컬럼 우려**: B-1로 `clean_text`(원본 보존+정제본) 추가 완료 → raw/정제본 2층은 해소. 3층(정규화 지식)은 `knowledge_entries` 별 테이블이라 컬럼 충돌 없음.
- **taxonomy**: category 묶음(상품/인수/보장)은 `knowledge_entries`에만 적용, 914 재태깅 불요(기존 확정)와 정합.
- 직전 결정(OCR B-2 원본/정제본, 선택상태)과 **모순 없음** — 한 줄로 꿰짐: *raw 보존 → 정제본(왜곡0 검수) → 정규화 지식(승인) → 검색은 정규화·파일은 근거.*

## ② 감당 가능 범위 (개인사업자 + AI팀)
- **감당 가능**: 노출/표시, 정제본 검수 게이트, 소량 직접 업로드→검증, board_type 분리. (오늘 하루 검색 7PR 처리 = 프론트·검수는 충분히 감당)
- **사람 병목**: (가)정규화 지식 검수 (나)심의필 판별 (다)시점/만료 라벨. → **반자동화로 완화**: 정제본은 B-2 자동+표본 검수, 채굴은 cron+검수 큐, 만료는 publish 날짜 기반 자동 계산.
- **외부 불가피**: 저작권/심의필 재가공권 경계(§3①) = 법률 확인 영역.

## ③ 권장 Phase 순서 (1안)
- **Phase 0 — 지금 (프론트, DB 0):** 노출 표시 마무리 = 범위 라벨 의미 안내 + 자료별 공개범위 배지. (RLS는 대표 수차 확인 완료 → 표시만)
- **Phase 1 — 검증 후:** 채굴 정규화 **검수 기준 1장 확정** + B-2 정제본 품질 검수 게이트(대기 중) → 채굴팀 재가동(🔴 정체 해소).
- **Phase 2 — 로드맵 뒤:** 시간 모델(만료/아카이브 라벨) + "소량 검증→역순 대량 배치". **단 §3① 법률 확인 선행.**
- **의존성**: 검수기준(P1) → 대량적재(P2). 노출표시(P0)는 독립 → 지금 바로 가능. 법률(①)은 병렬로 대표가 외부 확인.

## ④ 선결 조건 (대표 결재 게이트)
1. **법률 자문 여부** — 심의필=배포승인 ≠ 가공·재배포권. 사실축(수치·기준) 추출은 저작권 약하나 리플릿 통째 재배포는 위험. **외부 확인 권고**(범위 정하기 전 대량 적재 금지).
2. **정규화 검수 기준** — 수치·담보명·보험사명 불변(B-2 기준 그대로 확장). 표본 대조 검수.
3. **대량 적재 게이트** — 검수 기준 확정 전 1년치 일괄 적재 금지. owner_id 보존 + DDL/DML 대표 직접.

## §3 리스크 회신 요약
- ① 저작권: 법률 영역(위 선결1). 보수적 운영 = 사실 추출 위주, 원문 통째 재배포 회피.
- ② 정규화 왜곡: **B-2에서 이미 다룸**(정제본 검수 게이트, 숫자·담보명·보험사명 0왜곡). 기준 그대로 적용.
- ③ 대량 적재 부채: "최신 소량 직접 업로드 검증 → 역순 대량 배치" **타당**. 그대로 권고.
- ④ owner/결재: 이미 운영 규칙(DB=대표 직접, owner 보존). 배치 시 재확인.

## §6. 현장 입력 거르기 기준 — 실전 예시 (2026-06-19 박영선님 카톡)
현장에서 카톡으로 들어온 자료: **고객 실명 설계안(송정화·박미선), 상담콜 녹음(mp3/wav, 가족 얘기 포함), odt, 사진 다수.** 보낸 분 본인이 "원세컨드에 올리는 건 아닌 것 같다(가족 얘기), 실장·대표 두 분이 교육용으로만 공유"라고 자체 판단.

**→ 판정: 채굴·검색·지식엔진行 절대 금지. 보낸 분 판단이 정답.** 이유:
- 고객 실명 + 상담콜 **녹음** + 가족/사생활 = 민감 PII + 통화 녹음(동의 범위 밖). [[customer_pii_consent_based_ok]]의 "동의 기반 저장 정상"은 **본인이 수집·동의한 고객 기본정보**를 말하며, **타인 간 상담 녹음·가족 얘기·타 고객 실명**은 동의 밖 = 별개.
- 가치(실전 상담 노하우)는 있으나, **지식이 되려면 익명화 + 사실축만 추출**(고객명·가족·녹음 원본 제거 → "암주요치료비 설계 접근법" 같은 일반화). 그 전엔 원천 금지.

**원칙 정합:** 이 케이스가 곧 (가)채굴≠노출 (나)수요≠공급 (다)검수 게이트 (라)화이트리스트(심의필) (마)익명화 강제가 **왜 필요한지**의 증거. 시스템은 이런 자료를 자동으로 빨아들이지 않음(채굴 원천=소식지·게시판 화이트리스트, 자료실 OCR도 본인 스코프 RLS) → 현재 구조로 사고 안 남.
**현장 안내(권고):** 안 올린 판단 잘함. 실장-대표 개인 공유로 충분. 자산화 원하면 고객정보·녹음 빼고 **익명 사실 정리본**만.
