# [결재 문서] 지식엔진 풀가동 — 사이클 규격 v1

> **결재일:** 2026-06-06 · **상태:** 활성(active) · **유형:** 운영 규격 결재

## 정의 (한 줄)
지식 적재를 1회성 작업이 아닌 **반복 사이클**로 운영한다.
1사이클 = 300건. 검색량이 풍부해질 때까지 **연료 로드맵 순서대로 연속 가동**.

## 1사이클 표준 흐름 (관문 2개 포함)
1. **수집** — 코드: 출처 포함 수집 → `docs/data/terms_draft/` 50건×6파일 push (DB 적재 0)
2. **검증** — 웹: raw 읽기 + 외부 출처 교차 대조 + `confidence`(상/중/하) 부여
   → 검증 리포트를 GitHub 문서로 push (검증 이력 보존)
3. **눈검수** — 팀장님: confidence 기준 검수 **[관문 1]**
4. **적재** — 코드: 승인분 INSERT + 학습일지 기록 SQL 생성 → 팀장님 단독 실행 **[관문 2]**
5. **기록** — 학습일지에 사이클 번호·건수·승인율 누적 → 다음 사이클 진입

## 사이클 주기
표준 **2일** (검증 2~3세션 + 눈검수 + 적재 1회 기준). 단축은 실적 보고 누적 후 재결재.

## 관문 완화 일정 (신뢰 = 사이클 실적으로 적립)
- **사이클 1~3:** '하' 전건 + '중' 표본 눈검수
- **사이클 4~:** '하' 전건만 (직전 사이클 승인율 95% 이상 시)
- **Phase 3 야간 자동 학습 가동:** 별도 결재 건 — 본 규격의 관문 구조 검증 후 진입

## 연료 로드맵 (사이클별 적재 대상 순서)
1. **기초 용어 300** ← 오늘밤 사이클 1 가동
2. 심화 용어 + 동의어 확장
3. 인수기준 지식 (질병×보험사) — 변동성 高: 출처 URL + 기준일 필드 필수, 규격 보강 후 진입
4. 소식지·상품 지식 — 6월호 검수 관문 리허설과 합류 (데이터레이크 API 수집은 v2 대전환 후 별도)

## 불변 원칙 (사이클 무관 유지)
- 출처 없는 항목 = 검증 불가 = **적재 금지**
- **SQL 실행 = 팀장님 단일** (사이클 자동화 대상 아님)
- 검수 미승인분 = 검색 비노출 (`discarded` 제외 규칙 유지)
- 특정 보험사 편향 콘텐츠 X (중립 SaaS) / 고객 식별정보 유입 0 (§4)
- 용어·지식 = 정보 레이어 = **전 등급 무료** (PLUS 게이트 X)

## 종료·전환 조건

**"FREE 일반검색 vs PLUS AI검색(RAG) 차별화 사양 결재"**로 전환 (기존 큐 정합).
