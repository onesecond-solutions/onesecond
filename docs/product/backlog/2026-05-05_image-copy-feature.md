# 게시판 이미지 원클릭 복사 기능 — 백로그 (단계별 진화 로드맵)

## § 0. 큰 그림 정합성

본 백로그는:
- 작업 진입 X (백로그 등록만)
- 메모리 카톡 마이그레이션 트랙 §본질("4팀 옮겨오게 만들려면 평소 쓰던 자료가 게시판에 있어야") 정합 — 자료를 게시판에 올렸을 때 카톡 등 외부로 다시 공유하는 흐름 단축
- 메모리 "모바일 채팅 대응 템플릿 라이브러리" 트랙 (카톡 마이그레이션 보류 메모리에 통합 검토 시점 명시)과 함께 재논의

---

## § 1. 동기

게시판에 자료를 올린 사용자가 카톡·문자·메일 등 외부로 다시 공유하는 흐름이 잦음. 매번:
1. 게시판 글 진입
2. 이미지 우클릭 또는 길게 눌러 저장
3. 외부 앱 열고 첨부

→ 3단계가 사용자 회귀(카톡 복귀) 유인의 작은 마찰 원인.

원클릭 복사 = 게시판 글에서 즉시 클립보드 복사 → 외부 앱에 붙여넣기 1회로 공유 완료.

---

## § 2. 단계별 진화 로드맵

### v1.1 (5/15 출시 시점) — 단순 이미지 복사

**범위:**
- 게시판 글 본문 이미지 우상단 또는 호버 시 "복사" 버튼 노출
- 클릭 시 `navigator.clipboard.write([new ClipboardItem({...})])` API 사용
- 모바일은 길게 누르기 → "이미지 복사" 컨텍스트 메뉴 (브라우저 기본)

**제약:**
- 이미지만 복사 (텍스트 X)
- 복사 후 외부 앱은 사용자가 직접 열어 붙여넣기

**기대 효과:**
- 카톡·문자 공유 마찰 감소 1단계 (3단계 → 2단계)

**기술 검토:**
- `navigator.clipboard.write` 호환성: Chrome 76+ / Safari 13.1+ / Firefox 90+ — 4팀 운영 환경 정합
- HTTPS 필수 (https://onesecond.solutions 정합)
- iOS Safari clipboard image 권한 issue 검증 필요

### v1.5 — 멘트 + 이미지 묶음 복사

**범위:**
- 게시판 글에 "공유 멘트" 필드 추가 (선택, 작성자 정의)
- 복사 버튼 클릭 시 **멘트 텍스트 + 이미지 묶음** 클립보드 저장
- 외부 앱 붙여넣기 시 멘트가 캡션·본문, 이미지가 첨부로 자동 분리 (앱마다 다름)

**제약:**
- ClipboardItem에 text/plain + image/png 동시 저장
- 카톡·라인 등 일부 앱은 텍스트와 이미지를 분리 처리 (한 메시지에 동시 X)

**기대 효과:**
- 자료 + 작성자 의도 멘트가 함께 공유 → 외부에서도 원세컨드 자료 노출 (브랜드)

**기술 검토:**
- 멘트는 글 작성 시 인라인 입력 필드 또는 자동 추출(첫 줄)
- 4팀 → 1,000명 시점 모니터링 후 진입 결정

### v2.0 — 채팅 템플릿 통합

**범위:**
- 메모리 "모바일 채팅 대응 템플릿 라이브러리" 트랙과 통합
- 자주 쓰는 멘트 템플릿 라이브러리 (예: "이번 주 자료입니다", "고객님 상담 시 활용 부탁드립니다")
- 게시판 자료 → 템플릿 적용 → 멘트 + 이미지 묶음 복사
- A1 영역에서도 빠른 진입 (멘트 + 자료 단일 클릭)

**제약:**
- 템플릿 라이브러리 자체가 큰 트랙 (별도 작업지시서 필요)
- 카톡 마이그레이션 트랙 의논 결과에 따라 우선순위 변동

**기대 효과:**
- 카톡 회귀 위험 감소 핵심 기능 (메모리 §본질 정합)
- 자료 + 멘트 + 채널 = 외부 공유 1회 클릭 완성
