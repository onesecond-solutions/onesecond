## 0. 현재 실태 조사 (근거)

기준 수립 전에 라이브 코드를 실측한 결과다. 결론부터: **색 토큰은 잘 정착했고, 타이포·radius·단위가 혼재 상태다.**

### 0-1. 토큰 소스가 3곳으로 갈라져 있다

| 소스 | 내용 | app.html 연결 여부 |
|---|---|---|
| `css/tokens.css` (576줄) | 구세대 테라코타 브라운(#A0522D) 브랜드 + neutral/brand 10단계 스케일 + elevation 5단계 + spacing/radius/leading 풀 세트 | **미연결** (pages/_template.html·admin_v2.html만 링크) |
| `css/tokens-v2.css` | 인디고(#6366F1) 라이트/다크 2모드 + 타이포 미러 | **미연결** (미러 전용, landing 계열만) |
| **app.html 인라인 `<style>` `:root` 7블록** | 실효 토큰. 색·스페이스색·카테고리10색·radius·타이포 3세대 | **이것이 라이브 진실** |

즉 "토큰의 단일 소스"가 없다. app.html 인라인이 사실상 진실 원천인데, 파일로 분리돼 있지 않고, tokens.css(브라운)·tokens-v2.css(미러)와 값이 서로 다르다. 예: radius-sm이 tokens.css=8px / tokens-v2.css=9px / app.html=11px.

### 0-2. 색 — 절반 성공

- 라이브 기본 팔레트(약칭 토큰)는 광범위하게 정착: `var(--ac)` 620회, `var(--bd)` 503회, `var(--ts)` 362회, `var(--tp)` 289회 등. **색의 토큰화는 이미 문화로 자리잡았다.**
- 그러나 하드코딩 잔존: `#6366F1`(=--ac와 동일값) 26회 직접 기입, 성공 계열이 `#0E9A6C`(21) / `#16A34A`(10) / `#34D399`(13) / `#059669`(5)로 4갈래, 위험 계열이 `#EF4444`(21) / `#DC2626`(12) 2갈래로 갈라져 있음.
- 인라인 `style=` 속성 493개 (색·간격 하드코딩 다수 포함).

### 0-3. 타이포 — 가장 심한 혼재

app.html 안에 **타이포 토큰 세트가 3세대 공존**한다.

| 세대 | 토큰 | 값 | 도입 |
|---|---|---|---|
| 1차 | `--fs-xs`~`--fs-xl` | 0.8125rem~1.375rem (rem) | 2026-06-13 |
| 2차 | `--fs-caption`~`--fs-heading` | 12~20px, **본문 14px** | 2026-06-27 "구글급 절제형" |
| 3차 | `--ts-badge`~`--ts-viewer-title` | 12~20px, **본문 16px** | 2026-07-04 "최소 12px" |

여기에 토큰을 안 거친 직접 지정이 대량: `font-size` 선언 약 700건 이상이 rem(0.75/0.76/0.78/0.8/0.8125/0.82/0.85/0.86/0.88/0.9/0.92/0.95/1.05rem…)과 px(12/13/14/15/17/18px…)로 뒤섞임. 사실상 임의값 수십 종.

기술 함정 하나: 폰트 크기 컨트롤러(`[data-fontsize]` → `--fscale` → `html{font-size:calc(100%*var(--fscale))}`)는 **rem 값만 확대**한다. px로 박힌 2·3차 토큰과 px 직접 지정은 컨트롤러가 커져도 안 커진다. 화면 안에서 어떤 글자는 커지고 어떤 글자는 안 커지는 원인.

### 0-4. radius·간격

- radius 토큰은 정의돼 있으나(`--radius-xs:8 ~ --radius-lg:18`) 직접 px 지정이 훨씬 많음: `border-radius:8px` 113회, 9px 64회, 10px 61회, 12px 54회, 6px 43회, 7px 29회… (5~18px 사이 임의값 10종+).
- 직각(0)은 사실상 없음 — "직각 금지" 원칙은 지켜지고 있음.
- hit target: `min-height:44px` 명시는 극소수. 고정 높이 20~39px 요소 61건.

### 0-5. 대표 패턴 (현재 잘 쓰이는 골격 — 버릴 게 아니라 표준으로 승격할 것)

- **목록 행 `.item`**: `padding:12px 14px 12px 16px; border-bottom:0.5px solid var(--bd); border-left:3px solid var(--tc)`(카테고리색 왼줄) + hover `var(--s2)` — Linear식 행 목록이 이미 있음.
- **칩 `.chip`**: `padding:5px 12px; border-radius:7px; background:var(--s1); border:0.5px solid var(--bd)` — 필터 칩 표준 후보.
- **모달 카드 `.cardbox`**: `var(--s1)` + `0.5px var(--bd)` + `radius 16px` + `padding 20px`.
- 공통 규칙으로 이미 굳은 것: **순백 표면(--s1) + 0.5px 얇은 테두리(--bd)로 구분, 그림자 최소** — 이 절제 톤이 원세컨드 룩의 뼈대다.

## 1. 전체 디자인 방향

### 정체성 → 지향 룩

| 정체성 | 화면에서의 의미 |
|---|---|
| 중립 독립 SaaS | 어떤 보험사 색도 시스템색이 아니다. 브랜드 인디고 + 무채색 중심. 보험사·상품 구분색은 "데이터 색"(--cat-*)으로만, 시스템 UI에는 침투 금지 |
| 절제 > 화려 | 색은 신호에만. 표면은 순백, 구분은 0.5px 테두리, 그림자는 떠 있는 것(모달·팝오버)에만. 그라데이션·글로우 남발 금지 |
| 딸깍 (one click) | 한 화면 = 한 목적. 주요 행동 버튼은 화면당 1개가 원칙(primary 남발 금지). 결과까지의 단계 수가 디자인 품질 지표 |
| 위계 = 굵기·색 | 크기 단계는 대/중/소 3단으로 제한. 그 안에서 굵기(400/500/600)와 색(주/보조/희미)으로 층위를 만든다. bold 남발 금지 |
| 가독·조작 스펙 | 본문 16px 이상, 클릭·터치 대상 44px 이상 (기능 스펙) |

### 하지 말 것 (금지 목록)

1. 색·radius·크기 하드코딩 — 토큰만. (예외: 도메인 특화 stage 팔레트, 포스트잇 파스텔 등 명시된 데이터 색)
2. 이모지·원문자 UI 사용 금지 — 아이콘은 라인 아이콘(Tabler류)·SVG만.
3. 직각 모서리 금지 — 모든 컴포넌트 radius-sm 이상.
4. 특정 보험사 편향 연출 금지 — 로고 크기·순서·색 강조로 우선순위를 암시하지 않는다.
5. 크기 단계 신설 금지 — 정해진 타이포 스케일 밖의 font-size 임의값 추가 금지.
6. 12px 미만 텍스트 금지 (이미 2026-07-04 결정 — 유지).
7. 한 화면에 primary 버튼 2개 이상 금지.
8. 위계를 만들려고 폰트를 키우는 습관 금지 — 먼저 굵기·색, 그래도 부족하면 여백.

## 2. 공통 스타일 기준 (컴포넌트별)

값은 4항의 신규 토큰 이름으로 표기. (환산 px 병기)

### 2-1. 카드

| 항목 | 기준 |
|---|---|
| 표면 | `--surface`(순백) + `1px --border`(현행 0.5px는 저해상도에서 소실 — 1px 통일 권장) |
| radius | `--radius-md`(12px). 모달·큰 패널만 `--radius-lg`(16px) |
| padding | `--space-5`(20px). 밀도 높은 목록 카드 `--space-4`(16px) |
| 그림자 | 기본 없음(테두리로 구분). hover 시 `--shadow-sm`만. 떠 있는 카드(모달·팝오버)만 `--shadow-lg` |
| 제목 | `--text-title-sm`(16px) / `--fw-semibold`(600) / `--text-primary` |
| 메타 | `--text-meta`(13px) / `--fw-regular` / `--text-faint` |
| hover | 배경 `--surface-2` 또는 테두리 `--border-strong`. transform·글로우 금지 |
| 빈 상태 | 아이콘(라인) + 한 줄 안내 + (가능하면) 딸깍 1버튼. 회색 텍스트만 덜렁 두지 않기 |

### 2-2. 표 (비교표·데이터)

| 항목 | 기준 |
|---|---|
| 헤더 행 | 배경 `--surface-2`, `--text-meta`(13px) / `--fw-medium`(500) / `--text-secondary`, 대문자·자간 연출 금지 |
| 본문 행 | `--text-body`(16px), 행 높이 44px 이상, 구분 `1px --border` 가로선만(세로선 없음 — Stripe식) |
| 숫자 열 | 우측 정렬 + `font-variant-numeric: tabular-nums`(자릿수 정렬) |
| hover | 행 전체 `--surface-2` |
| 상태 강조 | 셀 배경색 금지 — 배지 또는 텍스트 색으로만 (중립성: 표에서 특정 사 행 배경 강조 금지) |
| 스크롤 | 넓은 표는 컨테이너 `overflow-x:auto`, 첫 열 sticky 허용 |

### 2-3. 배지

| 항목 | 기준 |
|---|---|
| 크기 | `--text-badge`(12px) / `--fw-medium`(500) / padding 2px 8px / `--radius-full` 또는 `--radius-xs` — pill·사각 중 화면당 하나로 통일 |
| 색 공식 | 의미색 soft 배경 + 진한 텍스트 (예: `--success-soft` 배경 + `--success-strong` 텍스트). 원색 배경+흰 글자는 count·alert 뱃지만 |
| 종류 제한 | 의미 4종(success/warning/danger/info) + 중립 1종 + 카테고리(--cat-*) — 그 외 색 배지 신설 금지 |
| 텍스트 | 명사 1~2어절. 문장 금지 |

### 2-4. 버튼

| 항목 | 기준 |
|---|---|
| 높이 | 기본 40px·`min-height:44px`는 터치 맥락(모바일·바텀시트), 최소 36px(밀집 툴바 한정) |
| radius | `--radius-sm`(8~10px) |
| 타이포 | `--text-body-sm`(14px)~`--text-body`(16px) / `--fw-semibold`(600) |
| primary | `--accent` 배경 + 흰 글자. **한 화면 1개 원칙** |
| secondary | `--surface` + `1px --border` + `--text-primary` |
| ghost | 투명 + `--text-secondary`, hover 시 `--surface-2` |
| danger | 파괴 행동만. confirm 없는 danger 금지 |
| hover | primary=`--accent-hover`, 나머지=배경 한 단계. active=명도 한 단계 아래. transform 튀는 효과 금지 |
| disabled | opacity 아닌 색 교체(`--surface-2`+`--text-faint`) — 반투명 겹침 방지 |
| 로딩 | 버튼 안 스피너 교체, 폭 유지(레이아웃 점프 금지) |

### 2-5. 검색 결과 (목록 행)

현행 `.item` 패턴을 표준으로 승격.

| 항목 | 기준 |
|---|---|
| 구조 | 행 목록(카드 그리드 아님). 구분 `1px --border` 가로선 |
| 왼쪽 신호 | `border-left:3px` 카테고리색(--cat-* / --space-*) — 색은 이 줄과 배지에만, 행 배경 침투 금지 |
| 제목 | `--text-list-title`(15px) / 600 / `--text-primary`, 검색어 하이라이트는 `--accent-soft` 배경(형광 노랑 금지) |
| 미리보기 | `--text-preview`(14px) / `--text-faint` / 2줄 말줄임 |
| 메타 | `--text-meta`(13px) / `--text-faint`, 출처·날짜·작성자 순 |
| 행 높이 | 내용 기준 자동, 최소 44px, hover `--surface-2` |
| 클릭 | **행 전체가 클릭 영역** (제목만 링크 금지 — 딸깍) |
| 빈 결과 | "결과 없음" + 동의어·필터 완화 제안 1줄 + 검색어 유지 |

### 2-6. 상세 화면 (문서·자료 뷰어)

| 항목 | 기준 |
|---|---|
| 본문 폭 | 최대 720~760px 중앙(문서 성격) — 표·비교 화면은 예외로 전폭 |
| 제목 | `--text-heading`(20px)~상세 타이틀 / 600~700 |
| 본문 | `--text-body`(16px) / 400 / `--leading-relaxed`(1.7) |
| 메타 블록 | 제목 아래 한 줄(작성자·날짜·출처 배지), `--text-meta` |
| 섹션 구분 | 여백(`--space-8`) 우선, 필요 시 `1px --border` 가로선. 박스 안에 박스 중첩 금지 |
| 행동 버튼 | 상단 우측 고정 슬롯 1곳(복사·저장·공유 = 딸깍 허브), 본문 중간 버튼 산재 금지 |

## 3. 계열 매핑 — 무엇을 어디서 흡수하나 (통째 복제 아님)

| 계열 | 흡수할 특징 | 원세컨드 적용 화면 | 흡수하지 않을 것 |
|---|---|---|---|
| **Stripe** (표·배지) | 가로선만 있는 표, tabular-nums 숫자 정렬, soft 배경+진한 글자 배지, 13px 무채색 헤더 | 상품 비교표·라인업·담보표·요율 화면, 상태 배지 전반 | Stripe 보라 그라데이션 히어로, 마케팅 페이지 연출 |
| **Linear** (목록) | 행 목록 밀도, 왼쪽 3px 신호줄, hover 한 톤, 키보드 이동 감각, 절제된 무채색 | 검색 결과·Q&A 목록·업무노트·공지 목록 — 현행 `.item`이 이미 이 계열, 다듬어 표준화 | Linear의 다크 기본(우리는 라이트 단일 결정 유지) |
| **Apple** (홈·검색) | 큰 여백, 요소 수 최소, 검색창이 주인공인 첫 화면, 한 화면 한 메시지 | 홈(바닥=검색) 첫 화면, 통합검색 진입부 | SF 폰트·유리(blur) 과다 — blur는 오버레이 1곳 한정 |
| **Notion** (문서·상세) | 본문 폭 제한 중앙 정렬, 여백으로 섹션 구분, 조용한 메타 라인, 인라인 최소 장식 | 자료 상세·소식지 뷰어·스크립트 상세·알릴의무 문서 | Notion 이모지 아이콘 문화(이모지 금지 원칙과 충돌), 블록 핸들 UI |

공통 원리: 네 계열 모두 **"색은 신호, 구조는 무채색, 위계는 굵기·여백"** — 원세컨드 절제 정체성과 같은 방향. 화면 성격(데이터/목록/진입/문서)에 따라 참조 계열을 나눠 갖는다.

## 4. 디자인 토큰 정리안 — 혼재 → 단일 소스

### 4-1. 원칙

1. **단일 소스 1파일**: `css/tokens-v3.css` (신설) — app.html 인라인 7블록·tokens.css·tokens-v2.css의 실효값을 여기로 승격. 이행 완료 전까지 app.html 인라인은 그대로 두고(회귀 0), 신규 화면부터 v3 이름 사용.
2. **이름 정책**: 현행 약칭(--ac/--tp/--bd…)은 사용량 2,000회+ — 폐기하지 않고 **v3 토큰의 별칭으로 유지**(`--ac: var(--accent)`). 강제 개명 대공사는 하지 않는다.
3. **구세대 정리**: tokens.css의 브라운(#A0522D) 계열은 admin_v2 rail 외 신규 사용 금지(사양 동결). tokens-v2.css의 다크 모드 블록은 "라이트 단일" 결정(2026-06-13)과 충돌 — 보류 딱지 명기.

### 4-2. 색

| 그룹 | 토큰 (제안) | 값 | 비고 |
|---|---|---|---|

### 4-3. 타이포 (3세대 → 1세대)

크기 대/중/소 3단 + 보조 3단. **값은 rem**(폰트 크기 컨트롤러 `--fscale` 연동 — px 토큰은 컨트롤러가 못 키우는 현행 결함 해소), 이름은 역할.

| 토큰 | 값 | 환산 | 역할 | 굵기 짝 |
|---|---|---|---|---|
| `--text-heading` | 1.25rem | 20px | 대 — 페이지·뷰어 제목 | 600~700 |
| `--text-title` | 1.125rem | 18px | 중 — 섹션 제목 | 600 |
| `--text-title-sm` | 1rem | 16px | 중 — 카드 제목 | 600 |
| `--text-body` | 1rem | 16px | 중 — 본문 (기준) | 400 |
| `--text-list-title` | 0.9375rem | 15px | 소 — 목록 행 제목 | 600 |
| `--text-preview` | 0.875rem | 14px | 소 — 미리보기·보조 본문 | 400 |
| `--text-meta` | 0.8125rem | 13px | 소 — 메타·라벨 | 400~500 |
| `--text-badge` | 0.75rem | 12px | 소 — 배지·캡션 (최소값) | 500 |

- 굵기: `--fw-regular:400 / --fw-medium:500 / --fw-semibold:600 / --fw-bold:700` — 700 초과(800·900) 사용 금지(로고 예외).
- 행간: `--leading-tight:1.3`(제목) / `--leading-base:1.5`(UI) / `--leading-relaxed:1.7`(문서 본문).
- 폐기 예정 별칭: `--fs-*`(rem 1차), `--fs-caption~heading`(px 2차), `--ts-*`(px 3차) → v3 이름으로 별칭 매핑 후 신규 사용 금지.
- 12px 미만 금지 유지. 임의 rem(0.78, 0.82, 0.86…)은 위 8단으로 스냅.

### 4-4. spacing

4px 그리드 유지: `--space-1:4 / 2:8 / 3:12 / 4:16 / 5:20 / 6:24 / 8:32 / 10:40 / 12:48` (tokens.css 체계 승계, 소수 단계는 흡수하지 않음).

### 4-5. radius

app.html 현행(2026-06 "B안 넉넉히") 값을 표준으로: `--radius-xs:8 / --radius-sm:10 / --radius-md:12 / --radius-lg:16 / --radius-full:9999`. 임의값(5·6·7·9·11·13·14·18px)은 근접 단계로 스냅. 최소 = `--radius-xs`(직각 금지).

### 4-6. 그림자

3단으로 축소: `--shadow-sm`(hover 미세) / `--shadow-md`(드롭다운·팝오버) / `--shadow-lg`(모달·시트). elevation 5단·warm·glow(tokens.css)는 admin 외 동결. 원칙: **그림자는 "떠 있는 것"에만.**

## 7. 리스크·주의

1. **토큰 3소스 혼재가 최대 회귀 원인** — tokens.css(브라운)와 app.html(인디고)의 같은 이름·다른 값(`--radius-sm` 8 vs 11 등)이 이미 존재. pages/*.html 중 tokens.css를 링크한 파일이 app.html 안에 임베드되는 경로가 생기면 값 충돌. v3 확정 전 신규 토큰 이름 추가 동결 권장.
2. **px 타이포 토큰 = 폰트 크기 컨트롤러 무력화** — 2026-07-04 `--ts-*`(px)를 쓴 텍스트는 확대 설정이 안 먹는다. v3에서 rem으로 재정의하지 않으면 "일부만 커지는" 결함이 계속 늘어난다.
3. **하드코딩 관성** — #6366F1 직접 기입 26건처럼 토큰이 있어도 값이 복제되는 중. 브랜드색을 바꾸는 날(열린 선택 1) 이 26건은 안 바뀌는 지뢰가 된다.
4. **지금이 개편 적기** — 사용자가 적은 현 시점이 구조 개편 골든타임(기존 결정 정합). 사용자·화면이 늘수록 임의값 700건이 1,500건이 되고 영구 짐이 된다. 단, 라이브 사용자 0은 아니므로 "화면 단위 PR + 검수" 원칙 유지.
5. **절제 원칙의 자기 파괴 경계** — 통일 작업 중 "이왕 하는 김에" 재설계 확장 금지(작은 요청=작은 변경 원칙). 토큰 치환 PR에 레이아웃 변경 섞지 않는다.
6. **admin_v2 스코프 격리 유지** — `--admin-*` 토큰·다크 4톤은 본 시스템 밖. v3 이행 대상에 넣지 않는다.
7. **이모지·원문자 잔존 소지** — 기존 화면에 남은 emoji 아이콘은 토큰 이행과 별개 트랙으로 Tabler 라인 아이콘 치환 목록화.
