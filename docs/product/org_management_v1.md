# 조직 관리 시스템 v1

> **배경:** 처음부터 조직 시스템을 설계하고 시작하지 못하고, 진행하다 여기까지 옴.
> 지금 전체를 새로 잡아가는 재설계 트랙. (팀장님 = 비개발자, 화면 보며 조율 선호)

## 1. 한 줄 본질
원세컨드는 **지점 > 팀 > 사람**의 조직 네트워크 위에서 돌아간다. 사람은 **초대(상·하향)**로 묶이고, **가입하면서 팀으로 매핑**된다. (master_strategy §1 "운영 흐름 네트워크"의 뼈대)

## 2. 위계 (확정)

```
원세컨드
  └ 지점(branch) ── 지점장(ga_branch_manager) = 지점 대표, 팀 없음
       └ 팀(team) N개
            ├ 실장/팀장(ga_manager) 1~여러 명   ← 한 팀에 실장 여럿 정상(큰 팀)
            └ 팀원(ga_member)
            └ (스텝 ga_staff)
```

- 전국 비전: 원세컨드 아래 여러 지점(A·B·C…n), 각 지점 아래 실 1~N (onesecond_vision.png)
- 연결 키: `users.branch_id`(지점) · `users.team_id`(팀)

## 3. 현재 데이터 (AZ금융 1개 지점, 2026-06-03 실측)

```
김종무 지점장 (branch 306edf6a)
├ 팀 5fccd362: 실장 임태성·한재성 + 팀원 ~27
├ 팀 741bce8a: 실장 없음 + 팀원 1 (변영삼)
├ 팀 95088922: 실장 없음 + 팀원 6
└ 소속 미배정: 4명 (team/branch null)
```

⚠️ 팀명 테이블 유무 미확인 — A단계 진입 시 teams 테이블 존재 여부 확인 필요.

## 4. 화면 — 매니저룸 > 팀원관리

하위 탭 2개: **실장 | 지점장** (관점 전환, admin은 둘 다)

### 지점장 관점
```
[지점장 카드]  ← 좌측 list-col 너비(330px), split 위 단독  ✅(완료)
─────────────────────────────────────────
[좌: N팀 목록]            [우: 선택 팀의 팀원]
 · 팀 카드들              · 팀원 카드 (세로)
 · [+ 실장 추가(초대)]      · 접기/펼치기 기능 ✏️
                          · [+ 팀장 추가(초대)]
```

### 실장 관점
- 본인 팀 중심 (내 팀의 실장·팀원)

## 5. 초대 시스템 (위계별 · 양방향)

**기반:** `team_invitations` 테이블 이미 존재 (token·만료·status·team/branch, `signup?invite={token}` 가입흐름). 재사용.

| 초대 주체 | 초대 대상 | 위치 |
|---|---|---|
| 지점장 | 실장 초대 / 팀 생성 | 좌측 팀 영역 `[+ 실장 추가]` |
| 실장 | 지점장 초대 / 팀장 초대 | 우측 팀원 영역 `[+ 팀장 추가]` |

→ 상향·하향 모두 가능이 기본 베이스.

## 6. 가입 매핑

- 4팀 방식 = **초대 링크/코드로 가입 → 자동 팀 매핑** (`signup?invite=token` → team_invitations → users.team_id/branch_id 세팅)
- team_invitations 인프라 위에 구축.
