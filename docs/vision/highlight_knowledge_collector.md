# [비전·백로그] 형광펜 = 개인 지식 수집기

> 발신: 기획실장(Web) / 출처: 전략실장(김실장) 대화 정리 + 기획실장 가드 보강 / 저장: 총괄팀장(Code) / 2026-06-10
> 상태: **비전·백로그** (착수 아님). 착수 조건 = 자료함·미리보기(1·2단계) 라이브 완료 후 3단계 진입.

## 본질
- 형광펜 = UI 기능 아님. **"자료를 읽으며 개인 지식을 축적하는 시스템"**.
- 자료 = 회사 것 / 형광펜 = 사용자의 지식. **이 구분이 핵심.**
- 검색 중심("보험판 구글")과 직결: 공식 데이터 + 사용자가 친 형광펜 = 사용자가 직접 큐레이션한 검색 신호. → MY SPACE(라이브러리)가 **"나만의 보험 두뇌"**로 전환되는 분기점.

## 저장 방식 (오버레이, 원본 불변)
- 원본 PDF·파일 수정 금지. **별도 테이블에 좌표 오버레이.**
- `library_highlights`: id / user_id / file_id / page / rect(x,y,width,height) / selected_text / color / memo / created_at
- 같은 사용자가 같은 파일 열면 형광펜 재현, 타인은 안 보임(**RLS owner 기준**).

## 6단계 로드맵 (한 번에 가지 말 것 · 단계별 분리 PR)
1. 자료함 (현재 라이브)
2. 미리보기 (현재 라이브)
3. **형광펜(저장형)** ← 1·2 완료 후 착수
4. 형광펜 검색 (공식 데이터 + 내 형광펜 동시 노출)
5. 딸깍 연동 (형광 영역 → 요약·카드·스크립트 생성·MY SPACE 저장)
6. 개인 지식 DB
- 3단계부터 차례로. 각 단계 진입 시 별도 작업지시서.

## §4 — 신규 침투 경로 (반드시 관리)
- `selected_text`가 서버에 저장됨 → 고객 청약서·녹취 등에 형광펜 치면 **고객 식별정보가 library_highlights로 유입**.
- 자료함 본문보다 미묘: 사용자는 "강조"로 인식, "저장"으로 인식 안 함.
- 형광펜 설계에 **§4 고지·정책 필수 동반**. 유료화·PRO 격리 전환과 맞물려 검토.

## 발전 항목
- 형광펜 + 메모(영역 클릭 시 메모 표시).
- 형광펜 모음 자동 정리(MY SPACE > 형광펜 > 보험/실손/암/뇌혈관…).

## 착수 조건
- 자료함·미리보기(1·2단계) **라이브 완료**(2026-06-10 충족) → 3단계 진입 시점에 형광펜 작업지시서 작성.
- DB(`library_highlights` 신설) = 결재 관문(DDL, 대표 직접 실행).
