## 2. 브랜드 정체성 (0순위, 모든 결정의 기준) ⭐⭐⭐

### ✅ 중립 독립 SaaS

**원세컨드는 어느 보험회사·GA(대리점)에도 소속되지 않은 독립 SaaS이며, 모든 사업 결정은 특정 회사 이해관계가 아닌 "현장 전체 기준"의 중립 위치에서 내린다.**

- ❌ 특정 보험사 기능 중심 사고 (NH 전용 / 삼성생명 편향 / 메리츠 단독 등)
- ❌ 특정 GA 편향 UX
- ❌ 폐쇄형 조직 SaaS 해석
- ❌ 4팀(AZ 더원) = 영구 단독 고객 가정

### 왜 독립성이 중요한가
- 보험 현장 = 다양한 보험사·GA 이해관계 충돌하는 영역
- 특정 회사 종속 시 → 그 회사 영업 도구로 전락 → 다른 회사 설계사 거부
- 원세컨드 = 모든 보험 현장이 사용 가능한 **운영 OS**

### 4팀 ↔ 원세컨드 관계 (오해 방지)
- 4팀(AZ 더원 GA의 한 팀) = 5/15 첫 사용자 검증
- 영구 단독 고객 X / AZ 종속 X / GA 편향 X
- 향후 다른 GA / 다른 보험사 매니저 입점 시 동일 OS

### AI 판단 기준
- "이 기능이 특정 보험사 편향인가?" 검증
- "다른 GA가 사용해도 자연스러운가?" 검증
- 폐쇄형 조직 SaaS 패턴 발견 시 즉시 보고

## 2-1. 핵심 운영 철학 4종 (절대 흔들리지 않을 것) ⭐⭐⭐

원세컨드는 위 브랜드 정체성(중립 독립 SaaS) 위에 다음 4 운영 철학으로 정의된다. 어떤 작업이든 이 철학을 회귀시키면 큰 사고:

### ✅ 현장의 소리 = 질문 운영 시스템
- ❌ 커뮤니티형 게시판 / CRM·SaaS / 일반 게시판
- 설계사가 현장에서 질문하고 운영자가 답하는 시스템
- 게시판 형태이지만 본질은 "질문 답변 운영"

### ✅ home_v2 = 운영형 홈
- ❌ 랜딩페이지 / 마케팅 페이지
- 4팀 가입자가 들어오는 메인 진입로
- Hero + 카드 그리드 + 통합 인증 모달
- 5/15 후 root `index.html`로 이전 예정

### ✅ Quick 메뉴 = 긴급 실행도구
- ❌ 일반 메뉴 / 부가 기능
- 설계사가 통화 중 멈춰서는 순간 1초 안에 실행하는 도구
- 1턴 골격 + 4그룹 + 검색

### ✅ 고정 프레임 구조 = 앱 셸
- ❌ 다중 페이지 / iframe·include 동적 frame
- `app.html` 단일 셸 + `pages/*` 동적 메뉴 로드
- `window.AppState` / `window.db` / `window.Auth` 통신
- 사이드바·탭바·검색은 셸이 담당
