### 5-4. 단일 데이터 + 가시성 분기 (RLS) — 복제 ❌ ⭐

> **본질:** 같은 `posts` row 1개를, RLS 정책이 **누가 보느냐에 따라** 다른 게시판에서 노출시킨다.
> **복제 ❌ / dual-write ❌.**

예시: 보험사 임직원이 `board_type='insurer'` + `insurer_id=NH`로 INSERT 1회.
- NH 직원 본인 → "보험사 게시판" 탭에서 봄
- NH 매핑된 지점 매니저 → "보험사 게시판" 탭에서 봄 (RLS OR 분기)
- NH 미매핑 지점 → 안 보임
- admin 승격 시 → `is_promoted_to_hub = true` UPDATE → 모든 사용자에게 "허브" 탭에서 노출

`docs/decisions/2026-05-08_*` 또는 spec v2 § 6 RLS 골격 참조.

### 5-5. 운영 단위 = 지점 / 실장님 공지 = 팀 단위 격리

> **흐름은 지점 단위로 닫는다.** 다른 지점 = 격리. 격리된 지점들은 **허브로만 연결**됨 (선별 승격).
>
> **실장님 공지는 더 좁게** = 팀 단위 격리. 같은 지점 다른 팀 매니저도 못 봄.

1. **`app.html` / `app.js` 는 명시 요청 없이 수정 금지**

3. **`defer` 속성 금지** — 표준 부트 패턴은 `appstate:ready` 이벤트

5. **직각(0px, 2px) border-radius 금지** — 최소 `--radius-sm` (8px) 이상
6. **폰트 크기는 `em` 단위** — `px` 고정 금지
7. **E영역 콘텐츠 파일만 납품** — 프레임 파일은 건드리지 않음
8. `quick.html`은 IIFE 래핑 예외
