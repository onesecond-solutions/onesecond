```
도입 → 필요성 → 상황확인 → 보장분석 → 상품설명 → 클로징 → 반론대응
```

1. 📞 도입 인사 (opening)

5. 📊 보장 분석 (analysis)
6. 📦 상품 설명 (product)
7. 🔥 필요성 강조 2 (need_emphasis_2)
8. ✅ 클로징 (closing)
9. ⚡ 반론 대응 (objection)
10. 🔵 2차 클로징 (closing_second)

### 6-7. 코드 규칙 (위반 시 동작 불가)

00_MASTER.md § 3-3 (라인 107~115):

1. **`app.html` / `app.js` 는 명시 요청 없이 수정 금지**
2. child pages의 모든 함수는 **`window.functionName` 전역 등록 필수** (IIFE 래핑 때문)
3. **`defer` 속성 금지** — 표준 부트 패턴은 `appstate:ready` 이벤트
4. **CSS는 `tokens.css` custom property만 사용** — 하드코딩 색상·radius 금지
5. **직각(0px, 2px) border-radius 금지** — 최소 `--radius-sm` (8px) 이상
6. **폰트 크기는 `em` 단위** — `px` 고정 금지
7. **E영역 콘텐츠 파일만 납품** — 프레임 파일은 건드리지 않음
8. `quick.html`은 IIFE 래핑 예외
