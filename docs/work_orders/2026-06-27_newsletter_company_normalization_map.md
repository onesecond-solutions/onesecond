# 소식지 company 정규화 매핑안 (읽기 전용 초안 — 대표 승인 전 DB 무변경)

> 작성 2026-06-27 / 총괄팀장(Code). **원본 `newsletters.company`는 절대 UPDATE 금지·raw 보존.** 본 문서는 매핑 초안이며, 승인 전 어떤 DDL/DML도 실행하지 않는다.

## 0. 권장 구조 — 매핑 테이블 + 조회 뷰 (3안 중 가장 안전)

| 안 | 원본 영향 | 평가 |
|---|---|---|
| **A. 별도 `insurer_company_aliases` 테이블 + 조회 뷰** ✅추천 | **0 (company 불변=raw 자동 보존)** | 코드(소식지 표시 company 참조 3곳) 영향 0, 롤백=테이블 DROP, noise 유입 시 alias 행 추가만(멱등) |
| B. `normalized_company` 컬럼 추가 | 원본 테이블 ALTER + 재적재마다 채움 | 더 침습·재적재 시 누락 위험 |
| C. 정규화 뷰만(매핑 인라인 CASE) | 0 | 매핑이 뷰 SQL에 박혀 유지보수 어려움 |

**추천 = A.** 원본 `newsletters.company`가 곧 raw 추적값(별도 `raw_company` 컬럼 불필요). 정식 회사명은 alias 테이블에서 LEFT JOIN.

```sql
-- (승인 후 실행 예정 — 지금은 제안만)
CREATE TABLE public.insurer_company_aliases (
  raw_value      text PRIMARY KEY,   -- newsletters.company 원본값(raw)
  canonical_name text NOT NULL,      -- 정식 회사명
  map_type       text,               -- exact | alias_typo | from_filename | manual
  confidence     text,               -- high | mid | low
  needs_review   boolean DEFAULT false
);
CREATE VIEW public.v_newsletters_company AS
  SELECT n.*, COALESCE(a.canonical_name, n.company) AS normalized_company
  FROM public.newsletters n
  LEFT JOIN public.insurer_company_aliases a ON a.raw_value = n.company;
```

## 3. 기존 260 씨드 영향
- **표기 정렬 필요**: 씨드 표기가 `한화손보`/`하나손해보험`처럼 손보·손해보험 혼용. canonical을 "손해보험"으로 통일 시 씨드 8사(한화손보·롯데손보·DB손보·KB손보·AIG손보 등)도 동일 표기로 정렬 권장. **→ 표기 통일 정책은 대표 결재 항목.**
- **0건 3사 메워짐**: 씨드의 미래에셋생명·푸본현대생명·DB손보는 소식지가 각 20·13·18건씩 본문까지 존재 → 추출 실패였을 뿐, 정규화로 모수 확보돼 보강 가능.
- **신규 발견 회사**: IBK연금보험(7)·처브라이프생명(9)·AIA생명(1) 등은 씨드 29사 외 → 라인업 표 확인 시 씨드 확장 후보.
