# 첫 공개(검색엔진 등록 + 더원지점 실사용) 준비 체크리스트

> **성격:** 대표 방향 = "설명전 → 사용전". 외부 설명 중단 → 제품 완성도 정리 → 검색엔진 등록 → 내부 실사용 전환 → 반응 데이터 수집.
> **한 문장:** 공개 순간 원세컨드가 할 일 = 사용자가 검색했을 때 **"어? 진짜 되네?"**를 보여주는 것.
> **정합:** `docs/strategy/organism_architecture_v1.md`(§9 확장 모델) · 대표+김실장(GPT) 정리(2026-07-07).

## 남은 핵심 3덩어리
1. **공개 화면 정비** — 홈 인덱스 전환(랜딩 폐기·auth-modal 전역로드 금지=로그아웃 사고 재발 0) + 간단 푸터(최소 고지) + app.html 제목·noindex. 문구는 대표 결정 자리.
2. **속도** — 홈 JS 분리 계속(안전 단위·브라우저 검증·revert 단위).
3. **피드백 수집 구조(신규)** — "좋다/별로다"가 아니라 **검색어·안 나온 검색(0건)·필요했던 자료**를 모은다.

## ★ 피드백 수집 = 유기체의 시작 (총괄 판단)
검색 로그가 곧 **다음 채굴의 씨앗**이다. 사용자가 "안 나온 것"이 채굴팀이 다음에 채울 목록이 된다. **공개는 완성이 아니라 유기체의 첫 호흡** — 쓸수록 검색어가 쌓이고 → 채굴되고 → 사이트가 더 똑똑해지는 순환이 공개 순간 시작된다. (§9 확장 모델 = 그 순환이 조직마다 돌며 각 조직의 원세컨드로 적응.)

## 공개 직전 최종 게이트 점검 (필수)
- 알릴의무·스마트설계서·상품로드맵·영업노트·AI검색 = 게이트 안(비게이트 미노출) 재확인
- 어드민(#v-admin)·시안(design-preview)·관리자 페이지(pages/admin*) = 검색·비로그인 미노출 재확인
- 개인/팀/지점 격리 자료 = 검색 누출 0 재확인(RLS 실측)
