## 디자인 토큰 정리 트랙 (색+폰트 단일 소스 통합, 2026-06-12 결재 — 현재 트랙 정착 후 단독 PR)

**배경:** 두 차례 진단으로 구조적 토큰 부재 확인.
- **색**: app.html이 tokens.css 미링크 → 자체 인라인 `:root` 사본 보유. 게다가 `--s2`(#F4F4F6)=`--bg`(#F4F4F6) 동일 + `--bd`(#E4E4E7) 옅음 → 영역 구분 흐림.
- **폰트**: app.html에 font-size 스케일 토큰 0개. **font-size 하드코딩 663개**(em 246/rem 336/px 76). **em 246개가 중첩 곱연산 → 점진 축소**(진단 2026-06-12). 토큰 변경 아닌 **하드코딩 누적**.

**작업 범위 (단독 PR):**
1. **단일 소스화** — app.html을 `tokens.css`(또는 통합 토큰)에 연결. 인라인 `:root` 사본 제거/통합.
2. **font-size 스케일 토큰 도입** — `--fs-xs/sm/md/lg/xl` 등. 본문 기준 ≥0.875rem(14px).
3. **em → rem 스윕** — 곱연산 차단(중첩 축소 해소). 663개 하드코딩을 토큰으로 수렴.
4. **색 토큰 대비 보정** — `--s2`≠`--bg`, `--bd` 진하게(영역 구분 또렷).

**전제:** 회귀 광범위(전 화면 영향) → **현재 트랙(가입 위저드 등) 정착 후 단독 PR + 충분한 검수**. 진행 중 신규 UI는 **em 금지·rem·본문 ≥0.875rem** 준수(누적 차단). 색/폰트 또렷화는 이 트랙에서 일괄.
