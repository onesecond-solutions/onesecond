## 구현 결과 (①~⑤)
1. **업로드 실동작**: 기존 #519 경로 유지(Storage POST + DB insert). 라이브 실계정 검증은 대표/검수팀(AC = 새로고침·재로그인 후 유지).
2. **CRUD 완비(가드3)**:
   - 가드3-①: 폴더 삭제 = **빈 폴더만 단순 확인 / 비어있지 않으면 강한 확인 모달**(개수 표시 + 빨강 "모두 삭제"). 조용한 cascade 금지.
   - 가드3-②: 파일 삭제 = **Storage 객체 + DB 행 동시 실삭제**(고아·용량 방지). 폴더 삭제 시 하위 파일도 동일.
   - 가드3-③: 이름 변경 시 `search_text` 동시 갱신(파일 업로드·이름변경 모두 search_text=원본명).
   - 기존 라이브러리(평면) 삭제: **버그 아님 확인** — `library_delete_own` RLS 존재 + `deleteMyspaceItem` 0행 표면화 + UI 연결됨. 구체 증상 보고 시 재현·수정.
3. **폴더 UX**: 폴더 0개 시 "+ 첫 폴더 만들기" CTA. 더미/인수사 샘플폴더 0(중립).
4. **레이아웃 정상화(대표 확정)**: §4 고지 **풀폭 상단 1줄** / 3컬럼 **좌20·중30·우50**(flex 2:3:5) / 3컬럼 헤더 동일 상단선(좌"폴더"·중 현재폴더명·우"미리보기") / "+ 파일 올리기" = 중앙 헤더 우측. 900px↓ 세로 스택.
5. **§4 톤 완화**: "⚠️ …금지" → "안내 · 본인 영업 자산 보관용… 식별정보 파일은 올리지 말아 주세요." 의미 유지·위압 감소. 확인 체크 유지.
