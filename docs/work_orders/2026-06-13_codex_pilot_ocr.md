# 코덱스 파일럿 작업지시서 #01 — 소식지 PDF 본문 추출 파이프라인

> 🚨 **코덱스에게:** 이 지시서 범위만 수행한다. 시작 전 repo 루트 `AGENTS.md`의 "코덱스 경계"를 반드시 읽고 따른다. 범위 밖(제품 코드 수정·approved 승격·제품 데이터 쓰기·구버전 DB)은 절대 하지 않는다. 막히면 보고하고 멈춘다.

> **작성:** 2026-06-13 총괄팀장(Code) / **대상:** 코덱스(Codex Cloud) / **성격:** 파일럿(소량 검증 먼저, 대량 X)

## 0. 이 파일럿의 목표 (작게)

- **전체 534건을 한 번에 처리하지 않는다.** 먼저 **샘플 5건**으로 PDF→텍스트 추출 파이프라인을 만들고 품질을 검증한다.
- 검증 통과 후 대량 처리는 **별도 지시(Pro 전환 후)**. 이번엔 파이프라인 + 샘플 5건까지.

## 1. 배경 (사실)

- `newsletters` 테이블 = 보험사 소식지. 현재 534건. Supabase **private 버킷** `newsletters`에 PDF/PPTX 원본 저장.
- 2026-06 업로드분 42건 포함. 적재 시 `source_filename`에 한글 원본명 보존, Storage 키는 ASCII(uuid).
- 이 소식지들이 **검색기의 연료**다 — 본문이 텍스트로 추출돼야 검색에 걸린다. 현재는 본문 미추출 상태.

## 2. 할 일 (순서대로)

### STEP 1 — 실측 (추측 금지)
1. `newsletters` 테이블 **스키마를 직접 SELECT로 확인**한다. 어떤 컬럼이 본문/OCR 상태를 담는지(예: `full_text`, `ocr_status`, `source_filename`, Storage 경로 컬럼) 실제 컬럼명을 파악한다.
2. 본문 추출이 안 된 행(미처리 status) **5건만** 샘플로 고른다. PDF 위주(PPTX 제외).
3. 파악한 스키마·샘플 5건 목록을 보고에 적는다.

### STEP 2 — 파이프라인 스크립트 작성
1. `scripts/` 하위에 PDF 본문 추출 스크립트를 작성한다(언어·도구는 코덱스 판단, 단 재현 가능하게).
2. 동작: private 버킷에서 샘플 PDF 다운로드 → 텍스트 추출(가능하면 텍스트 레이어 우선, 없으면 OCR) → 결과를 `newsletters`의 **본문 컬럼만 UPDATE**.
3. **가드 필수:** 신버전 DB(`pdnwgzneooyygfejrvbg`)만 · 멱등(이미 처리된 행 skip) · 부분 실패 시 성공분 보존 · 한글 깨짐 방지(UTF-8) · 처리 로그 출력.
4. 키는 환경 secret에서만 읽는다. 코드·로그·PR에 키 노출 0.

### STEP 3 — 샘플 5건 실행 + 검증
1. 샘플 5건에 파이프라인을 돌린다.
2. 추출 텍스트 품질을 자가 점검: 한글 정상? 본문 핵심이 들어왔나? 빈 추출·깨짐 없나?
3. 5건 각각 결과 요약(파일명·추출 길이·품질 한줄)을 보고에 적는다.

## 3. 산출물 (PR로 제출)

- `scripts/` 하위 파이프라인 스크립트
- 처리 로그/결과 요약 (PR 본문 또는 `docs/`에 짧은 리포트)
- DB 변경 = **샘플 5건의 본문 컬럼 UPDATE만** (그 외 테이블·컬럼·status 건드리지 않음)
- PR 제목 = `chore(codex): 소식지 OCR 파이프라인 파일럿(샘플 5건)`

## 4. 절대 경계 (재확인)

- ❌ 제품 코드(`app.html`·`pages`·`js`·`css`) 수정 — 손대지 않는다.
- ❌ `newsletters` 본문 컬럼 외 다른 테이블/컬럼 쓰기.
- ❌ `knowledge_entries` 적재(이번 파일럿 범위 아님 — OCR만).
- ❌ main 직접 push — 반드시 PR.
- ❌ 키 노출 / 구버전 DB.
- ❌ 5건 초과 대량 처리(이번엔 검증까지).

## 5. 완료 정의

- 파이프라인 스크립트 + 샘플 5건 본문 추출·UPDATE + 품질 보고가 **PR로 올라오면** 완료.
- **"완료" 단정 대신 보고로 종료** — 총괄팀장이 검수 후 대량 전환 여부를 판단한다.
