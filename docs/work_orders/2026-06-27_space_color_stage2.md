# 스페이스 색상 2단계 (예약, 미착수) — 사이드메뉴 색상 포함

> 대표님 지시(2026-06-27): **"스페이스별 사이드메뉴 색상도 얘기할 거니까 저장해두고."** → 논의·확정 후 착수.
> 기획팀장(Web) 색상 의뢰서의 남은 신호점. 1단계(#975)·닫힘바(#976) 라이브 완료 이후.

## 이미 완료 (라이브)
- 1단계 #975: 공간 토큰 정의(마이 coral·팀 green·지점 purple) + 카드/현황/배지 공간색 + 지점색 파랑→보라 교정 + 카드 면 흰색
- 닫힘바 #976: 배너 접힘 시 공간 soft 배경 + 좌측 line

## 2단계 남은 신호점 (의뢰서 §B·§D·§E)

1. **★ 사이드바 선택 메뉴 색상** (대표님이 "얘기할 거"라 한 항목 — 논의 선행)
   - 현재 `.nav.on{background:var(--s2);color:var(--ac)}`(app.html:181) = 무채색+인디고, 공간 분기 없음
   - 의뢰서 §B-1: 현재 선택 메뉴 = 좌측 바 3px(line) + 배경(soft) + 텍스트(text), 나머지는 무채색
   - 단 사이드 nav는 마이/팀/지점이 **각각 별 nav 항목**(`#nav-sp-my/team/branch`) → 선택된 항목에 그 공간색
   - **대표님과 논의 후 확정** (어떤 색 강도·범위)

2. **콘텐츠 상단 4px 띠** (의뢰서 §B-2) — 배너 위에 공간 line 색 가로 띠

3. **카피 교체** (의뢰서 §D) — "배너 접기·펼치기"(`_snBannerToggle` aria-label) → "마이/팀/지점 스페이스 소개 보기" (공간명 = 위치 인지)

4. **배너 헤더 배지 상시 노출** (의뢰서 §E) — "나만/우리 팀/지점 전체" 배지(`.sp-aud`)를 콘텐츠 상단에도(현재 사이드바 트리에만). 색+텍스트 쌍(색각이상 대응)

## 코드 위치 (조사 완료)
- 사이드 선택: `.nav.on`(181), nav id `#nav-sp-my/team/branch`(2411~2413)
- 상단 띠: `.sn-banner`(신설)
- 카피: `_snBannerToggle`·`.sn-banner-toggle` aria-label(2428 등)
- 배지: `.sp-aud`(15274, 이미 `--sc` 토큰), 배너 헤더에 재사용
- 색: 1단계에서 정의된 `--space-*-line/soft/text/badge` 토큰 그대로

## 원칙 (의뢰서)
- 면/카드 바탕은 무채색 유지(색은 신호점에만)
- 색값 하드코딩 금지 — 전부 공간 토큰 경유
- 다크모드: 현재 data-theme=light 고정 → 라이트 우선

**상태: 미착수. 사이드메뉴 색상은 대표님 논의 후 착수.**
