## 대표님이 준 최종 레이아웃 (그대로)

```
마이 스페이스
├─ 오늘의 현황 (개인)
├─ [내 작업 본체] 다이어리·개인자료·스크립트·메모   ← 지금 그대로
│
├─ 🟢 우리 팀 (요약)        ← 새로 추가
│   최근 공지 3 · 새 자료 2 · [팀 스페이스 전체보기 →]
│
└─ 🟣 우리 지점 (요약)      ← 새로 추가
    최근 공지 3 · 새 자료 4 · [지점 스페이스 전체보기 →]
```

## 착수 시 구현 근거 (이미 조사된 코드 현실 — glance-row 토론 A1~A4)

- **부품:** `.glance-row` 신규(한 줄: `[●공간색] [공간명] [카운트] [전체보기 →]`). 기존 `.sores`는 스니펫 2줄이라 부적합 → 신설.
- **위치:** `_snMyspaceShow`(app.html, 마이 스페이스 렌더)에서 `.sn-dash`(오늘의 현황)·작업 본체 **아래**에 팀/지점 글랜스 2줄 추가.
- **색:** 공간 토큰 — 팀 `--space-team-line`(green) / 지점 `--space-branch-line`(purple). (1단계에서 이미 정의됨)
- **카운트:** `_diaCount(url)` 범용 헬퍼 재사용.
  - 공지: `team_notices?scope=eq.team(또는 branch)&deleted_at=is.null` (현 `loadDiaryDash`는 scope 필터 없음 → **scope 필터 추가** 필요). guide는 별 테이블이라 자동 제외(필터 불필요).
  - 새 자료: `myspace_files?scope=eq.team/branch&created_at=gte.{기준}` **신규 쿼리** 필요. ※"새 자료" 기준 = 오늘? 마지막 방문(lastread)? → **대표/기획팀장 확정 필요**(토론 RQ3).
- **클릭("전체보기 →"):** 해당 스페이스로 점프(B안, 가벼움) — 팀=`_navTeamspace`/지점=`_navBranchspace`(시트 스페이스 페이지). 미리보기(우측 패널, A안)는 그릇 통일 후 2차.

## 선결(착수 전 확정)
1. "새 자료" 기준(오늘 vs lastread) — 토론 RQ3
2. 클릭 동작 = 전체보기(점프)만으로 1차 (미리보기는 2차) — 토론 RQ1
3. 카운트 실시간성(진입 1회면 충분) — 토론 RQ2

→ 위 3개는 glance-row 토론(`2026-06-26_glance-row-pattern.md`)에서 기획팀장·대표님이 정한 뒤 작업지시서로 전환.
