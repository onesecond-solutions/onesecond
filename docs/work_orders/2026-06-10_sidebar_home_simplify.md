# [작업지시서] 사이드바 + 홈 배너 단순화

> 발신: 기획실장(Web) / 수신: 총괄팀장(Code) / 승인: 대표(임태성) / 2026-06-10
> 구현: PR #537. 전략 기조 "감추기 > 만들기 / 단순화 = 보안".

## 결재 결과 (대표 확정)
- **A (배너 우측 카드):** QUICK/FIELD VOICE 제거 → 라이브러리/스크립트 2개로 (권장 채택)
- **B (스크립트 위치):** 사이드바 별도 유지 (권장 채택)
- **menu_settings_by_role 범위:** 기존 게이팅으로 즉시 단순화 (권장, DB 0). 테이블 신설은 별도 결재 관문.

## 1. 사이드바 (기존 role 게이팅)
- 노출(일반 GA): 홈 / 라이브러리 / 스크립트 — (구분선) — 요금제 — (구분선) — 관리자(admin만).
- 숨김(GA): 보험사 자료실 / Quick / 현장의 소리 / 함께해요 / 보험이슈 → `ga-hide` 클래스.
- CSS: `#app:not(.is-admin):not(.is-insurer) .nav.ga-hide{display:none}` — **GA만 숨김. admin·insurer는 기존 게이팅 유지**(하드코딩 0).
- "어드민" → "관리자" 개명, 최하단. 요금제 = 관리자 위(수익화 유지).
- 보험사 자료실 = insurer 레이어(ins-show) → GA 비노출(ga-hide), insurer·admin 유지.

## 2. 홈 배너 호버 포털
- 5카테고리 → **3(공지/라이브러리/스크립트)**. 도구·소식·정보 컬럼 제거. `.home-cols` 3컬럼·max-width 620.
- 호버 텍스트 방식 유지.

## 3. 배너 우측 카드 (결재 A)
- tool-pill = 라이브러리 / SCRIPTS 2개. QUICK/FIELD VOICE 제거. 안내문도 "라이브러리, 스크립트"로 정합.

## 4. 스크립트 (결재 B)
- 사이드바 별도 유지. 라이브러리 안 '내 스크립트'(저장 목록) ↔ 스크립트 페이지(실행 화면) 역할 분리.

## 5. menu_settings_by_role
- DB 테이블 = "연결 예정"(미구축). 이번엔 기존 role 클래스 게이팅으로 일반화(특정 팀·역할 하드코딩 0). 완전한 DB RBAC = 별도 결재 트랙.

## AC
- GA 사이드바 = 홈/라이브러리/스크립트/요금제 (+ admin은 관리자). 숨김 5종 안 보임.
- 홈 배너 = 공지/라이브러리/스크립트 3개.
- 배너 우측 카드 = 라이브러리/스크립트.
- insurer 자료실 GA 비노출. admin은 전체 유지(운영).
- DDL/RLS·하드코딩 0.
