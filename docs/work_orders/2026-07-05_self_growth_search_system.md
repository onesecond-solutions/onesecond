# 자립 성장 검색 시스템 (Self-Growing Search) — 2026-07-05 대표 비전 박제

> **박제 사유:** 2026-07-05 세션에서 대표님이 확정한 핵심 시스템 비전. 채팅 기억 휘발 방지(원칙: 채팅기억 휘발→work_orders 박제). 총괄팀장이 자율로 돌리는 진실 원천.

## 🚨 환각 방지 = 절대 규칙 (다른 모든 것보다 우선, 2026-07-05 대표 확인)

**AI가 자체 환각으로 검색데이터를 업데이트하는 일은 절대 없다.** "똑똑해진다"는 = 실제 자료를 **더 잘 연결·구조화·조직**하는 것이지, 없는 사실을 만드는 게 아니다.

- **씨드 = 반드시 실제 자료** (대표 글·소식지·기획팀장 상품·김실장 고지 등 원본). 원본 없는 항목 생성 금지.
- **모르면 만들지 말고 제외** — 채굴팀장은 모호/불확실은 **문서화하고 배제**(억지 확장 금지). 예: 동의어 사전에서 모호한 단독약칭 6사·겹침어 제외 / 담보 v3 "금액 추정 0·강등 0·창작 0" audit.
- **동의어·통용어**는 실제 자료 + **검증 가능한 보험 통용어**(실비=실손 급, 누구나 아는)만. AI 창작 아님. 애매하면 제외.
- **대표 최종 승인 게이트** = 마지막 안전장치. 어떤 enrichment도 대표 승인 전 라이브 안 됨. 대표(도메인 전문가)가 틀린 건 반려. (실증: 2026-07-05 대표가 실손 그룹에 의료실비·의료비 직접 보강 = 게이트 정상 작동.)
- **총괄팀장 검증** = 배선 전 헤드리스 검증 + 채굴팀장 audit(제외 근거 문서) 확인.
- 사실(금액·조건·고지문항)은 **자료에 있는 것만**. 확정 아닌 건 "확인 필요"로 표시하지 확정처럼 쓰지 않음(예: 알릴의무 확정 5건만 세부문항, 코드/유형확인은 요약까지만).

→ 요약: **자료가 진실 원천, AI는 조직·연결·검증만. 없는 건 안 만들고, 애매하면 제외하고, 대표가 최종 승인.**

## 핵심 원리 (한 줄)

원세컨드에 들어오는 **모든 자료를 그대로 검색에 넣는 데서 끝내지 않고, 씨드(seed)로 삼아 검색 지능을 키운다.** 자료가 쌓일수록 검색이 더 똑똑해지는 **복리 구조** = "보험판 구글 그 이상".

대표님 표현: *"들어오는 모든 자료를 토대로 그대로 사용하는 게 아니라, 씨드로 활용해서 더 똑똑해지는 시스템"* / *"자립 성장 시스템"*.

## 두 개 층

### 1층 — RAW (이미 가동 중, 2026-07-05 실측 확인)
임태성 실장 계정으로 올리는 글이 **올리는 즉시 통합검색 노출** (Supabase RLS 라이브 fetch).
- 커버 소스: `posts`(board_type=qna 보험Q&A · insurer 원수사자료실 · community) · `scripts`(업무노트/스크립트) · `myspace_files`(자료실) · `team_notices`(팀·지점 공지) · `newsletters`(소식지) · `knowledge_entries`(지식엔진)
- → 대표님이 매일 글 쓰면 검색은 이미 자동으로 두꺼워짐.

### 2층 — 지능 (씨드 수확 = 이 시스템의 본질)
그 자료를 **채굴팀장이 채굴 → 동의어·태그·담보-상품 링크·질환 관계·구조 추출 → 검색 매칭이 똑똑해짐.**
- **증명된 첫 사례 (2026-07-05):** `cover_data_v3`(13담보) · `insurer_products_2607`(246상품) · `goji_records`(114고지)를 씨드로 → `search_synonyms.json`(48term/164alias) 수확 → "실비→실손", "뇌심→순환계", "355→3.5.5" 라이브 (PR #1124). 원자료를 넘어 검색이 **더 똑똑해진** 첫 결과물.

## 파이프라인 (자립 성장 루프)

```
대표 매일 글 등록 (업무노트·자료실·원수사자료실·Q&A)
  → ① RAW 라이브 검색 (자동, 이미 됨)
  → ② 채굴팀장 씨드 채굴 · 서치 조사 · 구조화
  → ③ 지능 자산화 (동의어/태그/담보-상품 링크 JSON)
  → ④ 총괄팀장 검색 배선 + 검수
  → ⑤ 대표 최종 승인 → 라이브
  ↺ 반복 = 복리로 똑똑해짐
```

## 역할 (자립 = 대표 최소 개입)

- **대표:** 매일 글쓰기 + 최종 승인만.
- **채굴팀장:** 씨드 채굴 · 서치 조사 · 구조화(JSON 산출). git 커밋 X.
- **총괄팀장:** 전 과정 총괄 · 검색 구현 · 검수 · 머지 · 배포 · 라이브 확인 · 보고. 틀 부족분 스스로 식별→채굴 지시→구현.

## 정직한 병목 (총괄팀장 해결 과제)

채굴팀장(Claude 데이터 워커)은 **라이브 Supabase 직접 조회 자격 없음.** → 임태성 글 내용이 채굴팀장에 닿아야 2층이 돈다.
- 해결: "글 내용 → 채굴팀장" 다리(주기적 export) 총괄팀장이 설계.
- 최초 1회 필요한 것(읽기 키 등)만 대표에 요청, 이후 무인 지향.
- 현재는 우리가 가진 데이터(246·13담보·114고지 + 라이브 검색에 붙은 지식·소식지·Q&A)로 씨드 수확 가능.
