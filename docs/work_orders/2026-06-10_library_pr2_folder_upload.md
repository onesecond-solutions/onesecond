## 구현 결과
1. **폴더 재귀 업로드**:
   - 드래그앤드롭: `webkitGetAsEntry()` → `_mvWalkEntry` 재귀(`readEntries` 배치 반복) → 폴더 계층 보존.
   - "+ 폴더" 버튼: `<input webkitdirectory>` → `webkitRelativePath` 분해로 계층 복원.
   - `_mvProcessUpload`: 폴더 경로 집합(접두 포함) 부모 먼저 생성 → path→folderId 매핑 → 파일을 해당 폴더에 업로드. 클라 uuid로 폴더 id 즉시 확보.
2. **빈 폴더**: 드래그 시 빈 디렉터리도 `out.dirs`에 보존·생성(파일 0개 허용). (picker는 브라우저 한계로 빈 하위폴더 미감지.)
3. **중복 폴더명**: `_mvUniqueName` — 같은 부모 안 동명 폴더 있으면 "이름 (2)" 자동. 조용한 병합/덮어쓰기 금지.
4. **search_text**: 파일명 검색 인덱스 갱신(가드3-③ 정합). 폴더는 `name` 컬럼이 검색 필드(별도 search_text 컬럼 없음 = DDL 회피).
5. **대량 처리 가드**:
   - 진행 표시(개수/완료 "n/total — 파일명").
   - **부분 실패**: 성공분 유지, 실패 목록·사유(`용량 초과`/`업로드 실패 nnn`/`DB 기록 실패`) 안내. 전체 롤백 금지. 폴더 생성 실패 시 하위 파일 스킵 카운트.
   - 한도/크기 초과: 413 → "용량 초과" 명시(상향은 별도 결재).
6. **빈 상태 CTA**: 폴더 0개 시 "+ 첫 폴더 만들기" (#521 적용분 유지).
