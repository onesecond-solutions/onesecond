## 0. 핵심 — "한 섹션 문제가 아니다"

이 규칙은 게시판 한 곳이 아니라, **"좌 목록(카드) / 우 미리보기" 패턴을 쓰는 전 영역에 공통 적용**한다.
검색 미리보기(`2026-06-19_search_preview_spec.md` §1, PR #842·#843)에서 검색 한정으로 잡았던 좌우 독립스크롤을, **전 영역 공통 규칙으로 승격**한 것.

적용 대상(좌-카드/우-미리보기 패턴을 쓰는 곳, 점검 필요):
- 스크립트 게시판 (board01 근거 화면)
- 검색 결과 미리보기 (이미 구현됨 — §1 기준)
- 자료실 / 공지 / 보험Q&A 등 동일 패턴을 쓰는 모든 목록+미리보기 화면

## 1. 화면에 스크롤은 3종이 섞여 있다 (혼동 방지)

| # | 스크롤 위치 | 처리 |
|---|---|---|
| 1 | **좌측(목록) 카드 스크롤** | ✅ 이것만 새로 다룬다 — 독립 스크롤 + 위로 가면 상단 영역 안으로 숨김 |
| 2 | **우측 미리보기 패널 내부 스크롤** (본문 길면 패널 안에서 위아래) | ❌ **절대 건드리지 마** — 그대로 둔다 (대표 명시) |
| 3 | 페이지(브라우저) 전체 스크롤 | 제거 — 목록 길이만큼 페이지가 통째로 늘어나면 안 됨 |

## 2. 동작 규칙

- **우측 미리보기 패널 = 위치 고정**. 좌측 카드를 아무리 스크롤해도 우측은 화면에서 안 움직인다.
- **우측 패널 내부 스크롤 = 유지 (손대지 않음)**. 본문이 길면 패널 안에서 자기 혼자 위아래로 읽힌다.
  - 멘탈모델: "액자(패널)는 벽에 박아두고, 액자 안 그림(본문)만 넘긴다." 위치 고정과 내부 스크롤은 양립한다(모순 아님).
- **좌측 카드 목록 = 독립 내부 스크롤**. 우측 길이에 끌려다니지 않는다.
- **좌측 카드는 위로 스크롤되면 상단 영역(헤더 / A영역) 안으로 빨려 들어가듯 숨겨진다.** 단순히 좌측 박스 안에서 사라지는 게 아니라, 고정된 상단 헤더 뒤로 가려져 사라지는 형태.
- **페이지 전체는 안 늘어난다** — 검색 §1과 동일(고정높이 flex + 좌/우 각자 overflow-y:auto, 전체 페이지 스크롤 제거).

## 3. 절대 금지

- ❌ 우측 미리보기 패널 내부 스크롤(위 #2) 제거·변경. 대표 명시 "제일 우측 스크롤 건드리지 마".
  - (참고: 검색에서도 #844로 우측 스크롤 제거했다가 #845로 원복한 전례 있음 — 우측 스크롤은 살린다.)
- ❌ 한 화면만 고치고 끝내기. 같은 패턴 전 영역에 일관 적용.
- ❌ 좌측이 길어질 때 페이지 전체가 통째로 늘어나는 구조.
