# 검색/책상 이원화 — 정식 전환 로드맵 (2026-06-21 새벽, 대표님과 설계 대화)

> START PROTOCOL 진입 시 읽는 자리. "검색은 검색, 내 일은 책상" 이원화 작업의 진실원천.
> 배경: 2026-06-21 새벽 세션. board01 스크롤 문제에서 출발 → 메뉴 이중구조 → IA 전면 재설계로 확장. 시안으로 방향 확정("이게 내 책상이야").

## 확정된 큰 그림 (메모리 [[project_floor_search_sheet_desk_ia]])

- **바닥(메인 화면) = 검색** — 정보 찾는 공용 인프라.
- **바텀시트 = 내 책상** — 내 작업 공간(개인 + 소속). 없애지 않고 존재해야 하는 핵심.
- **좌측 메뉴(트리/사이드바) 없이** 책상 안에서 **카드를 눌러 다 펼친다**(드릴다운). 예: 공지 카드 → 지점·팀·원수사 공지가 그 자리서 펼쳐짐.
- 사용자 적은 지금이 이런 근본 개편의 골든타임 ([[strategy_structural_refactor_golden_window]]).

## 책상 첫 화면 = thisis 시안 구조

상단 요약 카드(오늘의 할 일 · 영업현황 · 공지) + 다이어리(캘린더) + 도구(다이어리·영업노트·개인자료실·스크립트·메모). 근거 이미지 `upgrade_20260521/thisis.png`. 카드 = 요약이자 진입(누르면 상세 드릴다운).

## "부드러움" 4원칙 (정식 톤 — "허여멀거니 하지 마라")

대표가 시안이 라이브보다 부드럽다고 느낀 객관적 이유 = **여백 · 묶음(카드 위계) · 절제(흰 바탕+포인트색 하나) · 일관(간격·색·radius 통일)**. 정식은 이 4가지를 라이브에 입히는 것. 라이브는 과밀·색많음·드리프트가 문제 ([[project_onesecond_ux_core_problem]]).

## 결정 사항

- **파이프라인 → 단어 변경** (어색·외래어). 후보 "영업 현황"/"고객 진행". **최종 미확정** (대표 선택 대기).
- **공지 = "초대하는 빈 상태"** — 팀/지점/원수사 공지 메뉴는 두되(미래 생태계 자리), 비면 mock 금지하고 "지점장·매니저가 올리면 여기 모입니다" 안내로 초대장 역할. 비중은 낮추고 실질은 원세컨드 공급 소식(보험이슈 등)이 채움. 이유: 공지는 "남이 올려야" 채워지는데 초기엔 공급자 없음(대표뿐).
- **단어 획일화가 뿌리** — 같은 걸 여러 단어로 부르던 것(바닥/메인/검색, 바텀시트/책상/스페이스/라이브러리)을 하나로. 용어 사전 확정이 1단계.

## 6단계 로드맵 (위에서부터)

| 순서 | 할 일 | 효과 | 비고 |
|---|---|---|---|
| 1 | 용어 확정 + 디자인 톤 정비(여백·색절제·일관 토큰) | 부드러움 기반 | 먼저 권장 |
| 2 | 책상 첫 화면 = 대시보드 카드 신설(할일·영업현황·공지·캘린더·도구) + 톤 적용 | 시안 첫화면 라이브 등장 | 회귀 위험 낮음(책상 안) |
| 3 | 카드 드릴다운(카드→상세, 공지→지점/팀/원수사) | "메뉴 없이 카드로" 실현 | |
| 4 | 메뉴 일원화(사이드바/바텀시트 트리 정리 — 검색=바닥, 작업=책상) | 이중 메뉴 소멸 | **가장 큼·위험**. 골든타임 활용 |
| 5 | 바닥 = 검색 홈 구축 | 바닥이 진짜 검색 | 기존 병목 |
| 6 | 실데이터 배선(할일·영업현황·캘린더 → 영업노트/캘린더 DB) | 목업→진짜 | 영업노트 import 선행 |

추천: **1·2부터** (낯섦 가장 빨리 줄이고 부드러움 가져옴, 안전). 4는 책상 자리잡은 뒤 신중히.
