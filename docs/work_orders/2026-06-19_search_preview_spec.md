# 검색 우측 미리보기 패널 — 확정 사양 (2026-06-19, 대표님 직접 지시)

> START PROTOCOL 진입 시 반드시 읽는 자리. 검색 미리보기 작업은 이 사양을 진실원천으로 삼는다.
> 교훈: 같은 미리보기 동작을 세션마다 다르게 만들어 대표님이 반복 지시 → 여기 박제로 일관성 고정.

## 1. 레이아웃·스크롤 (구현 완료 — PR #842·#843)
- 상단 헤더(검색창·AI 검색바·소스 탭·범위)는 **고정**(2열 컨테이너 밖, `.sv-head`).
- 좌측 결과 목록 = 독립 내부 스크롤(`.sv-left`).
- 우측 미리보기 = 독립 내부 스크롤(`.sv-right`, overflow-y:auto). #844(우측 스크롤 제거)는 대표 지시로 #845 원복.
- 페이지 전체는 결과 수만큼 안 늘어남(데스크탑, `#v-home.on:has(#homeSearch.searching)` 고정높이 flex).
- 핵심 버그였던 것: `#app.home-search-preview #homeSearch{display:block}`(특이성 2,1,0)이 `#homeSearch.searching{display:flex}`를 눌러 flex가 안 됐음 → 데스크탑에서 `#app.home-search-preview #homeSearch.searching{display:flex}`(2,2,0)로 복원.

## 2. 소식지 미리보기 일관성 (구현 완료 — PR #846)
- 소식지 저장 2종: 옛 소식지=`source_pdf_url`(공개) / 2026-06 업로드 44건=`source_path`(newsletters private 버킷, source_pdf_url=null).
- `_searchOpen`이 source_pdf_url 없으면 `source_path`를 `_svSignUrl('newsletters', …)` 서명URL로 해석 → 6월 소식지도 미리보기 노출.
- **의존**: newsletters 스토리지 버킷이 인증 사용자 서명(SELECT) 허용해야 함. (2026-06-19 라이브에서 보임 확인 → 정책 이미 허용됨)

## 3. 미리보기 = "딸깍 허브" 일관 동작 (★ 대표님 확정 요구 — 구현 대상)

**모든 소스의 우측 미리보기 이미지/문서는 동일하게 동작한다 (자료실 이미지 = 기준):**

1. 미리보기 이미지(또는 소식지 PDF 첫 페이지) **위에 마우스를 올리면 돋보기(🔍) 아이콘**이 뜬다.
2. **클릭하면 전체화면 미리보기**(우리가 쓰던 라이트박스/전체화면)로 크게 본다.
3. 그 전체화면에 **딸깍 스킬**을 적용한다(자료실 이미지=`openLb`+4줄 딸깍 / 소식지=PDF 전체화면+`_nlDdalggak`).
4. 미리보기 아래: **딸깍 버튼 + "새 탭에서 열기"** 링크 (현행 유지 — 대표 OK).

- 자료실 이미지: 이미 `.sv-d-imgwrap`+`.sv-d-zoom`("🔍 클릭하면 크게")+`openLb`로 이 패턴 = **기준**.
- 소식지: 같은 패턴을 PDF 미리보기(`.sv-d-pdf`)에도 적용 = 이번 작업.
- 원칙: **미리보기 동작은 소스가 달라도 일관**(이미지든 PDF든 hover 돋보기 → 클릭 전체화면 → 딸깍).

## 4. 딸깍 스킬 참조
- `ddalggak` 스킬: 이미지 라이트박스 딸깍=4줄(다운로드/MY SPACE/복사/카드만들기), 콘텐츠 카드 팝오버=3줄.
- 소식지 딸깍 = `_nlDdalggak`(다운로드/MY SPACE/복사, 카드 제외).

## 5. 미해결(대표 "완전체 아니다")
- 미리보기 최종 모양(여러 페이지 표시·조판 등)은 일관성 확정 후 별도 디자인. 본 문서 §3까지가 현재 합의된 동작.
