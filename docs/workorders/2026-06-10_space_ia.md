## 작업
1. 상단 탭 2분할: 마이 스페이스(라이브러리/스크립트/메모/즐겨찾기/다이어리) · 공동 스페이스(팀방/지점방/보험사별 자료실/보험Q&A).
2. 팀원관리 = 매니저 전용 탭(ga_manager·ga_branch_manager 등). 일반 ga_member 비노출. 9역할 RBAC.
3. 공동 스페이스 하위: 팀방(team_id)/지점방(branch_id)/보험사별 자료실(회사별)/보험Q&A. 각 방 내부 탭: 공지/공유 자료/공유 스크립트/채팅 (구 단체방 공유 탭·실장 공지방 흡수).
4. 현장의 소리 폐지: 착수 전 실측(데이터 존재 여부 보고). 있으면 보험사별 자료실로 마이그레이션 후 폐지, 비었으면 바로 폐지. 사이드바·탭에서 제거.
5. RBAC 노출: 팀원=자기 팀방 / 지점장=지점방+산하 팀방 / 매니저=팀원관리 노출. 기존 role 게이팅 유지(menu_settings_by_role 미구축=별 트랙).
6. 보험사별 자료 시드(별 PR): author=원세컨드(author_type='platform', admin과 구분), status seed→published, 출처·면책, 전 회사 균등(중립).

## 단계
- 1차: 스키마 설계→대표 결재→DDL→탭 2분할 + 공동 스페이스 계층 + 현장의 소리 폐지·마이그레이션 + 팀원관리 게이팅.
- 시드(6)는 구조 완성 후 별 PR. 범위 크면 Code 단계 분리·보고.
- ※ 상단 탭은 사용자 첫 화면이라 체감 충격 큼 → 공동 스페이스 신설·안정화 후 탭 묶기 순서 권장.

## AC (라이브)
- 상단 탭 = 마이 스페이스 / 공동 스페이스 2개(+매니저면 팀원관리).
- 공동 스페이스 하위 팀방·지점방·보험사별 자료실·보험Q&A 노출, 각 방 안 탭 동작.
- 현장의 소리 사라짐, 기존 데이터 보존·이전(유실 0).
- 일반 사용자에게 팀원관리 안 보임, 매니저에게 보임(RBAC).
- 하드코딩 흔적 0.
