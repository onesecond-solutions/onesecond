## A. 비전 (대표님 + 전략실장)

소식지(원천) → 회사별 상품 라인업 → 상품별 가입기준(인수기준) 결합 → **전 보험상품 로드맵**.
핵심 = 상품명 모으기가 아니라 **동일 기준으로 좌표화**(회사→상품군→상품명→가입연령→기간→납입→금액→건강등급→병력기준→유병자형 여부→갱신→강점→판매상태→기준일→근거 소식지).
최종 = 고객조건 입력 → 가능 상품군·회사·일반/간편 구분·우선순위까지 내려가는 **상품 탐색 지도**.

검색기 진화: "자료 검색기" → "판매상품 검색기" → "고객조건 상품후보 검색기".

## B. 데이터 현실 (총괄팀장 조사, 2026-06-27)

| 요소 | 상태 |
|---|---|
| 회사 마스터 `insurers` | ✅ 31사 정형 |
| 소식지 `newsletters` | ✅ `company` 정형 / 상품·인수기준은 `full_text` 비정형 |
| 지식엔진 `knowledge_entries` | ✅ 구조 완비(type product/insurer·category 인수/상품·canonical_company) / ❌ product·insurer 적재 0건, 인수 7건(용어) |
| 상품 라인업 정형 테이블 | ❌ 없음 — 신규 필요 |

**결론: 구조 95% 준비 / 데이터 30% 적재.** "소식지에서 상품·인수기준을 뽑아 좌표화(채굴·추출)"가 본 작업의 8할.

## C. 5인 합심 역할

| 주체 | 역할 |
|---|---|
| 대표님 | 보험 도메인 진실(17년 현장)·좌표축 현실성·최종 결재 |
| 전략실장(GPT) | 좌표축 정의·우선순위·리스크/중립/법적 경계 |
| 기획팀장(Web) | 상품 탐색 UX(고객조건→후보 지도)·정보구조 |

| 검수팀(Chrome) | 데이터 정확성·인수기준 검증·라이브 검수 |

## D. 단계

```
소식지 → 상품 라인업·인수기준 추출(씨드, 임태성 계정 비공개)
       → 검수(검수팀)
       → published
       → 통합검색 합류 → ★모든 사용자가 상품·인수기준 검색
```
1. 상품 마스터 스키마(회사×상품×좌표축) — 총괄팀장
2. 소식지→상품 라인업 추출 — 총괄팀장(워크플로우)
3. 가입기준 보강 — **총괄팀장이 직접 보험회사 웹서치**(대표님 지시 2026-06-27) + Cycle3 채굴
4. 로드맵 탐색 UX/디자인 — 기획·디자인
5. 검색 배선(전 사용자 노출)

## E. ★ 종착점 — 전 사용자 검색 합류 (대표님 강조 2026-06-27)

상품 라인업·인수기준은 **공개 지식**(개인정보 아님) → 검수 후 published → 통합검색 소스에 연결 → **모든 사용자가 상품·가입기준 검색**. master_strategy "보험 검색 인프라"의 종착형.
