## 🚨 학습 / 결정
- **권한 핵심 = 작성이 아니라 가시성**(카톡 단톡방). 개인=개인만/팀=팀소속만/지점=지점소속만(처음부터 전제). 어드민=창시자라 글쓰기 주체 아님(메모리 `visibility_is_core_permission` 신설).
- **어드민 검수 함정** — 어드민 계정은 다 보여서 격리가 안 드러남. 가시성·RLS 검수는 role별 실계정으로. 어드민으로 통과 ≠ 사용자 통과(권한 다름) → 그래서 권한 검증 도구를 만듦.
- **내용 안 봄 ≠ 화면 안 봄** — 어드민이 사용자 글 내용을 평상시 안 보는 것과, 페이지 레이아웃을 검증하는 것은 별개. 화면은 보여야 함.
- **검증 자동화로 부담 흡수** — 실사용자 생기며 검증 시간 급증 → 9 role 수동 검증을 권한 검증 매트릭스(자동) + 동등클래스 대표 5개(눈)로. "개발자 많은 회사가 느린" 함정을 도구로 피함.
- **어드민/관리자 용어 구분**(메모리 정합). admin=어드민(창시자) / 관리자=매니저.
- **조회수 = 기록 컬럼 부재**(posts에 없음, "조회수 별 트랙" 미구현). 보여주려면 열람 시 +1 기능부터=라이브 게시판 동작 건드리는 별도 트랙.
- UI 톤: **카드가 크면 페이지만 늘어남 → 컴팩트 테이블 + 행클릭 상세**가 일목요연(대표 승인). 행 클릭 가능은 파란색+› 표시로 직관화.
