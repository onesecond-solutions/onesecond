- → 승인 = admin 몫. admin이 status pending→active + 지점 매핑을 해줘야 활성화 + 자료실 글쓰기 권한 부여.

### DB 실측 (신버전 pdnwgzneooyygfejrvbg, 팀장님 SQL Editor)

- `on_auth_user_created` 트리거 = `handle_new_user`(SECURITY DEFINER) → metadata에서 name/phone/company/role/insurer_id/branch_id/team_id/status 복사. role 8종 검증(admin 제외).

- RLS: `admin_select_all_users`(is_admin()) + `admin_update_all_users`(is_admin()) 이미 존재 → admin 전체 조회/수정 가능. **추가 정책 불필요.**

### ① approve_insurer_user RPC (신버전 생성 완료)
- `approve_insurer_user(p_user_id uuid, p_branch_ids uuid[]) RETURNS json`, SECURITY DEFINER, search_path='public'.
- 검증: 호출자 admin / 지점 최소1 / 대상 pending insurer / branch 실재 / 동시성 가드(`UPDATE ... WHERE id AND status='pending'` + NOT FOUND).
- 동작(atomic): users status→active + branch_id=첫 지점 / insurer_employee_branches 매핑(중복 NOT EXISTS) / json 결과(activated_branch_count 등).
- GRANT EXECUTE TO authenticated. 팀장님 SQL Editor 실행 → success.
- 설계 합의: 테이블 구조 변경 0(함수만) + 자기참조 회피 표준(SECURITY DEFINER) 정합.
