## 🔴 총괄 실수 박제 — 복명복창 게이트
- **"보험팩토리_클로드버전으로 라이브 시켜"를 코상무 카드 수정으로 오해** → 대표 "누가 그렇게 병신같이 이해했나". 처음부터 **별도 클로드 카피본**이었음.
- 대표 정문: **"왜 필요할 때는 복명복창 안 하고, 필요없을 때는 맘대로 하냐."** → 총괄이 기준을 뒤집어 적용(안전한 것엔 되묻고, 되돌리기 힘든/해석 갈리는 것엔 멋대로). **규칙: 되돌리기 힘들거나 해석 두 갈래 이상이면 → 실행 직전 "제가 이해한 건 X입니다" 한 줄. 읽기·로컬은 그냥.**
- 이후 지시(상세내역·스크롤·투입구·PDF/캡처)는 착수 전 이해 확정 후 진행 → 순조.

## 📌 기타 교훈
- **총괄 직접 vs 위임:** 대표가 총괄 직접 결과물을 원함(bojang-checkup 등). PDF/캡처는 위임했다가 대표 "총괄이 직접 투입해서 빨리 끝내라" → 작업자 중단·산출물 회수·총괄 직접 검수·완성·머지.
- **검수 오판:** 보장분석 확정→미리보기 "미반영"으로 오판(#previewBody만 보고 hero(pvHeroTitle) 놓침). 재현으로 정상 확인·정정.
- **홈으로/사이드바 먹통:** 로컬+라이브 코드 정상(diff 0, onclick 배선 정상, 클릭→전환 확인). **캐시 의심 → 대표 Ctrl+Shift+R 요청**(도구 캐시와 씨름 안 함=지난 교훈 정합).
- **조사 우선:** 투입구 착수 전 재사용 조사 — `ocr-extract`=fileUrl(Storage 필요)라 안 씀 / `gemini-customer-ocr`=base64(원본 미저장) / **pdf.js 클라 텍스트추출=서버 미전송**. 원본 미저장 원칙 유지.
