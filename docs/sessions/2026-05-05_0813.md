## 학습 raw (영구 메모리 후보 3건)

### 1. D-pre.8 sweep — Storage RLS 누락 영구 학습 (발견 #3)

D-pre.8 sweep은 public schema 인라인 EXISTS만 청산. **storage.objects RLS는 sweep 범위 밖**이라 옛 v1 시점 admin 3정책 (`bucket_id` 체크만, is_admin() 가드 부재)이 잔존. 5/4 D-4 K-1 보강 + 5/5 D-6 admin_read_all_logs 청산 누적이라도 storage 스키마는 미커버.

→ **D-final P-* 시점 Storage RLS 전수 sweep 1회 추가** 필수. D-9 Step 1.6 청산 후에도 다른 버킷이 신설될 시 동일 패턴 발생 가능.

### 2. group_name 명명 분기 — 옛 v1 코드 vs 신버전 DB 운영 (발견 #1)

옛 v1 admin.html 화면설정 4섹션 코드는 `banner_img` group_name으로 저장하는데 신버전 DB는 `page_banner` 명명. 즉 **옛 v1 코드와 별개로 다른 운영 트랙**(어드민 외 코드?)이 page_banner 그룹을 만들어 사용 중.

→ D-9 작업지시서 §1.2 Q-9 (a) 채택 시 단순 group_name 변경으로 청산 가능. 그러나 **추가 SQL 1건 필요** — page_banner의 정확한 key 패턴(`banner_<page>` vs `banner_img_<page>` vs 기타) raw 확인 후 코드 정합 보장.

### 3. menu_home = false 라이브 의도 vs 잔재 (발견 #2)

미해결 #17은 보험뉴스 메뉴 숨김만 다뤘으나, menu_home도 false 상태. 옛 admin이 운영 중 토글한 결과인지 또는 잔재인지 확인 필요. **D-9 진행 차단 아님** — admin이 D-9 화면설정 완성 후 즉시 ON 가능.
