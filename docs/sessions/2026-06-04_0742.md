### A. Quick 보험료 비교표 고도화 (#386~#391)
- **#386 제안서급 강화**: 표지(제목·고객명·작성일)+설계사 footer(os_user 자동) / 회사별 추천 배너(★수동지정/미지정 시 최저보험료 자동) / 리모델링 월차액 히어로(절감green·증가red)+연환산 / 가입현황 보장변화 마킹(강화·유지·신규·정리 설계사 선택, 색배지). 풀 보장분석 엔진 X(수동 마킹).
- **#387 리모델링 전·후 → "신규 제안용" 리네임**(모드버튼+표지). ※"카드스타일" 오해(전치표 유지+헤더만 정리가 정답)·"단순 텍스트 변경" 정정 학습.
- **#388 "리모델링 비교" 모드 신설**: 안1(전→후 양쪽 카드)+안2(항목별 상세표) 결합 = 표지+히어로+핵심3배지(보험료·보장·정리 자동판정)+전→후 카드+가입현황(현재/후/변화). 기존 전/후 인프라 재사용, 라벨 현재/리모델링 후. ⚠️작업 중 main 직접커밋 실수→push 전 발견·feature 브랜치로 이동·복구(origin 무손상).
- **#389 정리(팀장님 cal.png 지적)**: 회사별=신규제안용이라 **신규 제안용 탭 제거**(회사별+리모델링비교 2탭) / 하단 "이미지 저장" 버튼 → "직접 입력해서 사용하세요" 안내(다운로드 기능 제거). 출력=이미지 복사(카톡)만.
- **#390 PDF→JPG**: 이미지 변환기 확장(PNG·JPG·**PDF**). pdf.js(기존 loadPdfJs)로 각 페이지 렌더(2배·흰배경)→페이지별 JPG 그리드+카드별 복사/저장. 최대 20쪽. 기기 안 처리.
- **#391**: Quick 카드 문구 PNG→JPG → "PNG·JPG·PDF → JPG".
- 비교표 진화 과정: 단일→2분할(좌입력/우미리보기)→세로(전치)/가로 토글→균등분할·숫자 가운데→내용맞춤 자동폭→제안서급→리모델링 비교. (전부 카드만들기 html2canvas 패턴 재사용, Supabase저장0·자동견적아님·고객정보 이미지전용+고지문)

### B. 게시글 수정·삭제 일관화 (#392) + DB 대기
- 점검: 보험Q&A **4곳 전부**·단체방 **MY SPACE/매니저룸**에 수정·삭제 없었음(현장의소리 단체방·스마트·자료실 게시글엔 있었음).

- **⚠️ DB 대기**: posts qna DELETE/UPDATE RLS 미확인 → 라이브에서 **삭제·수정 시 403이면** `docs`/PR #392 본문의 `posts_delete_qna_own`·`posts_update_qna_own` 정책 SQL 신버전 실행. (단체방은 기존 정책 OK)
