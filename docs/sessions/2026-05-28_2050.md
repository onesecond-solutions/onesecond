### 1. PR #150 진단 격차 — CSS sticky 자료 잘못 인지
- 음수 margin이 sticky `top` 위치 영향 X (margin = layout / sticky top = scroll container content edge — 별개)
- 본인이 추정으로 "margin -24px = padding 영역 덮음" 결론 → 라이브 반영 후 팀장님이 격차 발견
- PR #151에서 `top: -24px` 직접 추가로 정정
- **재발 방지:** CSS sticky 자료 = 본인 추정 단정 금지, 라이브 검증 받기 전 "완료" 단정 금지

### 2. PR #154 git 흐름 격차 — feature 브랜치 생성 누락
- 본인이 `git checkout -b feature/...` 호출 안 함 → main 브랜치에서 commit + `push HEAD:fix/...`로 우회
- 결과적으로 PR 정상 머지·라이브 반영됐으나 본인 local main tracking 격차 발생 (`origin/fix/...`로 잘못 set-upstream)
- 즉시 `git branch --set-upstream-to=origin/main main`으로 복구
- **재발 방지:** 다음 PR부터 항상 `git checkout -b feature/...` 먼저, commit은 feature 브랜치에서
