이번 세션의 주제는 **"디자인 테스트 워크스페이스 신설 및 8개 페이지 시안 작업"**.

1. **디자인 원칙 수립**
   - 원칙 A: 브라운 95% → 5% 축소 (chrome 중성화, 강조 포인트에만 유지)
   - 원칙 B: 여백 확대 (홈 제외 — 팀장님 지적 반영)
   - 원칙 C: 사이드바 220 → 180px 축소 (한국어 가독성 유지하는 최소 폭)

2. **테스트 워크스페이스 신설** — `claude_code/design_test/`
   - 제품 코드 원본(pages, tokens.css, js, app.html)은 한 줄도 수정 안 함
   - 독립 실행 가능한 `v*.html` 구조 (file:// 더블클릭으로 미리보기)
   - 토큰 오버라이드는 각 테스트 파일 내부 `:root`에서만

3. **페이지별 시안 작성 (총 8개 페이지, 11개 HTML 파일)**
   - home v1 → v1-full (앱 쉘) → v2-full (프리미엄 리뉴얼, 확장 토큰 시스템)
   - scripts v1-full → v2-full (세로 탭 + 사이드바 180 + 슬라이드 애니메이션 복원)
   - board, myspace, quick, together v1-full (일괄 작업)
   - admin v1-full (Make.com 영감 2단 네비 + KPI/차트/타임라인/테이블 포함 분석 대시보드)
   - index v1-full (랜딩 재설계 — 프리미엄 sticky 네비, 듀얼 모니터 CSS 목업, Pain vs Solution 병합 섹션, 순서 교체(TOGETHER 먼저 → WHO 나중), 오버레이 없는 인라인 가입 폼)

4. **확장 디자인 시스템 제안**
   - tokens.css 확장본(`:root`)을 home/v2-full.html에서 프리뷰
   - Neutral/Brand/Accent 컬러 스케일 (10단계 등)
   - 타이포 xs~6xl, tracking, leading 스케일
   - Elevation 1~5 + 웜톤 그림자 2종
   - Easing 커브·duration·gradient 프리셋

5. **팀장님 피드백 실시간 반영**
   - 어제 작업한 홈 레이아웃(좌측 30px 패딩·hex -160px 상승)을 초기 오버라이드가 덮어쓴 문제 발견 → 즉시 수정
   - scripts v2의 세로 탭 활성 색상 "너무 밋밋" → solid fill + translateX + 웜 그림자로 강화
   - 슬라이드 애니메이션 누락 → grid-template-columns 전환 + JS 클릭 핸들러 추가
