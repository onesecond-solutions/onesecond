## 🚨 이번 세션 학습·원칙

- **권한은 상식으로** (메모리 `feedback_permission_common_sense` 신설): RLS·권한부여 같은 기술 용어로 어렵게 만들지 말 것. 마이=내 서랍(나만)/팀=우리 팀 방/지점=우리 지점 방, 윗사람일수록 넓게 봄, 삭제=작성자+그룹 최고위자. RLS는 뒤에서 자물쇠 채우는 구현일 뿐
- **이모지·이모티콘 금지** (메모리 `feedback_no_emoji_icons`): 라인 아이콘(Tabler)·SVG만. 트리 아이콘 전부 SVG `<symbol>`/`<use>` 스프라이트
- **모호한 디자인 요청 = 정석 판단+렌더 비교로 리드** (메모리 `feedback_design_lead_with_principles`): 글자만 반복하면 시간허비 → 헤드리스 Chrome 렌더로 검증
- **폰트 작다 지적 반복** = 16px는 브라우저 표준이나 노안+보조텍스트 13px 많아 작게 느껴짐. 부분(트리/설명문)은 사이즈 개별 상향함. **전체 기준(16px→17/18px) 상향은 별개 결정으로 분리 보류** — 추가로 "다 작다"면 루트 한 줄로 균일 상향
- **inplace DOM 이동** = 기존 라이브러리 화면(`#mys-vault`·`#mys-split`·`#mys-team`·`#mys-qna`)을 `#spContent`로 옮겼다가 탭 이탈 시 원위치 복귀(`_sp*Restore`, `mysTab`에서 전부 회수). 복제 아님 → 기능·데이터·id 그대로
- **헤드리스 검증** = `chrome.exe --headless=new`로 트리/레이아웃 mock 렌더 후 배포(우측 패널 JS init 타이밍 아티팩트는 무시 가능, 실제 함수는 단독 문법검증)
