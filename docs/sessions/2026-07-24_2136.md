## 이번 세션 한 줄
세션 시작 직후 대표가 라이브(`?view=axis-medical`)에서 **홈 버튼이 검색창 왼쪽이 아니라 상단바로 튀는** 걸 지적("아까 퇴근 전 고친 거 아니냐"). 어제(#1400·#1401·#1402) "완료·라이브"로 보고했으나 **라이브 실동작 미검증**이었음이 드러남. 근본원인을 바이트 단위로 확정하고 최소 수정으로 종결.

## 🔴 근본 원인 (라이브 렌더 + CSSOM + 바이트로 확정)
`css/knowledge-category.css` 주석 블록(라인 42~49) 안, **라인 45 끝 `#homeSearch/.hs-*/`의 `*/`가 CSS 주석을 조기 종료**시킴.
- 주석이 라인 45에서 일찍 닫히자 라인 46~49의 한글 설명이 CSS 파서에 쓰레기 선택자가 되고, **바로 뒤 라인 50 `#v-axis-medical .kc-searchslot{...position:relative;}` 규칙이 통째로 무효·드롭**.
- 결과: **의료실비(axis-medical) 슬롯만 `position:relative`를 잃어 `static`** → 홈 버튼(`position:absolute; left:calc(50% - 370px); top:9px`)의 offsetParent가 슬롯이 아닌 **BODY**로 잡혀 `top:9px`(=상단바 라인)에 붙음. 검색기(#homeSearch)는 슬롯에 정상 이동돼 중앙에 남음.
- 라인 51~55(암·뇌심장·수술비·간병·기타)는 정상 파싱 → **의료실비 하나만** 깨진 이유가 이것. (어제 세션노트 "의료실비만" 증상과 정확히 일치)
- 검증: `awk 'NR>=42&&NR<=50' | grep -o '\*/' | wc -l` = **2**(수정 전, 조기종료 1 + 의도종결 1) → 수정 후 **1**.
