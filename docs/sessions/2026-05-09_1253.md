## 사고 학습 2건 (spec v2 § 7-4 + 메모리 영구 명문화)

### 학습 1 — Supabase SQL Editor RUN 단위 세션 분리
- **본질:** 각 RUN은 독립 connection. RUN 종료 시 BEGIN 트랜잭션이 COMMIT 명시 안 되어 있으면 자동 ROLLBACK.
- **표준:** Pre-flight RUN → 메인 트랜잭션 RUN(BEGIN~COMMIT 모두 한 RUN) → 검증 RUN → 실패 시 수동 DROP. 함수 의존 시 트랜잭션 분리.
- **메모리 신설:** `supabase_sql_editor_session_isolation.md`

- **본질:** spec v2 § 7-2-3 SQL UUID 가정 → 라이브 posts.id = bigint 불일치 → FK 위반 → 자동 ROLLBACK
- **표준:** Pre-flight RUN에 참조 PK 타입 raw 검증 SQL 포함
