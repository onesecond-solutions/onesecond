## 교훈
- dual-write는 **양쪽 테이블의 폴리모픽 id/스코프 컬럼을 동시에** 점검(shares.item_id 사각).
- 알림/푸시는 **RLS 밖 별도 경로** — 테이블 RLS 정상이어도 push가 전체 broadcast 가능.
- 조직은 **team_id가 진실원**, `team`/`branch` 텍스트는 레거시 표시값(드리프트 원천). 화면별로 다른 컬럼 보면 인원 불일치.
- 계정 삭제는 **읽기 점검(콘텐츠·FK·auth/public) 후 소프트 우선**. 이름만으로 중복 단정 금지.
- 큰 구조 변경은 **플래그 격리(UAT) + 단계 PR** — 한 방 빅뱅 금지.
