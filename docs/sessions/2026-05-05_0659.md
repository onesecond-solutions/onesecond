## 학습 raw (영구 메모리 후보 6건)

### 1. RPC SECURITY DEFINER 5함수 일괄 표준 (D-2 별 트랙 #3 정합)

- 5 RPC (get_dau / get_wau / get_mau / get_feature_usage / get_retention_d30) 모두 동일 패턴
- `LANGUAGE plpgsql STABLE SECURITY DEFINER SET search_path = public`
- `IF NOT public.is_admin() THEN RAISE EXCEPTION 'permission denied' USING ERRCODE = '42501'` 가드
- `REVOKE ALL ON FUNCTION ... FROM PUBLIC` + `GRANT EXECUTE ... TO authenticated` + `REVOKE EXECUTE ... FROM anon`
- `COMMENT ON FUNCTION ...` 메타 추가
- 1 트랜잭션 (BEGIN ... COMMIT) 일괄 신설 — 부분 실패 시 자동 ROLLBACK

### 2. anon 차단 검증 = `has_function_privilege` 직접 조회 (Dashboard NOTICE 한계 회피)

- Supabase Dashboard SQL Editor는 RAISE NOTICE 메시지를 UI 결과 패널에 표시하지 않음
- 회피 패턴: `has_function_privilege('anon', 'public.get_wau()'::regprocedure, 'EXECUTE')` 직접 조회
- 결과 raw: `result = 'PASS-anon blocked'` 패턴으로 명확
- D-5 Step 3 사후 학습 — 향후 RPC 신설 시 본 패턴 표준 채택

### 3. admin 시뮬 호출 = `set_config('request.jwt.claims', ...)` 패턴

- Dashboard SQL Editor에서 `auth.uid()` 시뮬 시
- `SELECT set_config('request.jwt.claims', '{"sub":"<admin_uuid>","role":"authenticated"}', true);`
- 트랜잭션 LOCAL 스코프 (true 인자)
- `is_admin()` 가드 통과 → RPC 결과 반환
- D-5 Step 3 사후 학습 — 향후 RBAC 검증 SQL 표준

### 4. KST 자정 = `AT TIME ZONE 'Asia/Seoul'` (L-1 정합)

- DB 측: `(created_at AT TIME ZONE 'Asia/Seoul')::date` 변환
- JS 측: `setUTCHours(15,0,0,0)` (D-2 fetchContentKPI 패턴) — 두 측 정합 보장
- L7 UTC 시간대 이슈 발견: JS `new Date().toISOString().slice(0,10)`은 UTC today라 KST 새벽 검증 시 1일 차이
- 코드 수정 불요 — 라벨 표시만 UTC 기반, 산식은 KST 정합

### 5. 6메뉴 매핑 = features CTE + LEFT JOIN (UX 안정)

- L-4 (b) 채택 — script/home/together/myspace/library/notice 6종 강제 보장
- 라이브 데이터 분포 부족(target_type 1종)이라도 6종 모두 0으로 표시
- `WITH features AS (SELECT unnest(ARRAY['script',...]) AS feature)` + `LEFT JOIN counts c ON c.feature = f.feature`
- 향후 활동 분포 확장 시 자연 누적 — 시드 데이터 부족 회피 표준

### 6. last_seen_at 갱신 메커니즘 미구현 사후 발견 (별 트랙 부채 #1)

- D-pre.5에서 컬럼 신설(timestamptz NULL)만 했고 갱신 로직 부재
