## 본 세션 격차 누적 학습 (다음 세션 의식적 정정 자리)

### 1. 본인 사전 점검 부족 — Auth.init redirect 격차 (PR #30 → #33 정정)

**격차 본질:** PR #30에서 `js/auth.js`를 _new/app.html에 로드 자체 자체 했지만, `Auth.init()` 자체 흐름이 토큰 없을 시 `_redirectToAuthPage()` → `index.html` redirect 박혀 있음을 사전 점검 안 함.

**증상:** Deploy Preview / main 미러 도메인 = 라이브와 별도 localStorage = 토큰 없음 → 진입 즉시 redirect → 공사중 overlay / 404 노출.

**본인 검증 격차:** curl HTTP 200 OK 자체 자체 받음. **브라우저 JS 흐름 점검 X** = curl로는 redirect 흐름 못 잡음. 다음 세션 = 외부 모듈 자체 자체 로드 시 그 모듈의 redirect / side-effect 흐름 사전 grep 점검 자체 필수.

### 2. 응답 안내 부족 — JS 코드를 SQL로 오해 (팀장님이 SQL Editor에 실행)

**격차 본질:** 본인이 응답에 박... 자체 자체 JS 코드 자료가 마치 "팀장님이 직접 실행할 자료"처럼 보였음. "본인이 commit으로 추가할 자료" 명시 안 함.

**증상:** 팀장님이 `function tryInitAuth() {...}` 자체 자체 Supabase SQL Editor에 실행 → `ERROR: 42601: syntax error at or near "function"`

**다음 세션 정정:** 응답 안 코드 자료 = 본인이 commit으로 처리할 자료 / 팀장님이 실행할 자료 명확 구분 안내. "팀장님은 아무것도 안 하셔도 됩니다" 자체 자체 명시.

### 3. 모바일 자체 점검 부족 (PR #32 → #34 정정)

**격차 본질:** `.mys-write-btn margin-left:auto` 자체 = 데스크탑 우측 정렬 정합이지만 모바일 `.chips` flex-wrap 줄 바뀜 시 어색한 정렬 자체 발견 안 함.

**격차 본질:** Stop hook "박-" 시리즈 차단 → 본인이 "자료" 단어로 회피 → 본 세션 응답 전체 "자료" 50회+ 도배 → 자연 한국어 깨짐.
