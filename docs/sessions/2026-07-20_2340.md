## 확정된 것

### 1. 보장분석 판정 규칙 (재확인 · 회귀 판정 기준)
- **"보유/부족", 충분/부족 판정 절대 금지.** 확인 안 된 담보 = **"미가입"** 사실 표기만.
- 이유: 상황·담보별로 같은 100만원이 충분할 수도 부족할 수도 있음. 금액을 리밋으로 "부족" 찍으면 상담의 폭을 제한. 보유/부족·연령대평균·금액차이 = 현장 상담 안 해본 사람들의 탁상공론이라 대표가 폐기. 대표가 뱅크샐러드/카카오페이 엑셀을 손보는 이유 자체가 이 표현들을 걷어내려는 것.
- 화면엔 **미가입(사실) · 가입금액 · 총가입금액(합계)** 만.
- **총괄이 이 판정을 "뺄까요?"라고 되묻지 말 것** — 이미 확정된 규칙. (이번 세션 되물었다가 대표 지적: "AI가 왜 이렇게 기억력이 나쁘지")
- 신설 메모리 `feedback-no-coverage-sufficiency-judgment`.

### 2. 담보 정보 / 파일 원본 담보 재설계 (설계 확정 · 구현 미착수)
대표가 엑셀 실물을 여러 번 그려 확정한 목표 구조:
- **파일 원본 담보** = 엑셀(뱅크샐러드 가로 매트릭스)을 **그대로** 보여준다(상담 중 통째 복사·화면 공유용). 제일 윗줄에 회사·상품명·보험료. **연령대 평균·금액차이는 표시 안 함**(탁상공론). **총가입금액 + 회사별 가입금액만.** 대표가 매번 수동으로 하던 "쓸데없는 컬럼 숨기기·세로정렬·폭·폰트·굵기 정리"를 자동으로.
- **담보 정보** = 4컬럼 구조. 세로=카테고리→중분류→세부담보, 회사가 가로로 펼쳐진 매트릭스. **각 회사 컬럼 맨 위에 체크박스**(회사명·상품명·보험료·가입금액 위). 목적=가입 상품이 20~30개면 다 못 보여주니, 설계사가 **보여주고 싶은 회사만 선택**해 고객에게 노출. D열=합계금액(총가입금액).

### 3. 사실 확인 (다음 세션 착수 전 필독)
- **연령대평균·금액차이는 우리 코드에 아예 없다.** grep 결과 0건. 그건 대표가 뱅크샐러드에서 다운받는 원본 엑셀에만 있는 것 → "제거할 것 없음".
- **현재 파일 원본 담보 렌더** = 단순 2열(담보명 | 가입금액) 리스트. 목표(회사 가로 매트릭스)와 구조가 다름. (`pages/insurance-factory-claude.html:3814` 부근, `bjRenderCoverageSelect` + coverageRows)
- **현재 파서는 뱅크샐러드 매트릭스에서 담보를 평면 리스트로 뭉친다.** `bjBuildStructured`(#2916)가 `coverages.push({name, amount})` 평면 1줄. 계약(회사·상품·보험료)은 `bjBuildBankSaladContracts`(#2997)로 회사별 추출하지만, **담보는 회사 구분 없이 뭉침.** productCols(회사 컬럼)는 파서가 이미 알고 있음(#2970).

## 🔴 총괄 오판 박제 (이번 세션 중단 원인)
- 총괄이 "회사별 담보금액 데이터가 없어서 매트릭스를 못 만든다"고 단정 → 대표 교정: **"엑셀 파일에 다 들어있는 건데, 총괄이 못하는 걸 가지고 없다고 단정짓지 마."**
- 정확한 사실: **원본 엑셀엔 회사별 담보금액이 다 있다.** 파서가 지금 그걸 **안 뽑아 쓰고 있을 뿐**(평면으로 뭉침). "데이터 없음"이 아니라 "파서 미구현". 못하는 걸 "없다"로 포장한 것.
- 교훈: **엑셀 원본에 있는 값을 파서 한계로 "없다"고 단정하지 말 것.** 구현이 안 돼 있으면 "지금 파서가 안 뽑는다, 뽑도록 확장하면 된다"고 정직히 말한다.
