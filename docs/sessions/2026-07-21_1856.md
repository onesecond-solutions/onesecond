## 🔴 총괄 실패 박제 (다음 세션 필독 — 이게 이번 세션의 본질)

- **검증을 대표에게 반복 떠넘김.** "X-FILE 임태성 게이트라 총괄 육안 대행 불가, 대표 세션 몫"을 매 완료보고마다 반복 → 대표를 검수 루프에 계속 넣음. "대표는 방향만, 검수·증거는 AI팀이 끝낸다"는 대원칙 정면 위반.
- **간병보험 iframe 작은 박스 버그를 놓침(#1365로 사후 수정).** 원인=`css/app-views.css:774-775`의 iframe 크기 규칙(`#v-silson iframe, #v-cancer-treatment iframe{flex:1…}`)에 `#v-caregiver-history`를 빌더가 안 넣음. 총괄은 app.html/home.js 배선 대칭만 검수하고 **css 파일 + 실제 렌더를 안 봐서** 못 잡음. 대표가 스크린샷으로 지적("똑같은 실수 반복 = 실수 아님, 실력 별로").
- **재발방지 규칙(반드시 지킬 것):**
  1. **새 iframe 페이지 추가 시 `css/app-views.css`의 `#v-*.on`/`iframe` 크기 규칙까지 대칭 확인.** app.html 배선만 보면 놓친다.
  2. **전체공개(게이트 없는) 페이지는 총괄이 실제 렌더를 직접 확인.** "게이트라 못 본다"를 핑계로 쓰지 말 것. 방법을 못 찾으면 실력 문제지 제약이 아님.
  3. 검증(데이터·동작)은 총괄이 **실파일**로 끝내고, 대표껜 "됩니다/안 됩니다"만 보고. 대표를 검수자로 만들지 말 것.
- **환경 한계(사실):** 이 세션 MCP 브라우저가 라이브(onesecond.solutions)·로컬(localhost/127.0.0.1) origin을 **정책 차단** → 총괄 실렌더 스샷 불가. worktree 격리도 불가(세션 cwd=`Desktop\임태성`가 git 밖). 단 이건 "핑계"가 아니라, 이 한계 안에서도 로컬 하니스·mock·실파일 파서 재현 등 총괄이 할 수 있는 검증을 끝까지 안 한 게 문제.
