## 본 세션 본인 격차 학습 (5건)

### 1. PR #155 먹통 사고 — 본인 추정 격차 8세션째 패턴
- `selectSite()` 안 신규 코드 (DOM 재배치 + `resetInsurerOtp` 호출) 중 하나가 throw → 함수 중단 → `authNextSignup()` 미호출 → signup-1 카드 클릭 통째 먹통
- 정적 점검만으로 정확한 throw 자료 확정 X (추정만)
- **재발 방지:** PR #157 = 함수 통째 try/catch 안전망 + DOM 재배치 / OTP 리셋 내부 try/catch 추가. 격차 자료 발생해도 함수 중단 X. 진단 자료는 `console.error`로 자동 노출.

### 2. revert 진입 — destructive 자료 결재 받기 정합 학습
- 라이브 먹통 발견 즉시 = Code 단독 revert 진입 X. 팀장님 결재 받은 후 진입 = 1~3분 회복.
- main 직접 push 금지 정합 = revert도 PR + 머지 흐름 정합 (PR #156).

### 3. "병렬 작업 진행 안 한 자료" — 결재 대기에 갇힘
- 팀장님 정정 안내 = Code가 결재 대기 자세에 갇혀 새 작업 진입 못 한 자료.
- **재발 방지:** 결재 대기 자료 = 결과 받기 전 의존 작업 진입 X / 다만 독립 작업 = 병렬 진입 가능. 결재 흐름 정리 + 병렬 가능 자료 동시 안내.

### 4. "자체" 어휘 도배 — 영구 메모리 있음에도 지속
- Stop hook 정정 안내 다수 발생. 답변 송출 전 자가 검열 누락 격차.
- **재발 방지:** 답변 송출 전 본인 grep 자가 점검 강화. 코드/문서 둘 다 적용.

### 5. "박힘" 어휘 — Stop hook 정정 다수
- 응답 본문 안 "박-" 시리즈 도배 격차. 평이한 한국어로 의식적 정정 자체.
- **재발 방지:** 송출 전 grep "박" 점검 추가.
