## 3. 영구 학습 6건 (RLS 자기 참조 회피 표준)

본 세션 사고로 영구 명문화된 학습. 향후 Phase D 잔여 섹션(D-2~D-8) RLS 작업 시 표준 적용:

1. **RLS USING/WITH CHECK 동일 테이블 SELECT 서브쿼리 절대 금지** — 무한 재귀(42P17) 위험
2. **admin/role 검증은 SECURITY DEFINER 함수 패턴 표준** — 함수 정의자 권한으로 RLS 우회
3. **DB 메타 검증(정의 raw + 카운트) 통과 ≠ 라이브 안전** — 반드시 라이브 검증까지 통과해야 안전 확정
4. **`admin_update_all_users`(UPDATE) 작동해도 같은 패턴 SELECT 추가 시 chain 형성으로 재귀** — 단일 cmd 작동만으로 안전 단정 금지
5. **Code 자체 "재귀 안전 ✅" 단정 결론 금지** — 사양상 안전해 보여도 PostgreSQL RLS 동작 메커니즘 1차 출처 확인 필수
6. **🆕 같은 테이블 다른 cmd(UPDATE/INSERT/DELETE) 정책에도 동일 패턴 잔존 가능 → 사전 검증 단계 자기 참조 sweep 필수** (점검 3 사후 발견 학습)
