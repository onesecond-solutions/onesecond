### DB — `public.script_usage` 신설 (Dashboard RUN 마감, 검증 통과)

```
id          bigint identity PK
user_id     text   NOT NULL                       -- = os_user.id (scripts.owner_id 동일 도메인)
script_id   bigint NOT NULL FK → scripts(id) CASCADE
action_type text   NOT NULL DEFAULT 'view'         -- view/copy/run + 향후 확장 (CHECK 미설정)
used_at     timestamptz DEFAULT now()              -- 집계·정렬 기준
created_at  timestamptz DEFAULT now()
```
- 인덱스 3방향: `(user_id,used_at)` 홈 / `(script_id,used_at)` 인기·기간 / `(action_type,used_at)` 통계
- RLS 3정책: `script_usage_select_own` / `insert_own` / `update_own` = 모두 `(auth.uid())::text = user_id`
- 확장 설계: 팀/지점 인기·추천·사용 통계의 기준 테이블. 소속은 `user_id → users` join(현재 소속 기준), denorm 안 함

```sql
-- 공용=admin 소유 / 일반계정 owner_id 행 보유 확인 (캐스팅)
SELECT s.owner_id, u.email, u.role, s.scope, s.is_active, count(*) AS cnt
FROM public.scripts s
LEFT JOIN public.users u ON u.id::text = s.owner_id
GROUP BY s.owner_id, u.email, u.role, s.scope, s.is_active ORDER BY cnt DESC;
```
