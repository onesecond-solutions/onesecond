## ⏳ 대표 판단 대기 / 결정 사항
1. **⭐ 실데이터 status 레거시 불일치(재확인):** 999명이 "신규"(레거시)로 저장돼 8단계와 안 맞음 → 상담 플로우 카운트 대부분 0(김나나 진행중 1명만 매칭). 데이터 정리(마이그레이션/별칭) = 실데이터 변경, 대표 승인 자리.
2. **Phase 3 calendar_events 자동 생성 = 보류:** 라이브 캘린더 CRUD(saveCalEvent 등) 완비돼 기술 가능하나, 대표 실 공유 캘린더에 자동 쓰기=부작용 → 임의 실행 안 함. "자동 일정 캘린더 반영 ON" 결정 + 이벤트 범위/타입 확정 필요. (오늘의 할 일은 이미 클라이언트로 표출.)
3. **Phase 4(병력 건강 콘텐츠) = 블록:** 건강지식 DB 부재 → 구현 불가(가짜 안 만듦). 지식 DB(JSON 협업) 방향 결정 선행. 원장 먼저·하드코딩 지양 대원칙.
4. **고객관리(청약 후) 화면 = 미존재:** app.html 주석·가격페이지 "CRM 준비중"만. Phase 2 이관 대상 화면 자체가 없음 → 별도 화면 신설 vs 상담관리 한 화면 통합 방향 결정 필요(총괄 권장=②통합, 하드코딩 지양).

## 교훈
- ★**미리보기 오버라이드가 실함수 미수정을 가림**: 하니스가 `_ciSelCard`를 오버라이드해 카드클릭=폼이 미리보기선 됐으나 라이브 `_ciSelCard`는 안 고쳐 안 됨(#1237). 미리보기서 데모한 동작은 실 함수도 반드시 반영 확인.
- ★**escape-quote 버그**: `_ciFormHtml` value 표현식(JS)에서 따옴표를 문자열처럼 `\'`로 이스케이프해 "Invalid or unexpected token"→상담관리 전체 깨짐. 문자열 안 vs JS 코드 구분 필수. 배포 전 미리보기 로드로 문법검증(추출 에러 0) 습관화.
- ★**Chrome 게이트 실검수 가능**: 대표 위임 시 임태성 로그인 Chrome로 직접 검수(읽기전용). 실데이터 파괴적 동작은 금지.
- ★**부작용·DB부재는 가짜로 안 채움**: calendar 자동쓰기·Phase4 지식DB는 임의 실행 대신 결재 대기로 정직하게 남김.
