## 🔍 점검 (수정 0 · 대표 방향 대기)
**v2 토큰(css/tokens-v2.css) 타이포 사이즈 점검 결과**
- **결론: 사이즈 값은 정상, 구조가 어지럽다.**
- **v2 토큰 파일은 앱에 미적용** — `tokens-v2.css`는 app.html에 **미링크**(app.html은 app-core.css+app-views.css만 로드). 랜딩(index.html) 전용, 앱 2단계 적용 안 됨. → v2 타이포는 앱 화면 영향 0.
- **앱 타이포 실효 진실 = app-core.css + tokens.css, 스케일 3벌 병존:**
  - rem `--fs-xs~xl` (app-core.css:16) = 셸(사이드바·네비).
  - px `--fs-caption~heading`(12·13·14·15·16·18·20, app-core.css:22) = 콘텐츠 기준선.
  - em `--text-tab~h2`(0.79~1.263em, tokens.css:73, content-area 19px 기준) = 뷰 CSS.
  - content-area 실제 px = font-scale.js 주입: 소 18 / 중 19 / 대 20 + 90/100/110/125% 컨트롤러(노안 정합). 값 위계 정상.
- **v2 미러 정합 = 값 100% 동일**(어긋남 0). tokens-v2.css는 실효본의 미러일 뿐.
- **구조 리스크 4건:** ⓐ 타이포 스케일 3~4벌 병존(단일 소스 아님) ⓑ `--fs-base`(rem) vs `--fs-body`(px) **같은 `--fs-` 접두어 충돌** ⓒ 낡은 주석("실효=app.html inline" → 실제는 2026-07-08 분리로 app-core.css 이동) ⓓ tokens-v2.css `--lh-tight` 중복(22행 1.4가 128행 1.2에 덮여 죽음).
- **총괄 판단:** 사이즈는 손댈 것 없음. 정리 대상은 "값"이 아니라 "구조". 단 타이포 토큰=전역 참조라 회귀 위험 큼 → 손대려면 별도 트랙(전수 사용처 조사→매핑→단계 교체) 신중히.
- **대표에게 던진 질문(미회신):** 점검 의도가 ⓐ 특정 화면 글자 크기 조정 / ⓑ v2 앱 2단계 실적용 / ⓒ 구조 혼란 정리 중 무엇인지 → 회신 시 그 방향으로 좁혀 설계안.
