## 학습 / 주의

- **OCR 과금 = 처리 파일 수에만 비례**(cron 빈도 무관). 5분/30분 비용 동일. 자료실 들어오며 반영속도 위해 5분 유지 권고(비용 동일·UX↑).
- 자료실 파일 OCR = 개인 파일 내용을 Gemini(구글)로 전송 → 고지 한 줄로 안내(자료함 기존 "식별정보 금지 + 확인 체크"에 추가). 검색 노출은 본인만(RLS).
- `myspace_files.search_text`는 업로드 시 파일명이 채워져 있어 'is null'로 미처리 판별 불가 → 별도 `ocr_status` 마커 컬럼 필요(그래서 DDL).
- 멱등 마킹: done(적재)/empty(빈추출=재시도 방지)/skip(office류 비대상). office(docx/xlsx/pptx)는 Gemini 직접 한계 → 후순위.
- CRON_SECRET 로테이션 시 cron에 박힌 값도 갱신 필요(메모리 `cron_secret_rotation` 절차).
