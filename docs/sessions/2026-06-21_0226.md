2. **#860 마이스페이스 스크롤 공용화** — 원인=`#v-myspace` 페이지-스크롤 오버라이드가 공용 `.split` 좌우 독립스크롤을 꺼둠. 오버라이드 제거 → 함께해요(#v-together)·현장의소리(#v-voice)와 동일하게 통일. 다이어리(#mys-today)·자료함(#mys-vault)만 자체 스크롤 보존. (사양: `docs/work_orders/2026-06-21_master_detail_scroll_pattern.md`)
3. **바텀시트엔 안 먹음 발견** — 스크립트가 바텀시트로 열리면 `#mys-split`이 `#spContent`로 inplace 이동(`_spSplitShow`) → `#v-myspace` 스코프가 안 통함. = 같은 콘텐츠가 사이드바·바텀시트 두 군데 = "수정 때마다 어디냐 구분" 문제.
4. **IA 전면 재설계로 확장** — 전체 지도 조사(Explore 3): 바텀시트 inplace 노드 / 위치 의존 CSS / 사이드바·바텀시트 진입 중복.
5. **⭐⭐큰 그림 확정**: **바닥(메인+사이드바)=검색**(공용 인프라) / **바텀시트=내 책상**(개인+소속 작업: 스크립트·메모·다이어리·자료실·팀방·지점방·Q&A·매니저방). "메뉴 통일"=물리 합치기 아닌 **역할 분리**(작업 콘텐츠는 책상 일원화, 사이드바 작업 항목 제거). **좌측 메뉴 없이 카드 드릴다운**(공지 누르면 지점/팀/원수사 펼침).
6. **시안 제작·확정** — `design-preview/desk-search-concept.html` (v2). 검색 바닥 + "내 책상 열기" 손잡이 → 바텀시트 슬라이드업 → thisis 대시보드(할 일·영업현황·공지 + 다이어리 캘린더 + 도구 5). 카드 클릭 드릴다운, 다이어리=월 캘린더. 대표 **"이게 내 책상이야" / "와우 이쁜데 좋아(시안으로)"** 확정.
