이번 세션은 **디자인 개편 회의 진행 중 기술 검증 5건**을 Claude Code에게 의뢰한 세션. 모든 항목은 "코드 수정 금지, 채팅 보고만" 조건으로 실행됨.

### 검증 1: SPA 구조 유지 여부
- 현재 app.html이 `pages/*.html`을 fetch + `innerHTML` 치환 + script IIFE 수동 실행 방식으로 라우팅 중임을 실측 확인 (`app.html:854 switchMenu`)
- 라우팅 방식: hash/pushState 아님. URL은 `app.html` 고정. `localStorage.selected_menu`로 새로고침 복원
- `appstate:ready` 부트 패턴이 10개 페이지 IIFE에서 일관 적용 중 (grep 17건 확인)
- 대안 (a) SPA 유지 / (b) Multi-page / (c) Vue·Svelte 비교 후 **(a) 유지 권고** — 디자인 개편 핵심(색 면적 축소·tokens.css 확장)은 SPA 그대로에서 90% 커버 가능

### 검증 2: UI 스케일 슬라이더 도입
- 현재 폰트 컨트롤러(가/소/중/대)는 `_fontSizeMap = {0.88:'18px', 1:'19px', 1.25:'20px'}` 로 content-area·overlay에만 적용 중 (`app.html:1333`)
- `js/font-scale.js`는 about·pricing 독립 페이지용
- tokens.css는 폰트만 em, spacing/radius/layout/shadow 전부 px (43건)
- 방식 (a) CSS zoom / (b) transform scale / (c) rem 전환 비교
- **방식 (a) zoom + localStorage + 6단계 권고** — 기존 토큰 수정 없이 `body.style.zoom` 한 줄, 롤백 쉬움
- 단계: 70/80/90/100/110/120% 6단계 (40~50대 사용자 인지 부담 고려)
- 폰트 컨트롤러는 새 슬라이더로 흡수·제거 권장. `os_font_scale` → `os_ui_scale` 마이그레이션 필요
- DB 컬럼 추가(`users.ui_scale`) 비권장 (DB 사고 직후라 회피). localStorage만으로 1차 구현

### 검증 3: 5/6 베타 재출시 일정 재검토
- 가용 시간: 4/27~5/1 평일 저녁 5일 × 2~3h + 5/2~3 주말 종일 2일 + 5/4~5 평일 저녁 2일 = **약 30~41시간**
- 디자인 개편 작업 추산: tokens.css 재설계, A1/A2 승화, B/C 색 면적 축소, scripts·board·admin 정리, 회귀 테스트 = **30~44시간**
- UI 스케일 추가 5h를 끼워 넣으면 버퍼 소진 → 회귀 테스트 시간 사라짐 → 품질 저하 위험
- **시나리오 A 권고**: 디자인 개편만 11일 집중 → 5/6 재출시 → UI 스케일은 5/9~10 별도 배포

### 검증 4: index.html Sticky 헤더 추가 가능성
- index.html `<header>`는 **이미 `position: fixed; top: 0; z-index: 100; height: 60px` + `backdrop-filter: blur(12px)` + `background: rgba(253,248,243,0.95)` + 스크롤 시 `.scrolled` 클래스 토글까지 구현 완료** (`index.html:30-43, 2267-2272`)
- "카카오뱅크식 반투명+블러" 효과는 이미 적용된 상태
- index.html은 자체 CSS 변수(`--brown-dark`, `--accent`, `--cream` 등) 사용 — `tokens.css` 미참조 (grep 확인). app.html과 완전 분리
- index.html 섹션 체계: A(헤더)/B/C/D/E/F/G/H/J. I 결번 (2026-04-23 요금제 삭제)
- 사용자가 부른 "A1"은 app.html 용어. index.html에서는 "A 영역"
- 방안 A(메뉴 추가) / B(2단 고정) / C(scroll-triggered) 비교
- **방안 A 권고** — 메뉴 5개 이하 + 모바일 햄버거. 데스크톱 2~3h + 햄버거 3~4h = 5~7h

### 검증 5: 헤더 기능 밀집 시 충돌·오류 가능성
- 현재 scroll 리스너는 `handleHeader` 1개뿐 (passive). 경량
- 메뉴 추가 시 **앵커 스크롤 offset 문제** 새로 발생 — 각 섹션에 `scroll-margin-top: 60px` 필수 (현재 코드에 없음)
- 드롭다운 hover는 모바일에서 작동 안 함 (터치는 hover 개념 없음)
- "스크롤 시 좌우 움직임" 표현은 카카오뱅크 실제 동작(상하 hide/show)과 다름 → **팀장님이 참고한 사이트 URL 확인 필요**. 해석 4가지 공존 중
- 신규 기능 시간 추산: 네비 5~7h + 스크롤 애니메이션 3~5h + 충돌 방지 테스트 2h = **10~14h** → 5/6 전 버퍼(4~6h) 초과
- **권고: 5/6 전에는 네비 + 햄버거 + 앵커 offset 보정까지만. 스크롤 애니메이션은 5/9~10 이후**
